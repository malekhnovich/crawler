<p><b>Implementation</b> is the realization of an application, or execution of a <a href="/wiki/Plan" title="Plan">plan</a>, idea, <a href="/wiki/Scientific_modelling" title="Scientific modelling">model</a>, <a href="/wiki/Design" title="Design">design</a>, <a class="mw-redirect" href="/wiki/Specification" title="Specification">specification</a>, <a href="/wiki/Standardization" title="Standardization">standard</a>, <a href="/wiki/Algorithm" title="Algorithm">algorithm</a>, or <a href="/wiki/Policy" title="Policy">policy</a>.</p>

<h2><span class="mw-headline" id="Industry-specific_definitions">Industry-specific definitions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Implementation&amp;action=edit&amp;section=1" title="Edit section: Industry-specific definitions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Computer_science">Computer science</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Implementation&amp;action=edit&amp;section=2" title="Edit section: Computer science">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In <a href="/wiki/Computer_science" title="Computer science">computer science</a>, an implementation is a realization of a <a class="mw-redirect" href="/wiki/Technical_specification" title="Technical specification">technical specification</a> or algorithm as a <a href="/wiki/Computer_program" title="Computer program">program</a>, <a class="mw-redirect" href="/wiki/Software_component" title="Software component">software component</a>, or other <a class="mw-redirect" href="/wiki/Computer_system" title="Computer system">computer system</a> through <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a> and <a href="/wiki/Software_deployment" title="Software deployment">deployment</a>. Many implementations may exist for a given specification or standard. For example, <a href="/wiki/Web_browser" title="Web browser">web browsers</a> contain implementations of <a href="/wiki/World_Wide_Web_Consortium" title="World Wide Web Consortium">World Wide Web Consortium</a>-recommended specifications, and software development tools contain implementations of <a class="mw-redirect" href="/wiki/Programming_languages" title="Programming languages">programming languages</a>.</p>
<p>A special case occurs in <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented programming</a>, when a <a class="mw-redirect" href="/wiki/Concrete_class" title="Concrete class">concrete</a> <a class="mw-redirect" href="/wiki/Class_(object-oriented_programming)" title="Class (object-oriented programming)">class</a> implements an <a class="mw-redirect" href="/wiki/Interface_(object-oriented_programming)" title="Interface (object-oriented programming)">interface</a>; in this case the concrete class is an <i>implementation</i> of the interface and it includes methods which are <i>implementations</i> of those <a class="mw-redirect" href="/wiki/Method_(object-oriented_programming)" title="Method (object-oriented programming)">methods</a> specified by the interface.</p>
<h3><span class="mw-headline" id="Information_technology">Information technology</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Implementation&amp;action=edit&amp;section=3" title="Edit section: Information technology">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Information_technology" title="Information technology">Information technology</a></div>
<p>In the <a href="/wiki/Information_technology" title="Information technology">information technology</a> <a href="/wiki/Industry" title="Industry">industry</a>, implementation refers to post-sales process of guiding a client from purchase to use of the software or hardware that was purchased. This includes requirements analysis, scope analysis, customizations, systems integrations, user policies, user training and delivery. These steps are often overseen by a project manager using <a href="/wiki/Project_management" title="Project management">project management</a> methodologies. Software Implementations involve several professionals that are relatively new to the knowledge based economy such as <a href="/wiki/Business_analysis" title="Business analysis">business analysts</a>, <a href="/wiki/Technical_analysis" title="Technical analysis">technical analysts</a>, <a class="mw-redirect" href="/wiki/Solutions_architect" title="Solutions architect">solutions architects</a>, and project managers.</p>
<p>To implement a system successfully, a large number of inter-related tasks need to be carried out in an appropriate sequence. Utilising a well-proven implementation methodology and enlisting professional advice can help but often it is the number of tasks, poor planning and inadequate resourcing that causes problems with an implementation project, rather than any of the tasks being particularly difficult. Similarly with the cultural issues it is often the lack of adequate consultation and two-way communication that inhibits achievement of the desired results.</p>
<h3><span class="mw-headline" id="Political_science">Political science</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Implementation&amp;action=edit&amp;section=4" title="Edit section: Political science">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Political_science" title="Political science">political science</a></div>
<p>In <a href="/wiki/Political_science" title="Political science">political science</a>, implementation refers to the carrying out of <a href="/wiki/Public_policy" title="Public policy">public policy</a>. <a class="mw-redirect" href="/wiki/Legislatures" title="Legislatures">Legislatures</a> pass laws that are then carried out by <a class="mw-redirect" href="/wiki/Public_servants" title="Public servants">public servants</a> working in <a href="/wiki/Bureaucracy" title="Bureaucracy">bureaucratic agencies</a>. This process consists of rule-making, rule-administration and rule-adjudication. Factors impacting implementation include the legislative intent, the administrative capacity of the implementing bureaucracy, interest group activity and opposition, and presidential or executive support.</p>
<h3><span class="mw-headline" id="Social_and_health_sciences">Social and health sciences</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Implementation&amp;action=edit&amp;section=5" title="Edit section: Social and health sciences">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Implementation is defined as a specified set of activities designed to put into practice an activity or program of known dimensions. According to this definition, implementation processes are purposeful and are described in sufficient detail such that independent observers can detect the presence and strength of the "specific set of activities" related to implementation. In addition, the activity or program being implemented is described in sufficient detail so that independent observers can detect its presence and strength."<sup class="reference" id="cite_ref-NIRN2014_1-0"><a href="#cite_note-NIRN2014-1">[1]</a></sup></p>
<h3><span class="mw-headline" id="Water_and_natural_resources">Water and natural resources</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Implementation&amp;action=edit&amp;section=6" title="Edit section: Water and natural resources">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In water and natural resources, implementation refers to the actualization of best management practices with the ultimate goals of conserving natural resources and improving the quality of water bodies.</p>

<h2><span class="mw-headline" id="Role_of_end_users">Role of end users</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Implementation&amp;action=edit&amp;section=7" title="Edit section: Role of end users">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>System implementation generally benefits from high levels of user involvement and management support. User participation in the design and operation of information systems has several positive results. First, if users are heavily involved in systems design, they move opportunities to mold the system according to their priorities and business requirements, and more opportunities to control the outcome. Second, they are more likely to react positively to the change process. Incorporating user knowledge and expertise leads to better solutions.</p>
<p>The relationship between users and information systems specialists has traditionally been a problem area for information systems implementation efforts. Users and information systems specialists tend to have different backgrounds, interests, and priorities. This is referred to as the user-designer communications gap. These differences lead to divergent organizational loyalties, approaches to problem solving, and vocabularies.<sup class="reference" id="cite_ref-Laudon2010_2-0"><a href="#cite_note-Laudon2010-2">[2]</a></sup> Examples of these differences or concerns are below:</p>
<h3><span class="mw-headline" id="User_concerns">User concerns</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Implementation&amp;action=edit&amp;section=8" title="Edit section: User concerns">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>Will the system deliver the information I need for my work?</li>
<li>How quickly can I access the data?</li>
<li>How easily can I retrieve the data?</li>
<li>How much clerical support will I need to enter data into the system?</li>
<li>How will the operation of the system fit into my daily business schedule?<sup class="reference" id="cite_ref-Laudon2010_2-1"><a href="#cite_note-Laudon2010-2">[2]</a></sup></li>
</ul>
<h3><span class="mw-headline" id="Designer_concerns">Designer concerns</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Implementation&amp;action=edit&amp;section=9" title="Edit section: Designer concerns">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul>
<li>How much disk storage space will the master file consume?</li>
<li>How many lines of program code will it take to perform this function?</li>
<li>How can we cut down on CPU time when we run the system?</li>
<li>What are the most efficient ways of storing this data?</li>
<li>What database management system should we use?<sup class="reference" id="cite_ref-Laudon2010_2-2"><a href="#cite_note-Laudon2010-2">[2]</a></sup></li>
</ul>
