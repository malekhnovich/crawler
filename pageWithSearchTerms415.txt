<h1 class="firstHeading" id="firstHeading" lang="ro">Inginerie software</h1>
<div class="mw-body-content" id="bodyContent">

<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">

<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="ro"><p><b>Ingineria software</b> (din <a class="mw-redirect" href="/wiki/Englez%C4%83" title="Engleză">engleză</a>: <i>software engineering</i>) este un domeniu ce implică proiectarea, crearea și întreținerea de <a href="/wiki/Software" title="Software">software</a> aplicând tehnologii și practici din <a href="/wiki/Informatic%C4%83" title="Informatică">informatică</a> (știința calculatoarelor), managementul proiectelor, <a href="/wiki/Inginerie" title="Inginerie">inginerie</a>, proiectarea interfețelor și a altor domenii.</p>
<h2><span class="mw-headline" id="Modele">Modele</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=Inginerie_software&amp;veaction=edit&amp;vesection=1" title="Modifică secțiunea: Modele">modificare</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Inginerie_software&amp;action=edit&amp;section=1" title="Modifică secțiunea: Modele">modificare sursă</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:302px;"><a class="image" href="/wiki/Fi%C8%99ier:Waterfall_model-ro.svg"><img alt="" class="thumbimage" data-file-height="600" data-file-width="800" height="225" src="//upload.wikimedia.org/wikipedia/commons/thumb/a/ac/Waterfall_model-ro.svg/300px-Waterfall_model-ro.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/a/ac/Waterfall_model-ro.svg/450px-Waterfall_model-ro.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/a/ac/Waterfall_model-ro.svg/600px-Waterfall_model-ro.svg.png 2x" width="300"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/Fi%C8%99ier:Waterfall_model-ro.svg" title="Extinde"></a></div>
<a href="/wiki/Proiectare_de_program" title="Proiectare de program">Proiectarea de program</a> se face după <a href="/wiki/Caiet_de_sarcini" title="Caiet de sarcini">caietul de sarcini</a> si rezultă în architectura de software.</div>
</div>
</div>
<p>Modelul standard de referință pentru dezvoltarea programelor de calculator este Modelul Waterfall. Acest model conține 4 etape în realizarea programelor de calculator:</p>
<dl>
<dd>a. Obținerea cerințelor – se scrie un <a href="/wiki/Caiet_de_sarcini" title="Caiet de sarcini">caiet de sarcini</a></dd>
<dd>b. Designul (<i>vezi <a href="/wiki/Proiectare_de_program" title="Proiectare de program">proiectare de program</a></i>) – se proiectează architectura de software</dd>
<dd>c. Programarea efectivă –</dd>
<dd>d1. Verificarea – dacă programul/biblioteca este conform cu caietul de sarcini</dd>
<dd>d2. Testarea – dacă funcționează stabil</dd>
<dd>e. Întreținerea –</dd>
</dl>
<p><i>Modelul Cascadă</i> (engl. Waterfall), care mai este încă folosit în <a href="/wiki/Statele_Unite_ale_Americii" title="Statele Unite ale Americii">Statele Unite</a> pentru proiectele destinate statului, are avantajul că fiecare fază a programării se poate gestiona separat. Astfel se poate plăti separat pentru îndeplinirea fiecărei din aceste etape. Un dezavantaj este faptul că dacă pe parcursul dezvoltării proiectului apar noi cerințe, acestea sunt greu de adăugat în etapa de design.</p>
<p>Un model mai bun este <i>modelul iterativ</i> de dezvoltare a programelor de calculator. În modelul iterativ proiectul este împărțit în iterații, fiecare durând între 2-3 luni. O iterație conține toate etapele modelului Waterfall, însă aceste etape sunt limitate la scopul iterației.</p>
<p>Un avantaj al modelului iterativ este faptul că tehnologia este testată mai rapid. De asemenea modificările cerințelor pot fi adăugate mai ușor.</p>
