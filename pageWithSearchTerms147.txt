<h1 class="firstHeading" id="firstHeading" lang="en">Null Object pattern</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"><span class="mw-redirectedfrom">  (Redirected from <a class="mw-redirect" href="/w/index.php?title=Null_object_pattern&amp;redirect=no" title="Null object pattern">Null object pattern</a>)</span></div>
<div class="mw-jump" id="jump-to-nav">

<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><div class="hatnote" role="note">"Null object" redirects here. For the concept in category theory, see <a class="mw-redirect" href="/wiki/Initial_object" title="Initial object">Initial object</a>.</div>
<p>In <a class="mw-redirect" href="/wiki/Object-oriented" title="Object-oriented">object-oriented</a> <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a>, a <b>Null Object</b> is an object with no referenced value or with defined neutral ("null") behavior. The Null Object <a href="/wiki/Design_pattern" title="Design pattern">design pattern</a> describes the uses of such objects and their behavior (or lack thereof). It was first published in the <a href="/wiki/Pattern_Languages_of_Programs#Publications" title="Pattern Languages of Programs"><i>Pattern Languages of Program Design</i> book series</a>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>

<h2><span class="mw-headline" id="Motivation">Motivation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=1" title="Edit section: Motivation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In most object-oriented languages, such as <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> or <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a>, <a href="/wiki/Reference_(computer_science)" title="Reference (computer science)">references</a> may be <a href="/wiki/Null_pointer" title="Null pointer">null</a>. These references need to be checked to ensure they are not null before invoking any <a class="mw-redirect" href="/wiki/Method_(computer_science)" title="Method (computer science)">methods</a>, because methods typically cannot be invoked on null references.</p>
<p>The <a href="/wiki/Objective-C" title="Objective-C">Objective-C language</a> takes another approach to this problem and does nothing when sending a message to <code>nil</code>; if a return value is expected, <code>nil</code> (for objects), 0 (for numeric values), <code>NO</code> (for <code>BOOL</code> values), or a struct (for struct types) with all its members initialised to <code>nil</code>/0/<code>NO</code>/zero-initialised struct is returned.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<h2><span class="mw-headline" id="Description">Description</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=2" title="Edit section: Description">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Instead of using a <a class="mw-redirect" href="/wiki/Null_reference" title="Null reference">null reference</a> to convey absence of an object (for instance, a non-existent customer), one uses an object which implements the expected interface, but whose method body is empty. The advantage of this approach over a working default implementation is that a Null Object is very predictable and has no side effects: it does <i>nothing</i>.</p>
<p>For example, a function may retrieve a list of files in a folder and perform some action on each. In the case of an empty folder, one response may be to throw an exception or return a null reference rather than a list. Thus, the code which expects a list must verify that it in fact has one before continuing, which can complicate the design.</p>
<p>By returning a null object (i.e. an empty list) instead, there is no need to verify that the return value is in fact a list. The calling function may simply iterate the list as normal, effectively doing nothing. It is, however, still possible to check whether the return value is a null object (e.g. an empty list) and react differently if desired.</p>
<p>The null object pattern can also be used to act as a stub for testing, if a certain feature such as a database is not available for testing.</p>
<h2><span class="mw-headline" id="Example">Example</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=3" title="Edit section: Example">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Given a <a href="/wiki/Binary_tree" title="Binary tree">binary tree</a>, with this node structure:</p>
<pre>
<b>class</b> node {
  node left
  node right
}
</pre>
<p>One may implement a tree size procedure recursively:</p>
<pre>
<b>function</b> tree_size(node) {
  return 1 + tree_size(node.left) + tree_size(node.right)
}
</pre>
<p>Since the child nodes may not exist, one must modify the procedure by adding non-existence or null checks:</p>
<pre>
<b>function</b> tree_size(node) {
  set sum = 1
  <b>if</b> node.left exists {
    sum = sum + tree_size(node.left)
  }
  <b>if</b> node.right exists {
    sum = sum + tree_size(node.right)
  }
  return sum
}
</pre>
<p>This however makes the procedure more complicated by mixing boundary checks with normal logic, and it becomes harder to read. Using the null object pattern, one can create a special version of the procedure but only for null nodes:</p>
<pre>
<b>function</b> tree_size(node) {
  return 1 + tree_size(node.left) + tree_size(node.right)
}
</pre>
<pre>
<b>function</b> tree_size(null_node) {
  return 0
}
</pre>
<p>This separates normal logic from special case handling, and makes the code easier to understand.</p>
<h2><span class="mw-headline" id="Relation_to_other_patterns">Relation to other patterns</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=4" title="Edit section: Relation to other patterns">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>It can be regarded as a special case of the <a href="/wiki/State_pattern" title="State pattern">State pattern</a> and the <a href="/wiki/Strategy_pattern" title="Strategy pattern">Strategy pattern</a>.</p>
<p>It is not a pattern from <i><a class="mw-redirect" href="/wiki/Design_Patterns_(book)" title="Design Patterns (book)">Design Patterns</a></i>, but is mentioned in <a href="/wiki/Martin_Fowler" title="Martin Fowler">Martin Fowler's</a> <i>Refactoring</i><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> and Joshua Kerievsky's<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> book on refactoring in the <i>Insert Null Object</i> <a class="mw-redirect" href="/wiki/Refactoring" title="Refactoring">refactoring</a>.</p>
<p>Chapter 17 is dedicated to the pattern in <a href="/wiki/Robert_Cecil_Martin" title="Robert Cecil Martin">Robert Cecil Martin's</a> <i>Agile Software Development: Principles, Patterns and Practices</i><sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<h2><span class="mw-headline" id="Alternatives">Alternatives</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=5" title="Edit section: Alternatives">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Extension_methods_and_Null_coalescing">Extension methods and Null coalescing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=6" title="Edit section: Extension methods and Null coalescing">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In some <a class="mw-redirect" href="/wiki/Microsoft_.NET" title="Microsoft .NET">Microsoft .NET</a> languages, <a href="/wiki/Extension_method" title="Extension method">Extension methods</a> can be used to perform what is called 'null coalescing'. This is because extension methods can be called on null values as if it concerns an 'instance method invocation' while in fact extension methods are static. Extension methods can be made to check for null values, thereby freeing code that uses them from ever having to do so. Note that the example below uses the <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a> <a href="/wiki/Null_coalescing_operator" title="Null coalescing operator">Null coalescing operator</a> to guarantee error free invocation, where it could also have used a more mundane if...then...else.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">// compile as Console Application, requires C# 3.0 or higher</span>
<span class="k">using</span> <span class="nn">System</span><span class="p">;</span>
<span class="k">using</span> <span class="nn">System.Linq</span><span class="p">;</span>
<span class="k">namespace</span> <span class="nn">MyExtensionWithExample</span> <span class="p">{</span>
    <span class="k">public</span> <span class="k">static</span> <span class="k">class</span> <span class="nc">StringExtensions</span> <span class="p">{</span> 
        <span class="k">public</span> <span class="k">static</span> <span class="kt">int</span> <span class="nf">SafeGetLength</span><span class="p">(</span><span class="k">this</span> <span class="kt">string</span> <span class="n">valueOrNull</span><span class="p">)</span> <span class="p">{</span> 
            <span class="k">return</span> <span class="p">(</span><span class="n">valueOrNull</span> <span class="p">??</span> <span class="kt">string</span><span class="p">.</span><span class="n">Empty</span><span class="p">).</span><span class="n">Length</span><span class="p">;</span> 
        <span class="p">}</span>
    <span class="p">}</span>
    <span class="k">public</span> <span class="k">static</span> <span class="k">class</span> <span class="nc">Program</span> <span class="p">{</span>
        <span class="c1">// define some strings</span>
        <span class="k">static</span> <span class="k">readonly</span> <span class="kt">string</span><span class="p">[]</span> <span class="n">strings</span> <span class="p">=</span> <span class="k">new</span> <span class="p">[]</span> <span class="p">{</span> <span class="s">"Mr X."</span><span class="p">,</span> <span class="s">"Katrien Duck"</span><span class="p">,</span> <span class="k">null</span><span class="p">,</span> <span class="s">"Q"</span> <span class="p">};</span>
        <span class="c1">// write the total length of all the strings in the array</span>
        <span class="k">public</span> <span class="k">static</span> <span class="k">void</span> <span class="nf">Main</span><span class="p">(</span><span class="kt">string</span><span class="p">[]</span> <span class="n">args</span><span class="p">)</span> <span class="p">{</span>
            <span class="kt">var</span> <span class="n">query</span> <span class="p">=</span> <span class="k">from</span> <span class="n">text</span> <span class="k">in</span> <span class="n">strings</span> <span class="k">select</span> <span class="n">text</span><span class="p">.</span><span class="n">SafeGetLength</span><span class="p">();</span> <span class="c1">// no need to do any checks here</span>
            <span class="n">Console</span><span class="p">.</span><span class="n">WriteLine</span><span class="p">(</span><span class="n">query</span><span class="p">.</span><span class="n">Sum</span><span class="p">());</span>
        <span class="p">}</span>
    <span class="p">}</span>
<span class="p">}</span>
<span class="c1">// The output will be:</span>
<span class="c1">// 18</span>
</pre></div>

<h3><span class="mw-headline" id="C.2B.2B">C++</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=8" title="Edit section: C++">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A language with statically typed references to objects illustrates how the null object becomes a more complicated pattern:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">class</span> <span class="nc">animal</span> <span class="p">{</span>
<span class="k">public</span><span class="o">:</span>
  <span class="k">virtual</span> <span class="kt">void</span> <span class="n">make_sound</span><span class="p">()</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span>
<span class="p">};</span>

<span class="k">class</span> <span class="nc">dog</span> <span class="o">:</span> <span class="k">public</span> <span class="n">animal</span> <span class="p">{</span>
  <span class="kt">void</span> <span class="n">make_sound</span><span class="p">()</span> <span class="p">{</span> <span class="n">cout</span> <span class="o">&lt;&lt;</span> <span class="s">"woof!"</span> <span class="o">&lt;&lt;</span> <span class="n">endl</span><span class="p">;</span> <span class="p">}</span>
<span class="p">};</span>

<span class="k">class</span> <span class="nc">null_animal</span> <span class="o">:</span> <span class="k">public</span> <span class="n">animal</span> <span class="p">{</span>
  <span class="kt">void</span> <span class="n">make_sound</span><span class="p">()</span> <span class="p">{</span> <span class="p">}</span>
<span class="p">};</span>
</pre></div>
<p>Here, the idea is that there are situations where a pointer or reference to an <code>animal</code> object is required, but there is no appropriate object available. A null reference is impossible in standard-conforming C++. A null <code>animal *</code> pointer is possible, and could be useful as a place-holder, but may not be used for direct dispatch: <code>a-&gt;make_sound()</code> is undefined behavior if <code>a</code> is a null pointer.</p>
<p>The null object pattern solves this problem by providing a special <code>null_animal</code> class which can be instantiated bound to an <code>animal</code> pointer or reference.</p>
<p>The special null class must be created for each class hierarchy that is to have a null object, since a <code>null_animal</code> is of no use when what is needed is a null object with regard to some <code>widget</code> base class that is not related to the <code>animal</code> hierarchy.</p>
<p>Note, that NOT having a null class at all is an important feature, in contrast to languages where "anything is a reference" (e.g. Java and C#). In C++, the design of a function or method may explicitly state whether null is allowed or not.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">// function which requires an animal instance,</span>
<span class="c1">// and will not accept null</span>
<span class="kt">void</span> <span class="nf">do_something</span><span class="p">(</span> <span class="k">const</span> <span class="n">animal</span><span class="o">&amp;</span> <span class="n">Inst</span> <span class="p">)</span> <span class="p">{</span>
  <span class="c1">// Inst may never be null here</span>
<span class="p">}</span>

<span class="c1">// function which may accept an animal instance or null</span>
<span class="kt">void</span> <span class="nf">do_something</span><span class="p">(</span> <span class="k">const</span> <span class="n">animal</span><span class="o">*</span> <span class="n">pInst</span> <span class="p">)</span> <span class="p">{</span>
  <span class="c1">// pInst may be null</span>
<span class="p">}</span>
</pre></div>
<h3><span class="mw-headline" id="C.23">C#</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=9" title="Edit section: C#">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>C# is a language in which the Null Object pattern can be properly implemented. This example shows animal objects that display sounds and a NullAnimal instance used in place of the C# null keyword. The Null Object provides consistent behaviour and prevents a runtime Null Reference Exception that would occur if the C# null keyword were used instead.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="cm">/* Null Object Pattern implementation:</span>
<span class="cm"> */</span>
<span class="k">using</span> <span class="nn">System</span><span class="p">;</span>

<span class="c1">// Animal interface is the key to compatibility for Animal implementations below.</span>
<span class="k">interface</span> <span class="n">IAnimal</span>
<span class="p">{</span>
	<span class="k">void</span> <span class="nf">MakeSound</span><span class="p">();</span>
<span class="p">}</span>

<span class="c1">// Animal is the base case.</span>
<span class="k">abstract</span> <span class="k">class</span> <span class="nc">Animal</span> <span class="p">:</span> <span class="n">IAnimal</span>
<span class="p">{</span>
	<span class="c1">// A shared instance that can be used for comparisons</span>
	<span class="k">public</span> <span class="k">static</span> <span class="k">readonly</span> <span class="n">IAnimal</span> <span class="n">Null</span> <span class="p">=</span> <span class="k">new</span> <span class="n">NullAnimal</span><span class="p">();</span>
	
	<span class="c1">// The Null Case: this NullAnimal class should be used in place of C# null keyword.</span>
	<span class="k">private</span> <span class="k">class</span> <span class="nc">NullAnimal</span> <span class="p">:</span> <span class="n">Animal</span>
	<span class="p">{</span>
		<span class="k">public</span> <span class="k">override</span> <span class="k">void</span> <span class="nf">MakeSound</span><span class="p">()</span>
		<span class="p">{</span>
			<span class="c1">// Purposefully provides no behaviour.</span>
		<span class="p">}</span>
	<span class="p">}</span>
	<span class="k">public</span> <span class="k">abstract</span> <span class="k">void</span> <span class="nf">MakeSound</span><span class="p">();</span>
<span class="p">}</span>

<span class="c1">// Dog is a real animal.</span>
<span class="k">class</span> <span class="nc">Dog</span> <span class="p">:</span> <span class="n">IAnimal</span>
<span class="p">{</span>
	<span class="k">public</span> <span class="k">void</span> <span class="nf">MakeSound</span><span class="p">()</span>
	<span class="p">{</span>
		<span class="n">Console</span><span class="p">.</span><span class="n">WriteLine</span><span class="p">(</span><span class="s">"Woof!"</span><span class="p">);</span>
	<span class="p">}</span>
<span class="p">}</span>

<span class="cm">/* =========================</span>
<span class="cm"> * Simplistic usage example in a Main entry point.</span>
<span class="cm"> */</span>
<span class="k">static</span> <span class="k">class</span> <span class="nc">Program</span>
<span class="p">{</span>
	<span class="k">static</span> <span class="k">void</span> <span class="nf">Main</span><span class="p">()</span>
	<span class="p">{</span>
		<span class="n">IAnimal</span> <span class="n">dog</span> <span class="p">=</span> <span class="k">new</span> <span class="n">Dog</span><span class="p">();</span>
		<span class="n">dog</span><span class="p">.</span><span class="n">MakeSound</span><span class="p">();</span> <span class="c1">// outputs "Woof!"</span>

		<span class="cm">/* Instead of using C# null, use the Animal.Null instance.</span>
<span class="cm">         * This example is simplistic but conveys the idea that if the Animal.Null instance is used then the program</span>
<span class="cm">         * will never experience a .NET System.NullReferenceException at runtime, unlike if C# null were used.</span>
<span class="cm">         */</span>
		<span class="n">IAnimal</span> <span class="n">unknown</span> <span class="p">=</span> <span class="n">Animal</span><span class="p">.</span><span class="n">Null</span><span class="p">;</span>  <span class="c1">//&lt;&lt; replaces: IAnimal unknown = null;</span>
		<span class="n">unknown</span><span class="p">.</span><span class="n">MakeSound</span><span class="p">();</span> <span class="c1">// outputs nothing, but does not throw a runtime exception        </span>
	<span class="p">}</span>
<span class="p">}</span>
</pre></div>
<h3><span class="mw-headline" id="Smalltalk">Smalltalk</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=10" title="Edit section: Smalltalk">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Following the Smalltalk principle, <i>everything is an object</i>, the absence of an object is itself modeled by an object, called <code>nil</code>. In the GNU Smalltalk for example, the class of <code>nil</code> is <code>UndefinedObject</code>, a direct descendant of <code>Object</code>.</p>
<p>Any operation that fails to return a sensible object for its purpose may return <code>nil</code> instead, thus avoiding the special case of returning "no object". This method has the advantage of simplicity (no need for a special case) over the classical "null" or "no object" or "null reference" approach. Especially useful messages to be used with <code>nil</code> are <code>isNil</code> or <code>ifNil:</code>, which make it practical and safe to deal with possible references to <code>nil</code> in Smalltalk programs.</p>
<h3><span class="mw-headline" id="Common_Lisp">Common Lisp</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=11" title="Edit section: Common Lisp">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In Lisp, functions can gracefully accept the special object <code>nil</code>, which reduces the amount of special case testing in application code. For instance, although <code>nil</code> is an atom and does not have any fields, the functions <code>car</code> and <code>cdr</code> accept <code>nil</code> and just return it, which is very useful and results in shorter code.</p>
<p>Since <code>nil</code> <b>is</b> the empty list in Lisp, the situation described in the introduction above doesn't exist. Code which returns <code>nil</code> is returning what is in fact the empty list (and not anything resembling a null reference to a list type), so the caller does not need to test the value to see whether or not it has a list.</p>
<p>The null object pattern is also supported in multiple value processing. If the program attempts to extract a value from an expression which returns no values, the behavior is that the null object <code>nil</code> is substituted. Thus <code>(list (values))</code> returns <code>(nil)</code> (a one-element list containing nil). The <code>(values)</code> expression returns no values at all, but since the function call to <code>list</code> needs to reduce its argument expression to a value, the null object is automatically substituted.</p>
<h3><span class="mw-headline" id="CLOS">CLOS</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=12" title="Edit section: CLOS">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In Common Lisp, the object <code>nil</code> is the one and only instance of the special class <code>null</code>. What this means is that a method can be specialized to the <code>null</code> class, thereby implementing the null design pattern. Which is to say, it is essentially built into the object system:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">;; empty dog class</span>

<span class="p">(</span><span class="nb">defclass</span> <span class="nv">dog</span> <span class="p">()</span> <span class="p">())</span>

<span class="c1">;; a dog object makes a sound by barking: woof! is printed on standard output</span>
<span class="c1">;; when (make-sound x) is called, if x is an instance of the dog class.</span>

<span class="p">(</span><span class="nb">defmethod</span> <span class="nv">make-sound</span> <span class="p">((</span><span class="nv">obj</span> <span class="nv">dog</span><span class="p">))</span>
  <span class="p">(</span><span class="nb">format</span> <span class="no">t</span> <span class="s">"woof!~%"</span><span class="p">))</span>

<span class="c1">;; allow (make-sound nil) to work via specialization to null class.</span>
<span class="c1">;; innocuous empty body: nil makes no sound.</span>
<span class="p">(</span><span class="nb">defmethod</span> <span class="nv">make-sound</span> <span class="p">((</span><span class="nv">obj</span> <span class="nb">null</span><span class="p">)))</span>
</pre></div>
<p>The class <code>null</code> is a subclass of the <code>symbol</code> class, because <code>nil</code> is a symbol. Since <code>nil</code> also represents the empty list, <code>null</code> is a subclass of the <code>list</code> class, too. Methods parameters specialized to <code>symbol</code> or <code>list</code> will thus take a <code>nil</code> argument. Of course, a <code>null</code> specialization can still be defined which is a more specific match for <code>nil</code>.</p>
<h3><span class="mw-headline" id="Scheme">Scheme</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=13" title="Edit section: Scheme">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Unlike Common Lisp, and many dialects of Lisp, the Scheme dialect does not have a nil value which works this way; the functions <code>car</code> and <code>cdr</code> may not be applied to an empty list; Scheme application code therefore has to use the <code>empty?</code> or <code>pair?</code> predicate functions to sidestep this situation, even in situations where very similar Lisp would not need to distinguish the empty and non-empty cases thanks to the behavior of <code>nil</code>.</p>
<h3><span class="mw-headline" id="Ruby">Ruby</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=14" title="Edit section: Ruby">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In <a href="/wiki/Duck_typing" title="Duck typing">duck-typed</a> languages like <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a>, language inheritance is not necessary to provide expected behavior.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
 <span class="k">class</span> <span class="nc">Dog</span>
   <span class="k">def</span> <span class="nf">sound</span>
     <span class="s2">"bark"</span>
   <span class="k">end</span>
 <span class="k">end</span>
 
 <span class="k">class</span> <span class="nc">NilAnimal</span>
   <span class="k">def</span> <span class="nf">sound</span><span class="p">(</span><span class="o">*</span><span class="p">);</span> <span class="k">end</span>
 <span class="k">end</span>

<span class="k">def</span> <span class="nf">get_animal</span><span class="p">(</span><span class="n">animal</span><span class="o">=</span><span class="no">NilAnimal</span><span class="o">.</span><span class="n">new</span><span class="p">)</span>
  <span class="n">animal</span>
<span class="k">end</span>

<span class="n">get_animal</span><span class="p">(</span><span class="no">Dog</span><span class="o">.</span><span class="n">new</span><span class="p">)</span><span class="o">.</span><span class="n">sound</span>
 <span class="o">=&gt;</span> <span class="s2">"bark"</span>
<span class="n">get_animal</span><span class="o">.</span><span class="n">sound</span>
 <span class="o">=&gt;</span> <span class="kp">nil</span>
</pre></div>
<p>Attempts to directly <a href="/wiki/Monkey_patch" title="Monkey patch">monkey-patch</a> NilClass instead of providing explicit implementations give more unexpected side effects than benefits.</p>
<h3><span class="mw-headline" id="JavaScript">JavaScript</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=15" title="Edit section: JavaScript">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In <a href="/wiki/Duck_typing" title="Duck typing">duck-typed</a> languages like <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>, language inheritance is not necessary to provide expected behavior.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kr">class</span> <span class="nx">Dog</span><span class="p">{</span>
  <span class="nx">sound</span><span class="p">(){</span>
    <span class="k">return</span> <span class="s1">'bark'</span><span class="p">;</span>
  <span class="p">}</span>
<span class="p">}</span>

<span class="kr">class</span> <span class="nx">NullAnimal</span><span class="p">{</span>
  <span class="nx">sound</span><span class="p">(){</span>
    <span class="k">return</span> <span class="kc">null</span><span class="p">;</span>
  <span class="p">}</span>
<span class="p">}</span>

<span class="kd">function</span> <span class="nx">getAnimal</span><span class="p">(</span><span class="nx">type</span><span class="p">)</span> <span class="p">{</span>
  <span class="k">return</span> <span class="nx">type</span> <span class="o">===</span> <span class="s1">'dog'</span> <span class="o">?</span> <span class="k">new</span> <span class="nx">Dog</span><span class="p">()</span> <span class="o">:</span> <span class="k">new</span> <span class="nx">NullAnimal</span><span class="p">();</span>
<span class="p">}</span>

<span class="p">[</span><span class="s1">'dog'</span><span class="p">,</span> <span class="kc">null</span><span class="p">].</span><span class="nx">map</span><span class="p">((</span><span class="nx">animal</span><span class="p">)</span> <span class="o">=&gt;</span> <span class="nx">getAnimal</span><span class="p">(</span><span class="nx">animal</span><span class="p">).</span><span class="nx">sound</span><span class="p">());</span>
<span class="c1">// Returns ["bark", null]</span>
</pre></div>
<h3><span class="mw-headline" id="Java">Java</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=16" title="Edit section: Java">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kd">public</span> <span class="kd">interface</span> <span class="nc">Animal</span> <span class="o">{</span>
	<span class="kt">void</span> <span class="nf">makeSound</span><span class="o">();</span>
<span class="o">}</span>

<span class="kd">public</span> <span class="kd">class</span> <span class="nc">Dog</span> <span class="kd">implements</span> <span class="n">Animal</span> <span class="o">{</span>
	<span class="kd">public</span> <span class="kt">void</span> <span class="nf">makeSound</span><span class="o">()</span> <span class="o">{</span>
		<span class="n">System</span><span class="o">.</span><span class="na">out</span><span class="o">.</span><span class="na">println</span><span class="o">(</span><span class="s">"woof!"</span><span class="o">);</span>
	<span class="o">}</span>
<span class="o">}</span>

<span class="kd">public</span> <span class="kd">class</span> <span class="nc">NullAnimal</span> <span class="kd">implements</span> <span class="n">Animal</span> <span class="o">{</span>
	<span class="kd">public</span> <span class="kt">void</span> <span class="nf">makeSound</span><span class="o">()</span> <span class="o">{</span>

	<span class="o">}</span>
<span class="o">}</span>
</pre></div>
<p>This code illustrates a variation of the C++ example, above, using the Java language. As with C++, a null class can be instantiated in situations where a reference to an <code>Animal</code> object is required, but there is no appropriate object available. A null <code>Animal</code> object is possible (<code>Animal myAnimal = null;</code>) and could be useful as a place-holder, but may not be used for calling a method. In this example, <code>myAnimal.makeSound();</code> will throw a NullPointerException. Therefore, additional code may be necessary to test for null objects.</p>
<p>The null object pattern solves this problem by providing a special <code>NullAnimal</code> class which can be instantiated as an object of type <code>Animal</code>. As with C++ and related languages, that special null class must be created for each class hierarchy that needs a null object, since a <code>NullAnimal</code> is of no use when what is needed is a null object that does not implement the <code>Animal</code> interface.</p>
<h3><span class="mw-headline" id="PHP">PHP</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=17" title="Edit section: PHP">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="mw-highlight mw-content-ltr" dir="ltr">

    <span class="k">public</span> <span class="k">function</span> <span class="nf">makeSound</span><span class="p">()</span> <span class="p">{</span>

<span class="p">}</span>

    <span class="k">public</span> <span class="k">function</span> <span class="nf">makeSound</span><span class="p">()</span> <span class="p">{</span>

    <span class="p">}</span>
<span class="p">}</span>

<span class="nv">$animalType</span> <span class="o">=</span> <span class="s1">'elephant'</span><span class="p">;</span>
<span class="k">switch</span><span class="p">(</span><span class="nv">$animalType</span><span class="p">)</span> <span class="p">{</span>
    <span class="k">case</span> <span class="s1">'dog'</span><span class="o">:</span>
        <span class="nv">$animal</span> <span class="o">=</span> <span class="k">new</span> <span class="nx">Dog</span><span class="p">();</span>
        <span class="k">break</span><span class="p">;</span>
    <span class="k">case</span> <span class="s1">'cat'</span><span class="o">:</span>
        <span class="nv">$animal</span> <span class="o">=</span> <span class="k">new</span> <span class="nx">Cat</span><span class="p">();</span>
        <span class="k">break</span><span class="p">;</span>
    <span class="k">default</span><span class="o">:</span>
        <span class="nv">$animal</span> <span class="o">=</span> <span class="k">new</span> <span class="nx">NullAnimal</span><span class="p">();</span>
        <span class="k">break</span><span class="p">;</span>
<span class="p">}</span>
<span class="nv">$animal</span><span class="o">-&gt;</span><span class="na">makeSound</span><span class="p">();</span> <span class="c1">// ..the null animal makes no sound</span>
</pre></div>
<h3><span class="mw-headline" id="Visual_Basic_.NET">Visual Basic .NET</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=18" title="Edit section: Visual Basic .NET">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The following NullObject pattern implementation demonstrates the concrete class providing its corresponding NullObject in a static field <code>Empty</code>. This approach is frequently used in the .NET Framework (<code>String.Empty</code>, <code>EventArgs.Empty</code>, <code>Guid.Empty</code>, etc.).</p>

<span class="k">Public</span> <span class="k">Class</span> <span class="nc">Animal</span>
    <span class="k">Public</span> <span class="k">Shared</span> <span class="k">ReadOnly</span> <span class="n">Empty</span> <span class="ow">As</span> <span class="n">Animal</span> <span class="o">=</span> <span class="k">New</span> <span class="n">AnimalEmpty</span><span class="p">()</span>

<h2><span class="mw-headline" id="Criticism">Criticism</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Null_Object_pattern&amp;action=edit&amp;section=19" title="Edit section: Criticism">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>This pattern should be used carefully as it can make errors/bugs appear as normal program execution.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>Care should be taken not to implement this pattern just to avoid null checks and make code more readable, since the harder to read code may just move to another place and be less standard - such as when different logic must execute in case the object provided is indeed the Null Object. The common pattern in most languages with reference types is to compare a reference to a single value referred to as null or nil. Also, there is additional need for testing that no code anywhere ever assigns null instead of the Null Object, because in most cases and languages with static typing, this is not a compiler error if the Null Object is of a reference type, although it would certainly lead to errors at run time in parts of the code where the pattern was used to avoid null checks. On top of that, in most languages and assuming there can be many Null Objects (i.e. the Null Object is a reference type but doesn't implement the <a href="/wiki/Singleton_pattern" title="Singleton pattern">singleton pattern</a> in one or another way), checking for the Null Object instead of for the null or nil value introduces overhead, as does the singleton pattern likely itself upon obtaining the singleton reference.</p>
