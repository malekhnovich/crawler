<h1 class="firstHeading" id="firstHeading" lang="en">Algorithm design</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><p><b>Algorithm design</b> is a specific method to create a mathematical process in solving problems. Applied algorithm design is <a href="/wiki/Algorithm_engineering" title="Algorithm engineering">algorithm engineering</a>.</p>
<p>Algorithm design is identified and incorporated into many solution theories of <a class="mw-redirect" href="/wiki/Operation_research" title="Operation research">operation research</a>, such as <a href="/wiki/Dynamic_programming" title="Dynamic programming">dynamic programming</a> and <a class="mw-redirect" href="/wiki/Divide_and_conquer_algorithm" title="Divide and conquer algorithm">divide-and-conquer</a>. Techniques for designing and implementing algorithm designs are algorithm design patterns,<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> such as <a href="/wiki/Template_method_pattern" title="Template method pattern">template method pattern</a> and <a href="/wiki/Decorator_pattern" title="Decorator pattern">decorator pattern</a>, and uses of data structures, and name and sort lists. Some current day uses of algorithm design can be found in internet retrieval processes of web crawling, packet routing and caching.</p>
<p>Mainframe programming languages such as <a href="/wiki/ALGOL" title="ALGOL">ALGOL</a> (for <i>Algo</i>rithmic <i>l</i>anguage), <a class="mw-redirect" href="/wiki/FORTRAN" title="FORTRAN">FORTRAN</a>, <a href="/wiki/COBOL" title="COBOL">COBOL</a>, PL/I, <a class="mw-redirect" href="/wiki/SAIL_programming_language" title="SAIL programming language">SAIL</a>, and SNOBOL are computing tools to implement an "algorithm design"... but, an "algorithm design" (a/d) is not a language. An a/d can be a hand written process, e.g. set of equations, a series of mechanical processes done by hand, an analog piece of equipment, or a digital process and/or processor.</p>
<p>One of the most important aspects of algorithm design is creating an algorithm that has an efficient runtime, also known as its <a href="/wiki/Big_O_notation" title="Big O notation">Big O</a>.</p>
<p>Steps in development of Algorithms</p>
<ol>
<li>Problem definition</li>
<li>Development of a model</li>
<li>Specification of Algorithm</li>
<li>Designing an Algorithm</li>
<li>Checking the correctness of Algorithm</li>
<li>Analysis of Algorithm</li>
<li>Implementation of Algorithm</li>
<li>Program testing</li>
<li>Documentation Preparation</li>
</ol>
