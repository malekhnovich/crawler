<h1 class="firstHeading" id="firstHeading" lang="en">Balking pattern</h1>

<p>The <b>balking pattern</b> is a <a class="mw-redirect" href="/wiki/Design_pattern_(computer_science)" title="Design pattern (computer science)">software design pattern</a> that only executes an action on an <a href="/wiki/Object_(computer_science)" title="Object (computer science)">object</a> when the object is in a particular state. For example, if an object reads <a class="mw-redirect" href="/wiki/ZIP_file_format" title="ZIP file format">ZIP</a> files and a calling method invokes a get method on the object when the ZIP file is not open, the object would "balk" at the request. In the <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> programming language, for example, an IllegalStateException might be thrown under these circumstances.</p>
<p>There are some specialists<sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Manual_of_Style/Words_to_watch#Unsupported_attributions" title="Wikipedia:Manual of Style/Words to watch"><span title="The material near this tag possibly uses too-vague attribution or weasel words. (November 2010)">who?</span></a></i>]</sup> in this field who think this is more of an <a href="/wiki/Anti-pattern" title="Anti-pattern">anti-pattern</a>, than a design pattern. If an object cannot support its <a class="mw-redirect" href="/wiki/API" title="API">API</a>, it should either limit the API so that the offending call is not available, or so that the call can be made without limitation it should:</p>
<ul>
<li>Be created in a "sane state"<sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="The text near this tag needs further explanation. (April 2015)">further explanation needed</span></a></i>]</sup></li>
<li>Not make itself available until it is in a sane state</li>
<li>Become a <a href="/wiki/Facade_pattern" title="Facade pattern">facade</a> and answer back an object that is in a sane state</li>
</ul>
<p></p>

<h2><span class="mw-headline" id="Usage">Usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Balking_pattern&amp;action=edit&amp;section=1" title="Edit section: Usage">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Objects that use this pattern are generally only in a state that is prone to balking temporarily but for an unknown amount of time.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2009)">citation needed</span></a></i>]</sup> If objects are to remain in a state which is prone to balking for a known, finite period of time, then the <a class="mw-redirect" href="/wiki/Guarded_suspension_pattern" title="Guarded suspension pattern">guarded suspension pattern</a> may be preferred.</p>
<h2><span class="mw-headline" id="Implementation">Implementation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Balking_pattern&amp;action=edit&amp;section=2" title="Edit section: Implementation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Below is a general, simple example for an implementation of the balking pattern.<sup class="reference" id="cite_ref-Grand_1-0"><a href="#cite_note-Grand-1">[1]</a></sup> As demonstrated by the definition above, notice how the "synchronized" line is utilized. If there are multiple calls to the job method, only one will proceed while the other calls will return with nothing. Another thing to note is the <code>jobCompleted()</code> method. The reason it is synchronized is because the only way to guarantee another thread will see a change to a field is to synchronize all access to it or declare it as volatile.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kd">public</span> <span class="kd">class</span> <span class="nc">Example</span> <span class="o">{</span>
    <span class="kd">private</span> <span class="kt">boolean</span> <span class="n">jobInProgress</span> <span class="o">=</span> <span class="kc">false</span><span class="o">;</span>

    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">job</span><span class="o">()</span> <span class="o">{</span>
        <span class="kd">synchronized</span><span class="o">(</span><span class="k">this</span><span class="o">)</span> <span class="o">{</span>
           <span class="k">if</span> <span class="o">(</span><span class="n">jobInProgress</span><span class="o">)</span> <span class="o">{</span>
               <span class="k">return</span><span class="o">;</span>
           <span class="o">}</span>
           <span class="n">jobInProgress</span> <span class="o">=</span> <span class="kc">true</span><span class="o">;</span>
        <span class="o">}</span>
        <span class="c1">// Code to execute job goes here</span>
        <span class="c1">// ...</span>
    <span class="o">}</span>

    <span class="kt">void</span> <span class="nf">jobCompleted</span><span class="o">()</span> <span class="o">{</span>
        <span class="kd">synchronized</span><span class="o">(</span><span class="k">this</span><span class="o">)</span> <span class="o">{</span>
            <span class="n">jobInProgress</span> <span class="o">=</span> <span class="kc">false</span><span class="o">;</span>
        <span class="o">}</span>
    <span class="o">}</span>
<span class="o">}</span>
</pre></div>
