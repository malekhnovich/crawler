<h1 class="firstHeading" id="firstHeading" lang="ru">Шаблон проектирования</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">Материал из Википедии — свободной энциклопедии</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Перейти к:					<a href="#mw-head">навигация</a>, 					<a href="#p-search">поиск</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="ru"><div class="dablink noprint">У этого термина существуют и другие значения, см. <a href="/wiki/%D0%9F%D0%B0%D1%82%D1%82%D0%B5%D1%80%D0%BD" title="Паттерн">Паттерн</a>.</div>
<p><b>Шаблон проектирования</b> или <b>паттерн</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <i><span lang="en" xml:lang="en">design pattern</span></i>) в <a href="/wiki/%D0%9F%D1%80%D0%BE%D0%B5%D0%BA%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B3%D0%BE_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D1%8F" title="Проектирование программного обеспечения">разработке программного обеспечения</a> — повторимая <a href="/wiki/%D0%90%D1%80%D1%85%D0%B8%D1%82%D0%B5%D0%BA%D1%82%D1%83%D1%80%D0%B0_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B3%D0%BE_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D1%8F" title="Архитектура программного обеспечения">архитектурная конструкция</a>, представляющая собой решение проблемы <a href="/wiki/%D0%9F%D1%80%D0%BE%D0%B5%D0%BA%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Проектирование">проектирования</a> в рамках некоторого часто возникающего <a href="/wiki/%D0%9A%D0%BE%D0%BD%D1%82%D0%B5%D0%BA%D1%81%D1%82" title="Контекст">контекста</a>.</p>
<p>Обычно шаблон не является законченным образцом, который может быть прямо преобразован в <a href="/wiki/%D0%9A%D0%BE%D0%BC%D0%BF%D1%8C%D1%8E%D1%82%D0%B5%D1%80%D0%BD%D0%B0%D1%8F_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B0" title="Компьютерная программа">код</a>; это лишь пример решения задачи, который можно использовать в различных ситуациях. <a href="/wiki/%D0%9E%D0%B1%D1%8A%D0%B5%D0%BA%D1%82%D0%BD%D0%BE-%D0%BE%D1%80%D0%B8%D0%B5%D0%BD%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Объектно-ориентированное программирование">Объектно-ориентированные</a> шаблоны показывают отношения и <a href="/wiki/%D0%92%D0%B7%D0%B0%D0%B8%D0%BC%D0%BE%D0%B4%D0%B5%D0%B9%D1%81%D1%82%D0%B2%D0%B8%D0%B5" title="Взаимодействие">взаимодействия</a> между <a href="/wiki/%D0%9A%D0%BB%D0%B0%D1%81%D1%81_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Класс (программирование)">классами</a> или <a href="/wiki/%D0%9E%D0%B1%D1%8A%D0%B5%D0%BA%D1%82_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Объект (программирование)">объектами</a>, без определения того, какие конечные классы или объекты приложения будут использоваться.</p>
<p>«Низкоуровневые» шаблоны, учитывающие специфику конкретного языка программирования, называются <a href="/wiki/%D0%98%D0%B4%D0%B8%D0%BE%D0%BC%D0%B0_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Идиома (программирование)">идиомами</a>. Это хорошие решения проектирования, характерные для конкретного языка или программной платформы, и потому не универсальные.</p>
<p>На наивысшем уровне существуют <b>архитектурные шаблоны</b>, они охватывают собой архитектуру всей <a href="/wiki/%D0%A1%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0" title="Система">программной системы</a>.</p>
<p><a href="/wiki/%D0%90%D0%BB%D0%B3%D0%BE%D1%80%D0%B8%D1%82%D0%BC" title="Алгоритм">Алгоритмы</a> по своей сути также являются шаблонами, но не проектирования, а <a href="/wiki/%D0%92%D1%8B%D1%87%D0%B8%D1%81%D0%BB%D0%B5%D0%BD%D0%B8%D0%B5" title="Вычисление">вычисления</a>, так как решают вычислительные задачи.</p>

<h2><span class="mw-headline" id=".D0.98.D1.81.D1.82.D0.BE.D1.80.D0.B8.D1.8F">История</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%A8%D0%B0%D0%B1%D0%BB%D0%BE%D0%BD_%D0%BF%D1%80%D0%BE%D0%B5%D0%BA%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F&amp;veaction=edit&amp;vesection=1" title="Редактировать раздел «История»">править</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%A8%D0%B0%D0%B1%D0%BB%D0%BE%D0%BD_%D0%BF%D1%80%D0%BE%D0%B5%D0%BA%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F&amp;action=edit&amp;section=1" title="Редактировать раздел «История»">править вики-текст</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>В <a class="mw-redirect" href="/wiki/1970-%D0%B5" title="1970-е">1970-е</a> годы <a href="/wiki/%D0%90%D1%80%D1%85%D0%B8%D1%82%D0%B5%D0%BA%D1%82%D0%BE%D1%80" title="Архитектор">архитектор</a> <a href="/wiki/%D0%90%D0%BB%D0%B5%D0%BA%D1%81%D0%B0%D0%BD%D0%B4%D0%B5%D1%80,_%D0%9A%D1%80%D0%B8%D1%81%D1%82%D0%BE%D1%84%D0%B5%D1%80" title="Александер, Кристофер">Кристофер Александр</a> составил набор шаблонов проектирования. В области архитектуры эта идея не получила такого развития, как позже в области программной разработки.</p>
<p>В <a href="/wiki/1987_%D0%B3%D0%BE%D0%B4" title="1987 год">1987 году</a> <a href="/wiki/%D0%91%D0%B5%D0%BA,_%D0%9A%D0%B5%D0%BD%D1%82" title="Бек, Кент">Кент Бэк</a> (Kent Beck) и <a class="mw-redirect" href="/wiki/%D0%9A%D0%B0%D0%BD%D0%BD%D0%B8%D0%BD%D0%B3%D0%B5%D0%BC,_%D0%92%D0%B0%D1%80%D0%B4" title="Каннингем, Вард">Вард Каннингем</a> (Ward Cunningham) взяли идеи Александра и разработали шаблоны применительно к разработке программного обеспечения для разработки графических оболочек на языке <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a>.</p>
<p>В <a href="/wiki/1988_%D0%B3%D0%BE%D0%B4" title="1988 год">1988 году</a> <a href="/wiki/%D0%93%D0%B0%D0%BC%D0%BC%D0%B0,_%D0%AD%D1%80%D0%B8%D1%85" title="Гамма, Эрих">Эрих Гамма</a> (Erich Gamma) начал писать докторскую диссертацию при цюрихском университете об общей переносимости этой методики на разработку программ.</p>
<p>В 1989—1991 годах Джеймс Коплин (James Coplien) трудился над разработкой идиом для программирования на <a href="/wiki/C%2B%2B" title="C++">C++</a> и опубликовал в 1991 году книгу Advanced C++ Idioms.</p>
<p>В этом же году Эрих Гамма заканчивает свою докторскую диссертацию и переезжает в <a class="mw-redirect" href="/wiki/%D0%A1%D0%A8%D0%90" title="США">США</a>, где в сотрудничестве с Ричардом Хелмом (Richard Helm), Ральфом Джонсоном (Ralph Johnson) и Джоном Влиссидсом (John Vlissides) публикует книгу <a href="/wiki/Design_Patterns" title="Design Patterns">Design Patterns — Elements of Reusable Object-Oriented Software</a>. В этой книге описаны 23 шаблона проектирования. Также команда авторов этой книги известна общественности под названием «Банда четырёх» (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a> <i><span lang="en" xml:lang="en">Gang of Four</span></i>, часто сокращается до <i>GoF</i>). Именно эта книга стала причиной роста популярности шаблонов проектирования.</p>
<h2><span class="mw-headline" id=".D0.9F.D0.BB.D1.8E.D1.81.D1.8B">Плюсы</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%A8%D0%B0%D0%B1%D0%BB%D0%BE%D0%BD_%D0%BF%D1%80%D0%BE%D0%B5%D0%BA%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F&amp;veaction=edit&amp;vesection=2" title="Редактировать раздел «Плюсы»">править</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%A8%D0%B0%D0%B1%D0%BB%D0%BE%D0%BD_%D0%BF%D1%80%D0%BE%D0%B5%D0%BA%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F&amp;action=edit&amp;section=2" title="Редактировать раздел «Плюсы»">править вики-текст</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>В сравнении с полностью самостоятельным проектированием, шаблоны обладают рядом преимуществ. Основная польза от использования шаблонов состоит в снижении сложности разработки за счёт готовых абстракций для решения целого класса проблем. Шаблон даёт решению своё имя, что облегчает коммуникацию между <a href="/wiki/%D0%9F%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%81%D1%82" title="Программист">разработчиками</a>, позволяя ссылаться на известные шаблоны. Таким образом, за счёт шаблонов производится унификация деталей решений: модулей, элементов проекта, — снижается количество ошибок. Применение шаблонов концептуально сродни использованию готовых библиотек кода. Правильно сформулированный шаблон проектирования позволяет, отыскав удачное решение, пользоваться им снова и снова. Набор шаблонов помогает разработчику выбрать возможный, наиболее подходящий вариант проектирования.<sup class="reference" id="cite_ref-.D0.9C.D0.B0.D0.BA.D0.BA.D0.BE.D0.BD.D0.BD.D0.B5.D0.BB.D0.BB.E2.80.942005.E2.80.94.E2.80.94100-101_1-0"><a href="#cite_note-.D0.9C.D0.B0.D0.BA.D0.BA.D0.BE.D0.BD.D0.BD.D0.B5.D0.BB.D0.BB.E2.80.942005.E2.80.94.E2.80.94100-101-1">[1]</a></sup></p>
<h2><span class="mw-headline" id=".D0.9C.D0.B8.D0.BD.D1.83.D1.81.D1.8B">Минусы</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%A8%D0%B0%D0%B1%D0%BB%D0%BE%D0%BD_%D0%BF%D1%80%D0%BE%D0%B5%D0%BA%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F&amp;veaction=edit&amp;vesection=3" title="Редактировать раздел «Минусы»">править</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%A8%D0%B0%D0%B1%D0%BB%D0%BE%D0%BD_%D0%BF%D1%80%D0%BE%D0%B5%D0%BA%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F&amp;action=edit&amp;section=3" title="Редактировать раздел «Минусы»">править вики-текст</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Хотя легкое изменение кода под известный шаблон может упростить понимание кода, по мнению <a href="/wiki/%D0%9C%D0%B0%D0%BA%D0%BA%D0%BE%D0%BD%D0%BD%D0%B5%D0%BB%D0%BB,_%D0%A1%D1%82%D0%B8%D0%B2" title="Макконнелл, Стив">Стива Макконнелла</a>, с применением шаблонов могут быть связаны две сложности. Во-первых, слепое следование некоторому выбранному шаблону может привести к усложнению программы. Во-вторых, у разработчика может возникнуть желание попробовать некоторый шаблон в деле без особых оснований.<sup class="reference" id="cite_ref-.D0.9C.D0.B0.D0.BA.D0.BA.D0.BE.D0.BD.D0.BD.D0.B5.D0.BB.D0.BB.E2.80.942005.E2.80.94.E2.80.94101_2-0"><a href="#cite_note-.D0.9C.D0.B0.D0.BA.D0.BA.D0.BE.D0.BD.D0.BD.D0.B5.D0.BB.D0.BB.E2.80.942005.E2.80.94.E2.80.94101-2">[2]</a></sup></p>
<p>Многие шаблоны проектирования в <a href="/wiki/%D0%9E%D0%B1%D1%8A%D0%B5%D0%BA%D1%82%D0%BD%D0%BE-%D0%BE%D1%80%D0%B8%D0%B5%D0%BD%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B5%D0%BA%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Объектно-ориентированное проектирование">объектно-ориентированном проектировании</a> можно рассматривать как <a href="/wiki/%D0%98%D0%B4%D0%B8%D0%BE%D0%BC%D0%B0_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Идиома (программирование)">идиоматическое</a> воспроизведение <i>элементов</i> <a href="/wiki/%D0%A4%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%BE%D0%BD%D0%B0%D0%BB%D1%8C%D0%BD%D0%BE%D0%B5_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5" title="Функциональное программирование">функциональных языков</a><sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup>. <a class="mw-redirect" href="/wiki/%D0%9F%D0%B8%D1%82%D0%B5%D1%80_%D0%9D%D0%BE%D1%80%D0%B2%D0%B8%D0%B3" title="Питер Норвиг">Питер Норвиг</a> утверждает, что 16 из 23 шаблонов, описанных в книге «<a href="/wiki/Design_Patterns" title="Design Patterns">Банды Четверых</a>», в <a class="mw-redirect" href="/wiki/Lisp" title="Lisp">Lisp</a> или <a href="/wiki/Dylan_(%D1%8F%D0%B7%D1%8B%D0%BA_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F)" title="Dylan (язык программирования)">Dylan</a> реализуются существенно проще, чем в <a class="mw-redirect" href="/wiki/%D0%A1%2B%2B" title="С++">С++</a>, либо оказываются незаметны<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup>. <a href="/wiki/%D0%93%D1%80%D1%8D%D0%BC,_%D0%9F%D0%BE%D0%BB" title="Грэм, Пол">Пол Грэхэм</a> считает саму идею шаблонов проектирования — <a href="/wiki/%D0%90%D0%BD%D1%82%D0%B8%D0%BF%D0%B0%D1%82%D1%82%D0%B5%D1%80%D0%BD" title="Антипаттерн">антипаттерном</a>, сигналом о том, что система не обладает достаточным уровнем <a href="/wiki/%D0%90%D0%B1%D1%81%D1%82%D1%80%D0%B0%D0%BA%D1%86%D0%B8%D1%8F_(%D0%B8%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%82%D0%B8%D0%BA%D0%B0)" title="Абстракция (информатика)">абстракции</a>, и необходима её тщательная переработка<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup>. Нетрудно видеть, что само определение шаблона как «<i>готового решения, но не прямого обращения к <a href="/wiki/%D0%91%D0%B8%D0%B1%D0%BB%D0%B8%D0%BE%D1%82%D0%B5%D0%BA%D0%B0_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Библиотека (программирование)">библиотеке</a></i>» по сути означает отказ от <a href="/wiki/%D0%9F%D0%BE%D0%B2%D1%82%D0%BE%D1%80%D0%BD%D0%BE%D0%B5_%D0%B8%D1%81%D0%BF%D0%BE%D0%BB%D1%8C%D0%B7%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5_%D0%BA%D0%BE%D0%B4%D0%B0" title="Повторное использование кода">повторного использования</a> в пользу <a href="/wiki/%D0%94%D1%83%D0%B1%D0%BB%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5_%D0%BA%D0%BE%D0%B4%D0%B0" title="Дублирование кода">дублирования</a>. Это, очевидно, может быть <i>неизбежным</i> для сложных систем при использовании языков, не поддерживающих <a class="mw-redirect" href="/wiki/%D0%9A%D0%BE%D0%BC%D0%B1%D0%B8%D0%BD%D0%B0%D1%82%D0%BE%D1%80" title="Комбинатор">комбинаторы</a> и <a class="mw-redirect" href="/wiki/%D0%9F%D0%BE%D0%BB%D0%B8%D0%BC%D0%BE%D1%80%D1%84%D0%B8%D0%B7%D0%BC_%D1%82%D0%B8%D0%BF%D0%BE%D0%B2" title="Полиморфизм типов">полиморфизм типов</a>, и это в принципе может быть <i>исключено</i> в языках, обладающих свойством <span class="iw plainlinks" data-title="Гомознаковость"><a class="new" href="/w/index.php?title=%D0%93%D0%BE%D0%BC%D0%BE%D0%B7%D0%BD%D0%B0%D0%BA%D0%BE%D0%B2%D0%BE%D1%81%D1%82%D1%8C&amp;action=edit&amp;redlink=1" title="Гомознаковость (страница отсутствует)">гомознаковости</a><sup class="noprint" style="font-style:normal; font-weight:normal;"><a class="extiw" href="https://en.wikipedia.org/wiki/Homoiconicity" title="en:Homoiconicity"><span title="Homoiconicity — версия статьи «Гомознаковость» на английском языке">[en]</span></a></sup></span> (хотя и не обязательно эффективно), так как любой шаблон может быть реализован в виде исполнимого кода<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup>.</p>
<h2><span class="mw-headline" id=".D0.A2.D0.B8.D0.BF.D1.8B_.D1.88.D0.B0.D0.B1.D0.BB.D0.BE.D0.BD.D0.BE.D0.B2_.D0.BF.D1.80.D0.BE.D0.B5.D0.BA.D1.82.D0.B8.D1.80.D0.BE.D0.B2.D0.B0.D0.BD.D0.B8.D1.8F">Типы шаблонов проектирования</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a class="mw-editsection-visualeditor" href="/w/index.php?title=%D0%A8%D0%B0%D0%B1%D0%BB%D0%BE%D0%BD_%D0%BF%D1%80%D0%BE%D0%B5%D0%BA%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F&amp;veaction=edit&amp;vesection=4" title="Редактировать раздел «Типы шаблонов проектирования»">править</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=%D0%A8%D0%B0%D0%B1%D0%BB%D0%BE%D0%BD_%D0%BF%D1%80%D0%BE%D0%B5%D0%BA%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F&amp;action=edit&amp;section=4" title="Редактировать раздел «Типы шаблонов проектирования»">править вики-текст</a><span class="mw-editsection-bracket">]</span></span></h2>
