<h1 class="firstHeading" id="firstHeading" lang="en">Factory method pattern</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><p>In <a href="/wiki/Class-based_programming" title="Class-based programming">class-based programming</a>, the <b>factory method pattern</b> is a <a href="/wiki/Creational_pattern" title="Creational pattern">creational pattern</a> that uses factory methods to deal with the problem of <a class="mw-redirect" href="/wiki/Object_creation" title="Object creation">creating objects</a> without having to specify the exact <a href="/wiki/Class_(computer_programming)" title="Class (computer programming)">class</a> of the object that will be created. This is done by creating objects by calling a factory method—either specified in an interface and implemented by child classes, or implemented in a base class and optionally overridden by derived classes—rather than by calling a <a href="/wiki/Constructor_(object-oriented_programming)" title="Constructor (object-oriented programming)">constructor</a>.</p>
<p></p>

<h2><span class="mw-headline" id="Definition">Definition</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=1" title="Edit section: Definition">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>"Define an interface for creating an object, but let subclasses decide which class to instantiate. The Factory method lets a class defer instantiation it uses to subclasses." (<a class="mw-redirect" href="/wiki/Gang_of_Four_(software)" title="Gang of Four (software)">Gang Of Four</a>)</p>
<p>Creating an object often requires complex processes not appropriate to include within a composing object. The object's creation may lead to a significant duplication of code, may require information not accessible to the composing object, may not provide a sufficient level of abstraction, or may otherwise not be part of the composing object's <a href="/wiki/Concern_(computer_science)" title="Concern (computer science)">concerns</a>. The factory method design pattern handles these problems by defining a separate <a class="mw-redirect" href="/wiki/Method_(computer_science)" title="Method (computer science)">method</a> for creating the objects, which <a class="mw-redirect" href="/wiki/Subclass_(computer_science)" title="Subclass (computer science)">subclasses</a> can then override to specify the <a class="mw-redirect" href="/wiki/Subtype" title="Subtype">derived type</a> of product that will be created.</p>
<p>The factory method pattern relies on inheritance, as object creation is delegated to subclasses that implement the factory method to create objects.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<h2><span class="mw-headline" id="Example">Example</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=2" title="Edit section: Example">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Structure">Structure</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=3" title="Edit section: Structure">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="center">
<div class="floatnone"><a class="image" href="/wiki/File:New_WikiFactoryMethod.png"><img alt="New WikiFactoryMethod.png" data-file-height="495" data-file-width="910" height="399" src="//upload.wikimedia.org/wikipedia/commons/thumb/d/df/New_WikiFactoryMethod.png/734px-New_WikiFactoryMethod.png" srcset="//upload.wikimedia.org/wikipedia/commons/d/df/New_WikiFactoryMethod.png 1.5x" width="734"/></a></div>
</div>
<p>Room is the base class for a final product (<code>MagicRoom</code> or <code>OrdinaryRoom</code>). <code>MazeGame</code> declares the abstract factory method to produce such a base product. <code>MagicRoom</code> or <code>OrdinaryRoom</code> are subclasses of the base product implementing the final product. <code>MagicMazeGame</code> and <code>OrdinaryMazeGame</code> are subclasses of <code>MazeGame</code> implementing the factory method producing the final products. Thus factory methods decouple callers (<code>MazeGame</code>) from the implementation of the concrete classes. This makes the "new" Operator redundant, allows adherence to the Open/Closed Principle and makes the final product more flexible in the event of change.</p>
<h3><span class="mw-headline" id="Example_implementations">Example implementations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=4" title="Edit section: Example implementations">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<h4><span class="mw-headline" id="Java">Java</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=5" title="Edit section: Java">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>A maze game may be played in two modes, one with regular rooms that are only connected with adjacent rooms, and one with magic rooms that allow players to be transported at random (this <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> example is similar to one in the book <i><a href="/wiki/Design_Patterns" title="Design Patterns">Design Patterns</a></i>). The MazeGame uses Rooms but it puts the responsibility of creating Rooms to its subclasses which create the concrete classes. The regular game mode could use this template method:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kd">public</span> <span class="kd">abstract</span> <span class="kd">class</span> <span class="nc">MazeGame</span> <span class="o">{</span>
    <span class="kd">private</span> <span class="kd">final</span> <span class="n">List</span><span class="o">&lt;</span><span class="n">Room</span><span class="o">&gt;</span> <span class="n">rooms</span> <span class="o">=</span> <span class="k">new</span> <span class="n">ArrayList</span><span class="o">&lt;&gt;();</span>

    <span class="kd">public</span> <span class="nf">MazeGame</span><span class="o">()</span> <span class="o">{</span>
        <span class="n">Room</span> <span class="n">room1</span> <span class="o">=</span> <span class="n">makeRoom</span><span class="o">();</span>
        <span class="n">Room</span> <span class="n">room2</span> <span class="o">=</span> <span class="n">makeRoom</span><span class="o">();</span>
        <span class="n">room1</span><span class="o">.</span><span class="na">connect</span><span class="o">(</span><span class="n">room2</span><span class="o">);</span>
        <span class="n">rooms</span><span class="o">.</span><span class="na">add</span><span class="o">(</span><span class="n">room1</span><span class="o">);</span>
        <span class="n">rooms</span><span class="o">.</span><span class="na">add</span><span class="o">(</span><span class="n">room2</span><span class="o">);</span>
    <span class="o">}</span>

    <span class="kd">abstract</span> <span class="kd">protected</span> <span class="n">Room</span> <span class="nf">makeRoom</span><span class="o">();</span>
<span class="o">}</span>
</pre></div>
<p>In the above snippet, the <code>MazeGame</code> constructor is a <a href="/wiki/Template_method_pattern" title="Template method pattern">template method</a> that makes some common logic. It refers to the <code>makeRoom</code> factory method that encapsulates the creation of rooms such that other rooms can be used in a subclass. To implement the other game mode that has magic rooms, it suffices to override the <code>makeRoom</code> method:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kd">public</span> <span class="kd">class</span> <span class="nc">MagicMazeGame</span> <span class="kd">extends</span> <span class="n">MazeGame</span> <span class="o">{</span>
    <span class="nd">@Override</span>
    <span class="kd">protected</span> <span class="n">Room</span> <span class="nf">makeRoom</span><span class="o">()</span> <span class="o">{</span>
        <span class="k">return</span> <span class="k">new</span> <span class="n">MagicRoom</span><span class="o">();</span> 
    <span class="o">}</span>
<span class="o">}</span>

<span class="kd">public</span> <span class="kd">class</span> <span class="nc">OrdinaryMazeGame</span> <span class="kd">extends</span> <span class="n">MazeGame</span> <span class="o">{</span>
    <span class="nd">@Override</span>
    <span class="kd">protected</span> <span class="n">Room</span> <span class="nf">makeRoom</span><span class="o">()</span> <span class="o">{</span>
        <span class="k">return</span> <span class="k">new</span> <span class="n">OrdinaryRoom</span><span class="o">();</span> 
    <span class="o">}</span>
<span class="o">}</span>

<span class="n">MazeGame</span> <span class="n">ordinaryGame</span> <span class="o">=</span> <span class="k">new</span> <span class="n">OrdinaryMazeGame</span><span class="o">();</span>
<span class="n">MazeGame</span> <span class="n">magicGame</span> <span class="o">=</span> <span class="k">new</span> <span class="n">MagicMazeGame</span><span class="o">();</span>
</pre></div>
<h4><span class="mw-headline" id="PHP">PHP</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=6" title="Edit section: PHP">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Another example in PHP follows, this time using interface implementations as opposed to subclassing (however the same can be achieved through subclassing). It is important to note that the factory method can also be defined as public and called directly by the client code (in contrast with the Java example above).</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="cm">/* Factory and car interfaces */</span>

<span class="k">interface</span> <span class="nx">CarFactory</span> 
<span class="p">{</span>
    <span class="k">public</span> <span class="k">function</span> <span class="nf">makeCar</span><span class="p">();</span>
<span class="p">}</span>

<span class="k">interface</span> <span class="nx">Car</span> 
<span class="p">{</span>
    <span class="k">public</span> <span class="k">function</span> <span class="nf">getType</span><span class="p">();</span>
<span class="p">}</span>

<span class="cm">/* Concrete implementations of the factory and car */</span>

<span class="k">class</span> <span class="nc">SedanFactory</span> <span class="k">implements</span> <span class="nx">CarFactory</span> 
<span class="p">{</span>
    <span class="k">public</span> <span class="k">function</span> <span class="nf">makeCar</span><span class="p">()</span> 
    <span class="p">{</span>
        <span class="k">return</span> <span class="k">new</span> <span class="nx">Sedan</span><span class="p">();</span>
    <span class="p">}</span>
<span class="p">}</span>

<span class="k">class</span> <span class="nc">Sedan</span> <span class="k">implements</span> <span class="nx">Car</span> 
<span class="p">{</span>
    <span class="k">public</span> <span class="k">function</span> <span class="nf">getType</span><span class="p">()</span> 
    <span class="p">{</span>
        <span class="k">return</span> <span class="s1">'Sedan'</span><span class="p">;</span>
    <span class="p">}</span>
<span class="p">}</span>

<span class="cm">/* Client */</span>

<span class="nv">$factory</span> <span class="o">=</span> <span class="k">new</span> <span class="nx">SedanFactory</span><span class="p">();</span>
<span class="nv">$car</span> <span class="o">=</span> <span class="nv">$factory</span><span class="o">-&gt;</span><span class="na">makeCar</span><span class="p">();</span>
<span class="k">print</span> <span class="nv">$car</span><span class="o">-&gt;</span><span class="na">getType</span><span class="p">();</span>
</pre></div>
<h4><span class="mw-headline" id="C.23">C#</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=7" title="Edit section: C#">edit</a><span class="mw-editsection-bracket">]</span></span></h4>
<p>Factory pattern deals with the instantiation of objects without exposing the instantiation logic. In other words, a Factory is actually a creator of objects which have a common interface.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">//Empty vocabulary of Actual object</span>
<span class="k">public</span> <span class="k">interface</span> <span class="n">IPeople</span>
<span class="p">{</span>
    <span class="kt">string</span> <span class="nf">GetName</span><span class="p">();</span>
<span class="p">}</span>

<span class="k">public</span> <span class="k">class</span> <span class="nc">Villagers</span> <span class="p">:</span> <span class="n">IPeople</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="kt">string</span> <span class="nf">GetName</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="k">return</span> <span class="s">"Village Person"</span><span class="p">;</span>
    <span class="p">}</span>
<span class="p">}</span>

<span class="k">public</span> <span class="k">class</span> <span class="nc">CityPeople</span> <span class="p">:</span> <span class="n">IPeople</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="kt">string</span> <span class="nf">GetName</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="k">return</span> <span class="s">"City Person"</span><span class="p">;</span>
    <span class="p">}</span>
<span class="p">}</span>

<span class="k">public</span> <span class="k">enum</span> <span class="n">PeopleType</span>
<span class="p">{</span>
    <span class="n">Rural</span><span class="p">,</span>
    <span class="n">Urban</span>
<span class="p">}</span>

<span class="c1">/// &lt;summary&gt;</span>
<span class="c1">/// Implementation of Factory - Used to create objects</span>
<span class="c1">/// &lt;/summary&gt;</span>
<span class="k">public</span> <span class="k">class</span> <span class="nc">Factory</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="n">IPeople</span> <span class="nf">GetPeople</span><span class="p">(</span><span class="n">PeopleType</span> <span class="n">type</span><span class="p">)</span>
    <span class="p">{</span>
        <span class="k">switch</span> <span class="p">(</span><span class="n">type</span><span class="p">)</span>
        <span class="p">{</span>
            <span class="k">case</span> <span class="n">PeopleType</span><span class="p">.</span><span class="n">Rural</span><span class="p">:</span>
                <span class="k">return</span> <span class="k">new</span> <span class="nf">Villagers</span><span class="p">();</span>
            <span class="k">case</span> <span class="n">PeopleType</span><span class="p">.</span><span class="n">Urban</span><span class="p">:</span>
                <span class="k">return</span> <span class="k">new</span> <span class="nf">CityPeople</span><span class="p">();</span>
            <span class="k">default</span><span class="p">:</span>
                <span class="k">throw</span> <span class="k">new</span> <span class="nf">NotSupportedException</span><span class="p">();</span>
        <span class="p">}</span>
    <span class="p">}</span>
<span class="p">}</span>
</pre></div>
<p>In the above code you can see the creation of one interface called IPeople and implemented two classes from it as Villagers and CityPeople. Based on the type passed into the factory object, we are sending back the original concrete object as the Interface IPeople.</p>
<p>A Factory method is just an addition to Factory class. It creates the object of the class through interfaces but on the other hand, it also lets the subclass decide which class is instantiated.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">public</span> <span class="k">interface</span> <span class="n">IProduct</span>
<span class="p">{</span>
    <span class="kt">string</span> <span class="nf">GetName</span><span class="p">();</span>
    <span class="kt">string</span> <span class="nf">SetPrice</span><span class="p">(</span><span class="kt">double</span> <span class="n">price</span><span class="p">);</span>
<span class="p">}</span>

<span class="k">public</span> <span class="k">class</span> <span class="nc">Phone</span> <span class="p">:</span> <span class="n">IProduct</span> 
<span class="p">{</span>
    <span class="k">private</span> <span class="kt">double</span> <span class="n">_price</span><span class="p">;</span>

    <span class="k">public</span> <span class="kt">string</span> <span class="nf">GetName</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="k">return</span> <span class="s">"Apple TouchPad"</span><span class="p">;</span>
    <span class="p">}</span>

    <span class="k">public</span> <span class="kt">string</span> <span class="nf">SetPrice</span><span class="p">(</span><span class="kt">double</span> <span class="n">price</span><span class="p">)</span>
    <span class="p">{</span>
        <span class="k">this</span><span class="p">.</span><span class="n">_price</span> <span class="p">=</span> <span class="n">price</span><span class="p">;</span>
        <span class="k">return</span> <span class="s">"success"</span><span class="p">;</span>
    <span class="p">}</span>
<span class="p">}</span>

<span class="cm">/* Almost same as Factory, just an additional exposure to do something with the created method */</span>
<span class="k">public</span> <span class="k">abstract</span> <span class="k">class</span> <span class="nc">ProductAbstractFactory</span>
<span class="p">{</span>
    <span class="k">protected</span> <span class="k">abstract</span> <span class="n">IProduct</span> <span class="nf">DoSomething</span><span class="p">();</span>

    <span class="k">public</span> <span class="n">IProduct</span> <span class="nf">GetObject</span><span class="p">()</span> <span class="c1">// Implementation of Factory Method.</span>
    <span class="p">{</span>
        <span class="k">return</span> <span class="k">this</span><span class="p">.</span><span class="n">DoSomething</span><span class="p">();</span>
    <span class="p">}</span>
<span class="p">}</span>

<span class="k">public</span> <span class="k">class</span> <span class="nc">PhoneConcreteFactory</span> <span class="p">:</span> <span class="n">ProductAbstractFactory</span>
<span class="p">{</span>
    <span class="k">protected</span> <span class="k">override</span> <span class="n">IProduct</span> <span class="nf">DoSomething</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="n">IProduct</span> <span class="n">product</span> <span class="p">=</span> <span class="k">new</span> <span class="n">Phone</span><span class="p">();</span>
        <span class="c1">//Do something with the object after you get the object. </span>
        <span class="n">product</span><span class="p">.</span><span class="n">SetPrice</span><span class="p">(</span><span class="m">20.30</span><span class="p">);</span>
        <span class="k">return</span> <span class="n">product</span><span class="p">;</span>
    <span class="p">}</span>
<span class="p">}</span>
</pre></div>
<p>You can see we have used GetObject in concreteFactory. As a result, you can easily call DoSomething() from it to get the IProduct. You might also write your custom logic after getting the object in the concrete Factory Method. The GetObject is made abstract in the Factory interface.</p>
<h2><span class="mw-headline" id="Uses">Uses</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Factory_method_pattern&amp;action=edit&amp;section=8" title="Edit section: Uses">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>In <a href="/wiki/ADO.NET" title="ADO.NET">ADO.NET</a>, <a class="external text" href="http://msdn2.microsoft.com/en-us/library/system.data.idbcommand.createparameter.aspx" rel="nofollow">IDbCommand.CreateParameter</a> is an example of the use of factory method to connect parallel class hierarchies.</li>
<li>In <a class="mw-redirect" href="/wiki/Qt_(toolkit)" title="Qt (toolkit)">Qt</a>, <a class="external text" href="http://qt-project.org/doc/qt-5.0/qtwidgets/qmainwindow.html#createPopupMenu" rel="nofollow">QMainWindow::createPopupMenu</a> is a factory method declared in a framework that can be overridden in <a class="mw-redirect" href="/wiki/Application_code" title="Application code">application code</a>.</li>
<li>In <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, several factories are used in the <a class="external text" href="http://download.oracle.com/javase/1.5.0/docs/api/javax/xml/parsers/package-summary.html" rel="nofollow">javax.xml.parsers</a> package. e.g. javax.xml.parsers.DocumentBuilderFactory or javax.xml.parsers.SAXParserFactory.</li>
</ul>
