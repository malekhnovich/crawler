<h1 class="firstHeading" id="firstHeading" lang="en">Binding properties pattern</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><p>The <b>Binding properties pattern</b> is combining multiple <a href="/wiki/Observer_pattern" title="Observer pattern">observers</a> to force properties in different objects to be synchronized or coordinated in some way. This <a href="/wiki/Software_design_pattern" title="Software design pattern">pattern</a> was first described as a technique by Victor Porton.<sup class="reference" id="cite_ref-:c3_bp_1-0"><a href="#cite_note-:c3_bp-1">[1]</a></sup><sup class="reference" id="cite_ref-:ec_btpoo_2-0"><a href="#cite_note-:ec_btpoo-2">[2]</a></sup> This pattern comes under <a class="mw-redirect" href="/wiki/Concurrency_patterns" title="Concurrency patterns">concurrency patterns</a>.</p>
<p></p>

<h2><span class="mw-headline" id="Comparison_with_aspect-oriented_implementation">Comparison with aspect-oriented implementation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Binding_properties_pattern&amp;action=edit&amp;section=1" title="Edit section: Comparison with aspect-oriented implementation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>As an alternative to the aspect-oriented implementation of mutual properties, property binding can be proposed. In LibPropC++ C++ library it is implemented too.<sup class="reference" id="cite_ref-:accm_aautbom_3-0"><a href="#cite_note-:accm_aautbom-3">[3]</a></sup></p>
<p>Some weakness in the LibPropC++ (with property binding):</p>
<ol>
<li>Its use is not transparent as it requires to be declared necessary object attributes as properties and appropriate accessor methods must be provided</li>
<li>Binding of attributes in LibPropC++ is not designed to replace method calls</li>
<li>The library does not maintain an interaction history.<sup class="reference" id="cite_ref-:accm_aautbom_3-1"><a href="#cite_note-:accm_aautbom-3">[3]</a></sup></li>
</ol>
<h2><span class="mw-headline" id="Implementation">Implementation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Binding_properties_pattern&amp;action=edit&amp;section=2" title="Edit section: Implementation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>There are two types of binding. One-way binding should be applied when one of the properties is read-only. In other cases, two-way binding must be applied.<sup class="reference" id="cite_ref-:ec_btpoo_2-1"><a href="#cite_note-:ec_btpoo-2">[2]</a></sup><sup class="reference" id="cite_ref-:accm_aautbom_3-2"><a href="#cite_note-:accm_aautbom-3">[3]</a></sup></p>
<p>Infinite loops can be eliminated by blocking the signal, or comparing the assigned value with the property value before assignment, or eliminating unnecessary assignments.<sup class="reference" id="cite_ref-:ec_btpoo_2-2"><a href="#cite_note-:ec_btpoo-2">[2]</a></sup><sup class="reference" id="cite_ref-:accm_aautbom_3-3"><a href="#cite_note-:accm_aautbom-3">[3]</a></sup></p>
<p>Binding properties of different types can be achieved through <a class="mw-redirect" href="/wiki/Type_Conversion" title="Type Conversion">Type Conversions</a>.<sup class="reference" id="cite_ref-:ec_btpoo_2-3"><a href="#cite_note-:ec_btpoo-2">[2]</a></sup><sup class="reference" id="cite_ref-:accm_aautbom_3-4"><a href="#cite_note-:accm_aautbom-3">[3]</a></sup></p>
<p>Binding properties with transformations can be achieved through reducing the transformation function to the problem of binding properties, and the function can be imaginary consider as Type Conversions.<sup class="reference" id="cite_ref-:ec_btpoo_2-4"><a href="#cite_note-:ec_btpoo-2">[2]</a></sup><sup class="reference" id="cite_ref-:accm_aautbom_3-5"><a href="#cite_note-:accm_aautbom-3">[3]</a></sup></p>
<div class="center">
<div class="floatnone"><a class="image" href="/wiki/File:Binding_Properties_Pattern.png" title="Binding properties pattern"><img alt="Binding properties pattern" data-file-height="230" data-file-width="389" height="230" src="//upload.wikimedia.org/wikipedia/en/4/4e/Binding_Properties_Pattern.png" width="389"/></a></div>
</div>
<p><sup class="reference" id="cite_ref-:ec_btpoo_2-5"><a href="#cite_note-:ec_btpoo-2">[2]</a></sup></p>
<h2><span class="mw-headline" id="Resulting_Cotext">Resulting Cotext</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Binding_properties_pattern&amp;action=edit&amp;section=3" title="Edit section: Resulting Cotext">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Properties are being kept synchronized automatically. Between library calls they always have the values expressed by the EqualityConstraints.<sup class="reference" id="cite_ref-:accm_aautbom_3-6"><a href="#cite_note-:accm_aautbom-3">[3]</a></sup></p>
<h2><span class="mw-headline" id="Deficiencies">Deficiencies</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Binding_properties_pattern&amp;action=edit&amp;section=4" title="Edit section: Deficiencies">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Property changes watching machanism acquires some resources.<sup class="reference" id="cite_ref-:accm_aautbom_3-7"><a href="#cite_note-:accm_aautbom-3">[3]</a></sup></p>
<h2><span class="mw-headline" id="Sample_Code">Sample Code</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Binding_properties_pattern&amp;action=edit&amp;section=5" title="Edit section: Sample Code">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Code sketch for one-way binding may look like as follows:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">bind_multiple_one_way</span><span class="p">(</span><span class="n">src_obj</span><span class="p">,</span> <span class="n">src_prop</span><span class="p">,</span> <span class="n">dst_objs</span><span class="p">[],</span> <span class="n">dst_props</span><span class="p">[])</span>
<span class="p">{</span>
  <span class="k">for</span> <span class="p">(</span><span class="n">i</span><span class="p">,</span> <span class="n">j</span><span class="p">)</span> <span class="n">in</span> <span class="p">(</span><span class="n">dst_objs</span><span class="p">,</span> <span class="n">dst_props</span><span class="p">)</span>
  <span class="p">{</span>
    <span class="n">bind_properties_one_way</span><span class="p">(</span><span class="n">src_obj</span><span class="p">,</span> <span class="n">src_prop</span><span class="p">,</span> <span class="n">i</span><span class="p">,</span> <span class="n">j</span><span class="p">);</span>
  <span class="p">}</span>
<span class="p">}</span>
</pre></div>
<p><sup class="reference" id="cite_ref-:ec_btpoo_2-6"><a href="#cite_note-:ec_btpoo-2">[2]</a></sup></p>
<p>Two-way binding can be expressed as follows (in C++):</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">// In this pseudo-code are not taken into the account initial values assignments</span>
<span class="n">bind_two_way</span><span class="p">(</span><span class="n">prop1</span><span class="p">,</span> <span class="n">prop2</span><span class="p">)</span>
<span class="p">{</span>
  <span class="n">bind</span><span class="p">(</span><span class="n">prop1</span><span class="p">,</span> <span class="n">prop2</span><span class="p">);</span>
  <span class="n">bind</span><span class="p">(</span><span class="n">prop2</span><span class="p">,</span> <span class="n">prop1</span><span class="p">);</span>
<span class="p">}</span>
</pre></div>
<p><sup class="reference" id="cite_ref-:accm_aautbom_3-8"><a href="#cite_note-:accm_aautbom-3">[3]</a></sup></p>
<p>Accomplishing the binding (i.e. connecting the property change notification in an event handler) may be like as follows:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">on_property_change</span><span class="p">(</span><span class="n">src_prop</span><span class="p">,</span> <span class="n">dst_prop</span><span class="p">)</span>
<span class="p">{</span>
  <span class="n">block_signal</span><span class="p">(</span><span class="n">src_obj</span><span class="p">,</span> <span class="n">on_property_change</span><span class="p">);</span>
  <span class="nl">dst_prop</span> <span class="p">:</span><span class="o">=</span> <span class="n">src_prop</span><span class="p">;</span>
  <span class="n">unblock_signal</span><span class="p">(</span><span class="n">src_obj</span><span class="p">,</span> <span class="n">on_property_change</span><span class="p">);</span>
<span class="p">}</span>
</pre></div>
<p><sup class="reference" id="cite_ref-:accm_aautbom_3-9"><a href="#cite_note-:accm_aautbom-3">[3]</a></sup></p>
