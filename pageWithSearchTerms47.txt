<p>In <a href="/wiki/Software_engineering" title="Software engineering">software engineering</a>, the <b>adapter pattern</b> is a <a href="/wiki/Software_design_pattern" title="Software design pattern">software design pattern</a> that allows the <a class="mw-redirect" href="/wiki/Interface_(computer_science)" title="Interface (computer science)">interface</a> of an existing <a class="mw-redirect" href="/wiki/Class_(computer_science)" title="Class (computer science)">class</a> to be used as another interface.<sup class="reference" id="cite_ref-HeadFirst_1-0"><a href="#cite_note-HeadFirst-1">[1]</a></sup> It is often used to make existing classes work with others without modifying their <a href="/wiki/Source_code" title="Source code">source code</a>.</p>
<p></p>

<h2><span class="mw-headline" id="Definition">Definition</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adapter_pattern&amp;action=edit&amp;section=1" title="Edit section: Definition">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>An adapter helps two incompatible interfaces to work together. This is the real world definition for an adapter. Interfaces may be incompatible but the inner functionality should suit the need. The Adapter design pattern allows otherwise incompatible classes to work together by converting the interface of one class into an interface expected by the clients.</p>
<h2><span class="mw-headline" id="Structure">Structure</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adapter_pattern&amp;action=edit&amp;section=2" title="Edit section: Structure">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>There are two types of adapter patterns:<sup class="reference" id="cite_ref-HeadFirst_1-1"><a href="#cite_note-HeadFirst-1">[1]</a></sup></p>
<h3><span class="mw-headline" id="Object_Adapter_pattern">Object Adapter pattern</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adapter_pattern&amp;action=edit&amp;section=3" title="Edit section: Object Adapter pattern">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In this type of adapter pattern, the adapter contains an instance of the class it wraps. In this situation, the adapter makes calls to the instance of the wrapped <a class="mw-redirect" href="/wiki/Object_(computing)" title="Object (computing)">object</a>.</p>

<h3><span class="mw-headline" id="Class_Adapter_pattern">Class Adapter pattern</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adapter_pattern&amp;action=edit&amp;section=4" title="Edit section: Class Adapter pattern">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>This type of adapter uses multiple <a href="/wiki/Subtyping" title="Subtyping">polymorphic interfaces</a> implementing or inheriting both the interface that is expected and the interface that is pre-existing. It is typical for the expected interface to be created as a pure <a href="/wiki/Interface_(Java)" title="Interface (Java)">interface</a> class, especially in <a href="/wiki/Programming_language" title="Programming language">languages</a> such as <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> (before jdk 1.8) that do not support <a href="/wiki/Multiple_inheritance" title="Multiple inheritance">multiple inheritance</a> of classes.<sup class="reference" id="cite_ref-HeadFirst_1-2"><a href="#cite_note-HeadFirst-1">[1]</a></sup></p>

<p>The adapter pattern is useful in situations where an already existing class provides some or all of the <a href="/wiki/Service_(systems_architecture)" title="Service (systems architecture)">services</a> needed but does not use the <a class="mw-redirect" href="/wiki/Interface_(computer_science)" title="Interface (computer science)">interface</a> needed. A good real life example is an adapter that converts the interface of a <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a> of an <a href="/wiki/XML" title="XML">XML</a> document into a tree structure that can be displayed.</p>
<h3><span class="mw-headline" id="A_further_form_of_runtime_Adapter_pattern">A further form of runtime Adapter pattern</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adapter_pattern&amp;action=edit&amp;section=5" title="Edit section: A further form of runtime Adapter pattern">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>There is a further form of runtime adapter pattern as follows:</p>
<p>It is desired for <code class="mw-highlight" dir="ltr"><span class="n">classA</span></code> to supply <code class="mw-highlight" dir="ltr"><span class="n">classB</span></code> with some data, let us suppose some <code class="mw-highlight" dir="ltr"><span class="n">String</span></code> data. A compile time solution is:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">classB</span><span class="o">.</span><span class="na">setStringData</span><span class="o">(</span><span class="n">classA</span><span class="o">.</span><span class="na">getStringData</span><span class="o">());</span>
</pre></div>
<p>However, suppose that the format of the string data must be varied. A compile time solution is to use inheritance:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kd">public</span> <span class="kd">class</span> <span class="nc">Format1ClassA</span> <span class="kd">extends</span> <span class="n">ClassA</span> <span class="o">{</span>
   <span class="nd">@Override</span>
   <span class="kd">public</span> <span class="n">String</span> <span class="nf">getStringData</span><span class="o">()</span> <span class="o">{</span>
      <span class="k">return</span> <span class="n">format</span><span class="o">(</span><span class="n">toString</span><span class="o">());</span>
   <span class="o">}</span>
<span class="o">}</span>
</pre></div>
<p>and perhaps create the correctly "formatting" object at runtime by means of the <a class="mw-redirect" href="/wiki/Factory_pattern" title="Factory pattern">Factory pattern</a>.</p>
<p>A solution using "adapters" proceeds as follows:</p>
<p>(i) define an intermediary "Provider" interface, and write an implementation of that Provider interface that wraps the source of the data, <code class="mw-highlight" dir="ltr"><span class="n">ClassA</span></code> in this example, and outputs the data formatted as appropriate:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kd">public</span> <span class="kd">interface</span> <span class="nc">StringProvider</span> <span class="o">{</span>
    <span class="kd">public</span> <span class="n">String</span> <span class="nf">getStringData</span><span class="o">();</span>
<span class="o">}</span>

<span class="kd">public</span> <span class="kd">class</span> <span class="nc">ClassAFormat1</span> <span class="kd">implements</span> <span class="n">StringProvider</span> <span class="o">{</span>
    <span class="kd">private</span> <span class="n">ClassA</span> <span class="n">classA</span> <span class="o">=</span> <span class="kc">null</span><span class="o">;</span>

    <span class="kd">public</span> <span class="nf">ClassAFormat1</span><span class="o">(</span><span class="kd">final</span> <span class="n">ClassA</span> <span class="n">A</span><span class="o">)</span> <span class="o">{</span>
        <span class="n">classA</span> <span class="o">=</span> <span class="n">A</span><span class="o">;</span>
    <span class="o">}</span>

    <span class="kd">public</span> <span class="n">String</span> <span class="nf">getStringData</span><span class="o">()</span> <span class="o">{</span>
        <span class="k">return</span> <span class="n">format</span><span class="o">(</span><span class="n">classA</span><span class="o">.</span><span class="na">getStringData</span><span class="o">());</span>
    <span class="o">}</span>
    
    <span class="kd">private</span> <span class="n">String</span> <span class="nf">format</span><span class="o">(</span><span class="n">String</span> <span class="n">sourceValue</span><span class="o">)</span> <span class="o">{</span>
        <span class="c1">//manipulate the source string into</span>
        <span class="c1">//a format required by the object needing the source object's</span>
        <span class="c1">//data</span>
        <span class="k">return</span> <span class="n">sourceValue</span><span class="o">.</span><span class="na">trim</span><span class="o">();</span>
    <span class="o">}</span>
<span class="o">}</span>
</pre></div>
<p>(ii) Write an Adapter class that returns the specific implementation of the Provider:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kd">public</span> <span class="kd">class</span> <span class="nc">ClassAFormat1Adapter</span> <span class="kd">extends</span> <span class="n">Adapter</span> <span class="o">{</span>
   <span class="kd">public</span> <span class="n">Object</span> <span class="nf">adapt</span><span class="o">(</span><span class="kd">final</span> <span class="n">Object</span> <span class="n">OBJECT</span><span class="o">)</span> <span class="o">{</span>
      <span class="k">return</span> <span class="k">new</span> <span class="n">ClassAFormat1</span><span class="o">((</span><span class="n">ClassA</span><span class="o">)</span> <span class="n">OBJECT</span><span class="o">);</span>
   <span class="o">}</span>
<span class="o">}</span>
</pre></div>
<p>(iii) Register the <code class="mw-highlight" dir="ltr"><span class="n">Adapter</span></code> with a global registry, so that the <code class="mw-highlight" dir="ltr"><span class="n">Adapter</span></code> can be looked up at runtime:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">AdapterFactory</span><span class="o">.</span><span class="na">getInstance</span><span class="o">().</span><span class="na">registerAdapter</span><span class="o">(</span><span class="n">ClassA</span><span class="o">.</span><span class="na">class</span><span class="o">,</span> <span class="n">ClassAFormat1Adapter</span><span class="o">.</span><span class="na">class</span><span class="o">,</span> <span class="s">"format1"</span><span class="o">);</span>
</pre></div>
<p>(iv) In code, when wishing to transfer data from <code class="mw-highlight" dir="ltr"><span class="n">ClassA</span></code> to <code class="mw-highlight" dir="ltr"><span class="n">ClassB</span></code>, write:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">Adapter</span> <span class="n">adapter</span> <span class="o">=</span>
    <span class="n">AdapterFactory</span><span class="o">.</span><span class="na">getInstance</span><span class="o">()</span>
        <span class="o">.</span><span class="na">getAdapterFromTo</span><span class="o">(</span><span class="n">ClassA</span><span class="o">.</span><span class="na">class</span><span class="o">,</span> <span class="n">StringProvider</span><span class="o">.</span><span class="na">class</span><span class="o">,</span> <span class="s">"format1"</span><span class="o">);</span>
<span class="n">StringProvider</span> <span class="n">provider</span> <span class="o">=</span> <span class="o">(</span><span class="n">StringProvider</span><span class="o">)</span> <span class="n">adapter</span><span class="o">.</span><span class="na">adapt</span><span class="o">(</span><span class="n">classA</span><span class="o">);</span>
<span class="n">String</span> <span class="n">string</span> <span class="o">=</span> <span class="n">provider</span><span class="o">.</span><span class="na">getStringData</span><span class="o">();</span>
<span class="n">classB</span><span class="o">.</span><span class="na">setStringData</span><span class="o">(</span><span class="n">string</span><span class="o">);</span>
</pre></div>
<p>or more concisely:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">classB</span><span class="o">.</span><span class="na">setStringData</span><span class="o">(</span>
    <span class="o">((</span><span class="n">StringProvider</span><span class="o">)</span>
            <span class="n">AdapterFactory</span><span class="o">.</span><span class="na">getInstance</span><span class="o">()</span>
                <span class="o">.</span><span class="na">getAdapterFromTo</span><span class="o">(</span><span class="n">ClassA</span><span class="o">.</span><span class="na">class</span><span class="o">,</span> <span class="n">StringProvider</span><span class="o">.</span><span class="na">class</span><span class="o">,</span> <span class="s">"format1"</span><span class="o">)</span>
                <span class="o">.</span><span class="na">adapt</span><span class="o">(</span><span class="n">classA</span><span class="o">))</span>
        <span class="o">.</span><span class="na">getStringData</span><span class="o">());</span>
</pre></div>
<p>(v) The advantage can be seen in that, if it is desired to transfer the data in a second format, then look up the different adapter/provider:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">Adapter</span> <span class="n">adapter</span> <span class="o">=</span>
    <span class="n">AdapterFactory</span><span class="o">.</span><span class="na">getInstance</span><span class="o">()</span>
        <span class="o">.</span><span class="na">getAdapterFromTo</span><span class="o">(</span><span class="n">ClassA</span><span class="o">.</span><span class="na">class</span><span class="o">,</span> <span class="n">StringProvider</span><span class="o">.</span><span class="na">class</span><span class="o">,</span> <span class="s">"format2"</span><span class="o">);</span>
</pre></div>
<p>(vi) And if it is desired to output the data from <code class="mw-highlight" dir="ltr"><span class="n">ClassA</span></code> as, say, image data in <code class="mw-highlight" dir="ltr"><span class="n">Class</span> <span class="n">C</span></code>:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">Adapter</span> <span class="n">adapter</span> <span class="o">=</span>
    <span class="n">AdapterFactory</span><span class="o">.</span><span class="na">getInstance</span><span class="o">()</span>
        <span class="o">.</span><span class="na">getAdapterFromTo</span><span class="o">(</span><span class="n">ClassA</span><span class="o">.</span><span class="na">class</span><span class="o">,</span> <span class="n">ImageProvider</span><span class="o">.</span><span class="na">class</span><span class="o">,</span> <span class="s">"format2"</span><span class="o">);</span>
<span class="n">ImageProvider</span> <span class="n">provider</span> <span class="o">=</span> <span class="o">(</span><span class="n">ImageProvider</span><span class="o">)</span> <span class="n">adapter</span><span class="o">.</span><span class="na">adapt</span><span class="o">(</span><span class="n">classA</span><span class="o">);</span>
<span class="n">classC</span><span class="o">.</span><span class="na">setImage</span><span class="o">(</span><span class="n">provider</span><span class="o">.</span><span class="na">getImage</span><span class="o">());</span>
</pre></div>
<p>(vii) In this way, the use of adapters and providers allows multiple "views" by <code class="mw-highlight" dir="ltr"><span class="n">ClassB</span></code> and <code class="mw-highlight" dir="ltr"><span class="n">ClassC</span></code> into <code class="mw-highlight" dir="ltr"><span class="n">ClassA</span></code> without having to alter the class hierarchy. In general, it permits a mechanism for arbitrary data flows between objects that can be retrofitted to an existing object hierarchy.</p>
<p><br/></p>
<h3><span class="mw-headline" id="Implementation_of_Adapter_pattern">Implementation of Adapter pattern</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Adapter_pattern&amp;action=edit&amp;section=6" title="Edit section: Implementation of Adapter pattern">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>When implementing the adapter pattern, for clarity one can apply the class name <code class="mw-highlight" dir="ltr"><span class="o">[</span><span class="n">ClassName</span><span class="o">]</span><span class="n">To</span><span class="o">[</span><span class="n">Interface</span><span class="o">]</span><span class="n">Adapter</span></code> to the provider implementation, for example <code class="mw-highlight" dir="ltr"><span class="n">DAOToProviderAdapter</span></code>. It should have a constructor method with an adaptee class variable as a parameter. This parameter will be passed to an instance member of <code class="mw-highlight" dir="ltr"><span class="o">[</span><span class="n">ClassName</span><span class="o">]</span><span class="n">To</span><span class="o">[</span><span class="n">Interface</span><span class="o">]</span><span class="n">Adapter</span></code>. When the clientMethod is called it will have access to the adaptee instance which allows for accessing the required data of the adaptee and performing operations on that data that generates the desired output.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kd">public</span> <span class="kd">class</span> <span class="nc">AdapteeToClientAdapter</span> <span class="kd">implements</span> <span class="n">Adapter</span> <span class="o">{</span>

    <span class="kd">private</span> <span class="kd">final</span> <span class="n">Adaptee</span> <span class="n">instance</span><span class="o">;</span>

    <span class="kd">public</span> <span class="nf">AdapteeToClientAdapter</span><span class="o">(</span><span class="kd">final</span> <span class="n">Adaptee</span> <span class="n">instance</span><span class="o">)</span> <span class="o">{</span>
         <span class="k">this</span><span class="o">.</span><span class="na">instance</span> <span class="o">=</span> <span class="n">instance</span><span class="o">;</span>
    <span class="o">}</span>

    <span class="nd">@Override</span>
    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">clientMethod</span><span class="o">()</span> <span class="o">{</span>
       <span class="c1">// call Adaptee's method(s) to implement Client's clientMethod</span>
    <span class="o">}</span>

<span class="o">}</span>
</pre></div>
<p>And a Scala implementation</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">implicit</span> <span class="k">def</span> <span class="n">adaptee2Adapter</span><span class="o">(</span><span class="n">adaptee</span><span class="k">:</span> <span class="kt">Adaptee</span><span class="o">)</span><span class="k">:</span> <span class="kt">Adapter</span> <span class="o">=</span> <span class="o">{</span>
  <span class="k">new</span> <span class="nc">Adapter</span> <span class="o">{</span>
    <span class="k">override</span> <span class="k">def</span> <span class="n">clientMethod</span><span class="k">:</span> <span class="kt">Unit</span> <span class="o">=</span> <span class="o">{</span> 
    <span class="c1">// call Adaptee's method(s) to implement Client's clientMethod */ </span>
    <span class="o">}</span>
  <span class="o">}</span>
<span class="o">}</span>
</pre></div>
