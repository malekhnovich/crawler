<h1 class="firstHeading" id="firstHeading" lang="en">Bridge pattern</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><p>The <b>bridge pattern</b> is a <a class="mw-redirect" href="/wiki/Design_pattern_(computer_science)" title="Design pattern (computer science)">design pattern</a> used in <a href="/wiki/Software_engineering" title="Software engineering">software engineering</a> which is meant to <i>"decouple an <a class="mw-redirect" href="/wiki/Abstraction_(computer_science)" title="Abstraction (computer science)">abstraction</a> from its <a href="/wiki/Implementation" title="Implementation">implementation</a> so that the two can vary independently"</i>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> The <i>bridge</i> uses <a class="mw-redirect" href="/wiki/Encapsulation_(computer_science)" title="Encapsulation (computer science)">encapsulation</a>, <a href="/wiki/Object_composition#Aggregation" title="Object composition">aggregation</a>, and can use <a href="/wiki/Inheritance_(object-oriented_programming)" title="Inheritance (object-oriented programming)">inheritance</a> to separate responsibilities into different <a class="mw-redirect" href="/wiki/Class_(computer_science)" title="Class (computer science)">classes</a>.</p>
<p>When a class varies often, the features of <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented programming</a> become very useful because changes to a <a href="/wiki/Computer_program" title="Computer program">program</a>'s <a href="/wiki/Source_code" title="Source code">code</a> can be made easily with minimal prior knowledge about the program. The bridge pattern is useful when both the class and what it does vary often. The class itself can be thought of as the <i>abstraction</i> and what the class can do as the <i>implementation</i>. The bridge pattern can also be thought of as two layers of abstraction.</p>
<p>When there is only one fixed implementation, this pattern is known as the <a class="mw-redirect" href="/wiki/Pimpl" title="Pimpl">Pimpl</a> idiom in the <a href="/wiki/C%2B%2B" title="C++">C++</a> world.</p>
<p>The <b>bridge pattern</b> is often confused with the adapter pattern. In fact, the <b>bridge pattern</b> is often implemented using the <a href="/wiki/Adapter_pattern#Class_Adapter_pattern" title="Adapter pattern">class <b>adapter pattern</b></a>, e.g. in the Java code below.</p>
<p><b>Variant</b>: The implementation can be decoupled even more by deferring the presence of the implementation to the point where the abstraction is utilized.</p>

<h2><span class="mw-headline" id="Example">Example</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Bridge_pattern&amp;action=edit&amp;section=2" title="Edit section: Example">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="C.23">C#</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Bridge_pattern&amp;action=edit&amp;section=3" title="Edit section: C#">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Bridge pattern compose objects in tree structure. It decouples abstraction from implementation. Here abstraction represents the client from which the objects will be called. An example to implement in C# is given below</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">// Helps in providing truly decoupled architecture</span>
<span class="k">public</span> <span class="k">interface</span> <span class="n">IBridge</span>
<span class="p">{</span>
    <span class="k">void</span> <span class="nf">Function1</span><span class="p">();</span>
    <span class="k">void</span> <span class="nf">Function2</span><span class="p">();</span>
<span class="p">}</span>

<span class="k">public</span> <span class="k">class</span> <span class="nc">Bridge1</span> <span class="p">:</span> <span class="n">IBridge</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="k">void</span> <span class="nf">Function1</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="k">throw</span> <span class="k">new</span> <span class="nf">NotImplementedException</span><span class="p">();</span>
    <span class="p">}</span>

    <span class="k">public</span> <span class="k">void</span> <span class="nf">Function2</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="k">throw</span> <span class="k">new</span> <span class="nf">NotImplementedException</span><span class="p">();</span>
    <span class="p">}</span>
<span class="p">}</span>

<span class="k">public</span> <span class="k">class</span> <span class="nc">Bridge2</span> <span class="p">:</span> <span class="n">IBridge</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="k">void</span> <span class="nf">Function1</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="k">throw</span> <span class="k">new</span> <span class="nf">NotImplementedException</span><span class="p">();</span>
    <span class="p">}</span>

    <span class="k">public</span> <span class="k">void</span> <span class="nf">Function2</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="k">throw</span> <span class="k">new</span> <span class="nf">NotImplementedException</span><span class="p">();</span>
    <span class="p">}</span>
<span class="p">}</span>

<span class="k">public</span> <span class="k">interface</span> <span class="n">IAbstractBridge</span>
<span class="p">{</span>
    <span class="k">void</span> <span class="nf">CallMethod1</span><span class="p">();</span>
    <span class="k">void</span> <span class="nf">CallMethod2</span><span class="p">();</span>
<span class="p">}</span>

<span class="k">public</span> <span class="k">class</span> <span class="nc">AbstractBridge</span> <span class="p">:</span> <span class="n">IAbstractBridge</span> 
<span class="p">{</span>
    <span class="k">public</span> <span class="n">IBridge</span> <span class="n">bridge</span><span class="p">;</span>

    <span class="k">public</span> <span class="nf">AbstractBridge</span><span class="p">(</span><span class="n">IBridge</span> <span class="n">bridge</span><span class="p">)</span>
    <span class="p">{</span>
        <span class="k">this</span><span class="p">.</span><span class="n">bridge</span> <span class="p">=</span> <span class="n">bridge</span><span class="p">;</span>
    <span class="p">}</span>

    <span class="k">public</span> <span class="k">void</span> <span class="nf">CallMethod1</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="k">this</span><span class="p">.</span><span class="n">bridge</span><span class="p">.</span><span class="n">Function1</span><span class="p">();</span>
    <span class="p">}</span>

    <span class="k">public</span> <span class="k">void</span> <span class="nf">CallMethod2</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="k">this</span><span class="p">.</span><span class="n">bridge</span><span class="p">.</span><span class="n">Function2</span><span class="p">();</span>
    <span class="p">}</span>
<span class="p">}</span>
</pre></div>
<p>As you can see the Bridge classes are the Implementation, which uses the same interface oriented architecture to create objects. On the other hand, the abstraction takes an object of the implementation phase and runs its method. Thus makes it completely decoupled with one another.</p>
<h3><span class="mw-headline" id="Java">Java</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Bridge_pattern&amp;action=edit&amp;section=4" title="Edit section: Java">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The following <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> (SE 6) program illustrates a 'shape'.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="cm">/** "Implementor" */</span>
<span class="kd">interface</span> <span class="nc">DrawingAPI</span> <span class="o">{</span>
    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">drawCircle</span><span class="o">(</span><span class="kt">double</span> <span class="n">x</span><span class="o">,</span> <span class="kt">double</span> <span class="n">y</span><span class="o">,</span> <span class="kt">double</span> <span class="n">radius</span><span class="o">);</span>
<span class="o">}</span>

<span class="cm">/** "ConcreteImplementor"  1/2 */</span>
<span class="kd">class</span> <span class="nc">DrawingAPI1</span> <span class="kd">implements</span> <span class="n">DrawingAPI</span> <span class="o">{</span>
    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">drawCircle</span><span class="o">(</span><span class="kt">double</span> <span class="n">x</span><span class="o">,</span> <span class="kt">double</span> <span class="n">y</span><span class="o">,</span> <span class="kt">double</span> <span class="n">radius</span><span class="o">)</span> <span class="o">{</span>
        <span class="n">System</span><span class="o">.</span><span class="na">out</span><span class="o">.</span><span class="na">printf</span><span class="o">(</span><span class="s">"API1.circle at %f:%f radius %f\n"</span><span class="o">,</span> <span class="n">x</span><span class="o">,</span> <span class="n">y</span><span class="o">,</span> <span class="n">radius</span><span class="o">);</span>
    <span class="o">}</span>
<span class="o">}</span>

<span class="cm">/** "ConcreteImplementor" 2/2 */</span>
<span class="kd">class</span> <span class="nc">DrawingAPI2</span> <span class="kd">implements</span> <span class="n">DrawingAPI</span> <span class="o">{</span>
    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">drawCircle</span><span class="o">(</span><span class="kt">double</span> <span class="n">x</span><span class="o">,</span> <span class="kt">double</span> <span class="n">y</span><span class="o">,</span> <span class="kt">double</span> <span class="n">radius</span><span class="o">)</span> <span class="o">{</span>
        <span class="n">System</span><span class="o">.</span><span class="na">out</span><span class="o">.</span><span class="na">printf</span><span class="o">(</span><span class="s">"API2.circle at %f:%f radius %f\n"</span><span class="o">,</span> <span class="n">x</span><span class="o">,</span> <span class="n">y</span><span class="o">,</span> <span class="n">radius</span><span class="o">);</span>
    <span class="o">}</span>
<span class="o">}</span>

<span class="cm">/** "Abstraction" */</span>
<span class="kd">abstract</span> <span class="kd">class</span> <span class="nc">Shape</span> <span class="o">{</span>
    <span class="kd">protected</span> <span class="n">DrawingAPI</span> <span class="n">drawingAPI</span><span class="o">;</span>

    <span class="kd">protected</span> <span class="nf">Shape</span><span class="o">(</span><span class="n">DrawingAPI</span> <span class="n">drawingAPI</span><span class="o">){</span>
        <span class="k">this</span><span class="o">.</span><span class="na">drawingAPI</span> <span class="o">=</span> <span class="n">drawingAPI</span><span class="o">;</span>
    <span class="o">}</span>

    <span class="kd">public</span> <span class="kd">abstract</span> <span class="kt">void</span> <span class="nf">draw</span><span class="o">();</span>                             <span class="c1">// low-level</span>
    <span class="kd">public</span> <span class="kd">abstract</span> <span class="kt">void</span> <span class="nf">resizeByPercentage</span><span class="o">(</span><span class="kt">double</span> <span class="n">pct</span><span class="o">);</span>     <span class="c1">// high-level</span>
<span class="o">}</span>

<span class="cm">/** "Refined Abstraction" */</span>
<span class="kd">class</span> <span class="nc">CircleShape</span> <span class="kd">extends</span> <span class="n">Shape</span> <span class="o">{</span>
    <span class="kd">private</span> <span class="kt">double</span> <span class="n">x</span><span class="o">,</span> <span class="n">y</span><span class="o">,</span> <span class="n">radius</span><span class="o">;</span>
    <span class="kd">public</span> <span class="nf">CircleShape</span><span class="o">(</span><span class="kt">double</span> <span class="n">x</span><span class="o">,</span> <span class="kt">double</span> <span class="n">y</span><span class="o">,</span> <span class="kt">double</span> <span class="n">radius</span><span class="o">,</span> <span class="n">DrawingAPI</span> <span class="n">drawingAPI</span><span class="o">)</span> <span class="o">{</span>
        <span class="kd">super</span><span class="o">(</span><span class="n">drawingAPI</span><span class="o">);</span>
        <span class="k">this</span><span class="o">.</span><span class="na">x</span> <span class="o">=</span> <span class="n">x</span><span class="o">;</span>  <span class="k">this</span><span class="o">.</span><span class="na">y</span> <span class="o">=</span> <span class="n">y</span><span class="o">;</span>  <span class="k">this</span><span class="o">.</span><span class="na">radius</span> <span class="o">=</span> <span class="n">radius</span><span class="o">;</span>
    <span class="o">}</span>

    <span class="c1">// low-level i.e. Implementation specific</span>
    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">draw</span><span class="o">()</span> <span class="o">{</span>
        <span class="n">drawingAPI</span><span class="o">.</span><span class="na">drawCircle</span><span class="o">(</span><span class="n">x</span><span class="o">,</span> <span class="n">y</span><span class="o">,</span> <span class="n">radius</span><span class="o">);</span>
    <span class="o">}</span>
    <span class="c1">// high-level i.e. Abstraction specific</span>
    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">resizeByPercentage</span><span class="o">(</span><span class="kt">double</span> <span class="n">pct</span><span class="o">)</span> <span class="o">{</span>
        <span class="n">radius</span> <span class="o">*=</span> <span class="o">(</span><span class="mf">1.0</span> <span class="o">+</span> <span class="n">pct</span><span class="o">/</span><span class="mf">100.0</span><span class="o">);</span>
    <span class="o">}</span>
<span class="o">}</span>

<span class="cm">/** "Client" */</span>
<span class="kd">class</span> <span class="nc">BridgePattern</span> <span class="o">{</span>
    <span class="kd">public</span> <span class="kd">static</span> <span class="kt">void</span> <span class="nf">main</span><span class="o">(</span><span class="n">String</span><span class="o">[]</span> <span class="n">args</span><span class="o">)</span> <span class="o">{</span>
        <span class="n">Shape</span><span class="o">[]</span> <span class="n">shapes</span> <span class="o">=</span> <span class="k">new</span> <span class="n">Shape</span><span class="o">[]</span> <span class="o">{</span>
            <span class="k">new</span> <span class="n">CircleShape</span><span class="o">(</span><span class="mi">1</span><span class="o">,</span> <span class="mi">2</span><span class="o">,</span> <span class="mi">3</span><span class="o">,</span> <span class="k">new</span> <span class="n">DrawingAPI1</span><span class="o">()),</span>
            <span class="k">new</span> <span class="n">CircleShape</span><span class="o">(</span><span class="mi">5</span><span class="o">,</span> <span class="mi">7</span><span class="o">,</span> <span class="mi">11</span><span class="o">,</span> <span class="k">new</span> <span class="n">DrawingAPI2</span><span class="o">())</span>
        <span class="o">};</span>

        <span class="k">for</span> <span class="o">(</span><span class="n">Shape</span> <span class="n">shape</span> <span class="o">:</span> <span class="n">shapes</span><span class="o">)</span> <span class="o">{</span>
            <span class="n">shape</span><span class="o">.</span><span class="na">resizeByPercentage</span><span class="o">(</span><span class="mf">2.5</span><span class="o">);</span>
            <span class="n">shape</span><span class="o">.</span><span class="na">draw</span><span class="o">();</span>
        <span class="o">}</span>
    <span class="o">}</span>
<span class="o">}</span>
</pre></div>
<p>It will output:</p>
<pre>
API1.circle at 1.000000:2.000000 radius 3.075000
API2.circle at 5.000000:7.000000 radius 11.275000
</pre>
