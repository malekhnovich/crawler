<h1 class="firstHeading" id="firstHeading" lang="en">Intercepting filter pattern</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><p><b>Intercepting Filter</b> is a <a href="/wiki/Java_Platform,_Enterprise_Edition" title="Java Platform, Enterprise Edition">JavaEE</a> <a href="/wiki/Software_design_pattern" title="Software design pattern">pattern</a> which creates pluggable filters to process common services in a standard manner without requiring changes to core request processing code. The filters intercept incoming requests and outgoing responses, allowing preprocessing and post-processing, and these filters can be added or removed unobstrusively without changing existing code.<sup class="reference" id="cite_ref-:o_c2p_if_1-0"><a href="#cite_note-:o_c2p_if-1">[1]</a></sup> This pattern applies reusable processing transparently before and after the actual request execution by the front and page controllers.<sup class="reference" id="cite_ref-:pjeesp_2-0"><a href="#cite_note-:pjeesp-2">[2]</a></sup></p>
<p></p>

<h2><span class="mw-headline" id="Structure">Structure</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Intercepting_filter_pattern&amp;action=edit&amp;section=1" title="Edit section: Structure">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Filter manager, filter chain, filters and target are components of the pattern.</p>
<div class="center">
<div class="floatnone"><a class="image" href="/wiki/File:Intercepting_filter_pattern_class_diagram.gif" title="Intercepting filter patter class diagram"><img alt="Intercepting filter patter class diagram" data-file-height="269" data-file-width="434" height="269" src="//upload.wikimedia.org/wikipedia/en/c/c8/Intercepting_filter_pattern_class_diagram.gif" width="434"/></a></div>
</div>
<div class="center">
<div class="floatnone"><a class="image" href="/wiki/File:Intercepting_filter_pattern_sequence_diagram.png" title="Intercepting filter patter sequence diagram"><img alt="Intercepting filter patter sequence diagram" data-file-height="532" data-file-width="840" height="532" src="//upload.wikimedia.org/wikipedia/en/a/ac/Intercepting_filter_pattern_sequence_diagram.png" width="840"/></a></div>
</div>
<h3><span class="mw-headline" id="Filter_manager">Filter manager</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Intercepting_filter_pattern&amp;action=edit&amp;section=2" title="Edit section: Filter manager">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>This manages filter processing and creates the filter chain with the appropriate filters, in the correct order, and initiates processing.<sup class="reference" id="cite_ref-:o_c2p_if_1-1"><a href="#cite_note-:o_c2p_if-1">[1]</a></sup></p>
<h3><span class="mw-headline" id="Filter_chain">Filter chain</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Intercepting_filter_pattern&amp;action=edit&amp;section=3" title="Edit section: Filter chain">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>This is an ordered collection of independent filters.<sup class="reference" id="cite_ref-:o_c2p_if_1-2"><a href="#cite_note-:o_c2p_if-1">[1]</a></sup></p>
<h3><span class="mw-headline" id="Filters">Filters</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Intercepting_filter_pattern&amp;action=edit&amp;section=4" title="Edit section: Filters">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>These are the individual filters that are mapped to a target and their processing is coordinated by filter chain.<sup class="reference" id="cite_ref-:o_c2p_if_1-3"><a href="#cite_note-:o_c2p_if-1">[1]</a></sup></p>
<h3><span class="mw-headline" id="Target">Target</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Intercepting_filter_pattern&amp;action=edit&amp;section=5" title="Edit section: Target">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>This is the resource requested by the client.<sup class="reference" id="cite_ref-:o_c2p_if_1-4"><a href="#cite_note-:o_c2p_if-1">[1]</a></sup></p>
<h2><span class="mw-headline" id="Consequences">Consequences</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Intercepting_filter_pattern&amp;action=edit&amp;section=6" title="Edit section: Consequences">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Following benefits can be considered:</p>
<ul>
<li>Improved reusability: Common code is centralized in pluggable components enhancing reuse.</li>
<li>Increased flexibility: Generic common components can be applied and removed declaratively, improving flexibility.<sup class="reference" id="cite_ref-:pjeesp_2-1"><a href="#cite_note-:pjeesp-2">[2]</a></sup><sup class="reference" id="cite_ref-:o_c2p_if_1-5"><a href="#cite_note-:o_c2p_if-1">[1]</a></sup></li>
</ul>
<p>Reduced performance can be concerned as unnecessarily long chains of interceptors and filters may hurt performance.<sup class="reference" id="cite_ref-:pjeesp_2-2"><a href="#cite_note-:pjeesp-2">[2]</a></sup></p>

      <span class="c1">// pass request &amp; response through various </span>
      <span class="c1">// filters</span>
      <span class="n">filter</span><span class="o">.</span><span class="na">execute</span><span class="o">(</span><span class="n">request</span><span class="o">,</span> <span class="n">response</span><span class="o">);</span>
    <span class="o">}</span>
  <span class="o">}</span>

  <span class="kd">public</span> <span class="kt">void</span> <span class="nf">addFilter</span><span class="o">(</span><span class="n">Filter</span> <span class="n">filter</span><span class="o">)</span>  <span class="o">{</span>
    <span class="n">myFilters</span><span class="o">.</span><span class="na">add</span><span class="o">(</span><span class="n">filter</span><span class="o">);</span>
  <span class="o">}</span>
<span class="o">}</span>
</pre></div>
