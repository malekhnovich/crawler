<h1 class="firstHeading" id="firstHeading" lang="en">Builder pattern</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><p>The <b>builder pattern</b> is an <a href="/wiki/Creational_pattern" title="Creational pattern">object creation</a> software <a class="mw-redirect" href="/wiki/Design_pattern_(computer_science)" title="Design pattern (computer science)">design pattern</a>. Unlike the <a href="/wiki/Abstract_factory_pattern" title="Abstract factory pattern">abstract factory pattern</a> and the <a href="/wiki/Factory_method_pattern" title="Factory method pattern">factory method pattern</a> whose intention is to enable <a href="/wiki/Polymorphism_(computer_science)" title="Polymorphism (computer science)">polymorphism</a>, the intention of the builder pattern is to find a solution to the telescoping constructor <a href="/wiki/Anti-pattern" title="Anti-pattern">anti-pattern</a><sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="Needs reliable source. The GoF book describes a different purpose for the pattern. (June 2015)">citation needed</span></a></i>]</sup>. The telescoping constructor anti-pattern occurs when the increase of object constructor parameter combination leads to an exponential list of constructors. Instead of using numerous constructors, the builder pattern uses another object, a builder, that receives each initialization parameter step by step and then returns the resulting constructed object at once.</p>
<p>The builder pattern has another benefit. It can be used for objects that contain flat data (<a href="/wiki/HTML" title="HTML">html</a> code, <a href="/wiki/SQL" title="SQL">SQL</a> query, <a href="/wiki/X.509" title="X.509">X.509</a> certificate...), that is to say, data that can't be easily edited. This type of data cannot be edited step by step and must be edited at once.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (July 2015)">citation needed</span></a></i>]</sup></p>
<p>Builder often builds a <a href="/wiki/Composite_pattern" title="Composite pattern">Composite</a>. Often, designs start out using <a href="/wiki/Factory_method_pattern" title="Factory method pattern">Factory Method</a> (less complicated, more customizable, subclasses proliferate) and evolve toward <a href="/wiki/Abstract_factory_pattern" title="Abstract factory pattern">Abstract Factory</a>, <a href="/wiki/Prototype_pattern" title="Prototype pattern">Prototype</a>, or Builder (more flexible, more complex) as the designer discovers where more flexibility is needed. Sometimes creational patterns are complementary: Builder can use one of the other patterns to implement which components are built. Builders are good candidates for a <a href="/wiki/Fluent_interface" title="Fluent interface">fluent interface</a>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="noprint Inline-Template noprint Template-Fact" style="white-space:nowrap;">[<i><a class="mw-redirect" href="/wiki/Wikipedia:NOTRS" title="Wikipedia:NOTRS"><span title="Citation cites this Wikipedia page (June 2016)">better source needed</span></a></i>]</sup></p>
<p></p>

<h2><span class="mw-headline" id="Definition">Definition</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Builder_pattern&amp;action=edit&amp;section=1" title="Edit section: Definition">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The intent of the Builder design pattern is to separate the construction of a complex object from its representation. By doing so the same construction process can create different representations. <sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<h2><span class="mw-headline" id="Advantages.5B3.5D">Advantages<sup class="reference" id="cite_ref-:0_3-0"><a href="#cite_note-:0-3">[3]</a></sup></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Builder_pattern&amp;action=edit&amp;section=2" title="Edit section: Advantages[3]">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>Allows you to vary a product’s internal representation.</li>
<li>Encapsulates code for construction and representation.</li>
<li>Provides control over steps of construction process.</li>
</ul>
<h2><span class="mw-headline" id="Disadvantages.5B3.5D">Disadvantages<sup class="reference" id="cite_ref-:0_3-1"><a href="#cite_note-:0-3">[3]</a></sup></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Builder_pattern&amp;action=edit&amp;section=3" title="Edit section: Disadvantages[3]">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ul>
<li>Requires creating a separate ConcreteBuilder for each different type of Product.</li>
</ul>
<h2><span class="mw-headline" id="Structure">Structure</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Builder_pattern&amp;action=edit&amp;section=4" title="Edit section: Structure">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="center">
<div class="floatnone"><a class="image" href="/wiki/File:Builder_UML_class_diagram.svg" title="Builder Structure"><img alt="Builder Structure" data-file-height="480" data-file-width="1400" height="171" src="//upload.wikimedia.org/wikipedia/commons/thumb/f/f3/Builder_UML_class_diagram.svg/500px-Builder_UML_class_diagram.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/f/f3/Builder_UML_class_diagram.svg/750px-Builder_UML_class_diagram.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/f/f3/Builder_UML_class_diagram.svg/1000px-Builder_UML_class_diagram.svg.png 2x" width="500"/></a></div>
</div>
<dl>
<dt>Builder</dt>
<dd>Abstract interface for creating objects (product).</dd>
</dl>
<dl>
<dt>ConcreteBuilder</dt>
<dd>Provides implementation for Builder. It is an <a class="mw-redirect" href="/wiki/Factory_(software_concept)" title="Factory (software concept)">object able to construct other objects</a>. Constructs and assembles parts to build the objects.</dd>
</dl>
<h2><span class="mw-headline" id="Pseudocode">Pseudocode</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Builder_pattern&amp;action=edit&amp;section=5" title="Edit section: Pseudocode">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>We have a <code>Car</code> class. The problem is that a car has many options. The combination of each option would lead to a huge list of constructors for this class. So we will create a builder class, <code>CarBuilder</code>. We will send to the <code>CarBuilder</code> each car option step by step and then construct the final car with the right options:</p>
<pre>
<b>class</b> Car <b>is</b>
  Can have GPS, trip computer and various numbers of seats.
  Can be a city car, a sports car, or a cabriolet.

<b>class</b> CarBuilder <b>is</b>
  <b>method</b> getResult() <b>is</b>
      <b>output: </b> a <i>Car</i> with the right options
    Construct and return the car.

  <b>method</b> setSeats(number) <b>is</b>
      <b>input: </b> the <i>number</i> of seats the car may have.
    Tell the builder the number of seats.

  <b>method</b> setCityCar() <b>is</b>
    Make the builder remember that the car is a city car.

  <b>method</b> setCabriolet() <b>is</b>
    Make the builder remember that the car is a cabriolet.

  <b>method</b> setSportsCar() <b>is</b>
    Make the builder remember that the car is a sports car.

  <b>method</b> setTripComputer() <b>is</b>
    Make the builder remember that the car has a trip computer.

  <b>method</b> unsetTripComputer() <b>is</b>
    Make the builder remember that the car does not have a trip computer.

  <b>method</b> setGPS() <b>is</b>
    Make the builder remember that the car has a global positioning system.

  <b>method</b> unsetGPS() <b>is</b>
    Make the builder remember that the car does not have a global positioning system.

<b>Construct</b> a CarBuilder called <i>carBuilder</i>
carBuilder.setSeats(2)
carBuilder.setSportsCar()
carBuilder.setTripComputer()
carBuilder.unsetGPS()
car := carBuilder.getResult()
</pre>
<p>Of course one could dispense with Builder and just do this:</p>
<pre>
car = new Car();
car.seats = 2;
car.type = CarType.SportsCar;
car.setTripComputer();
car.unsetGPS();
car.isValid();
</pre>
<p>So this indicates that the Builder pattern is more than just a means to limit constructor proliferation. It removes what could be a complex building process from being the responsibility of the user of the object that is built. It also allows for inserting new implementations of how an object is built without disturbing the client code.</p>

<h3><span class="mw-headline" id="C.2B.2B">C++</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Builder_pattern&amp;action=edit&amp;section=8" title="Edit section: C++">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">////// Product declarations and inline impl. (possibly Product.h) //////</span>
<span class="k">class</span> <span class="nc">Product</span><span class="p">{</span>
	<span class="k">public</span><span class="o">:</span>
		<span class="c1">// use this class to construct Product</span>
		<span class="k">class</span> <span class="nc">Builder</span><span class="p">;</span>

	<span class="k">private</span><span class="o">:</span>
		<span class="c1">// variables in need of initialization to make valid object</span>
		<span class="kt">int</span> <span class="n">i</span><span class="p">;</span>
		<span class="kt">float</span> <span class="n">f</span><span class="p">;</span>

    <span class="k">public</span> <span class="k">function</span> <span class="nf">setColour</span><span class="p">(</span><span class="nv">$colour</span><span class="p">)</span> 
    <span class="p">{</span> 
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">colour</span> <span class="o">=</span> <span class="nv">$colour</span><span class="p">;</span>
    <span class="p">}</span>
    
    <span class="k">public</span> <span class="k">function</span> <span class="nf">getColour</span><span class="p">()</span> 
    <span class="p">{</span> 
        <span class="k">return</span> <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">colour</span><span class="p">;</span>
    <span class="p">}</span>
<span class="p">}</span>

<span class="c1">//The builder abstraction</span>
<span class="k">interface</span> <span class="nx">ICarBuilder</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="k">function</span> <span class="nf">SetColour</span><span class="p">(</span><span class="nv">$colour</span><span class="p">);</span>
    <span class="k">public</span> <span class="k">function</span> <span class="nf">SetWheels</span><span class="p">(</span><span class="nv">$count</span><span class="p">);</span>
    <span class="k">public</span> <span class="k">function</span> <span class="nf">GetResult</span><span class="p">();</span>
<span class="p">}</span>

<span class="c1">//Concrete builder implementation</span>
<span class="k">class</span> <span class="nc">CarBuilder</span> <span class="k">implements</span> <span class="nx">ICarBuilder</span>
<span class="p">{</span>
    <span class="k">private</span> <span class="nv">$_car</span><span class="p">;</span>

    <span class="k">function</span> <span class="nf">__construct</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">_car</span> <span class="o">=</span> <span class="k">new</span> <span class="nx">Car</span><span class="p">();</span>
    <span class="p">}</span>

    <span class="k">public</span> <span class="k">function</span> <span class="nf">SetColour</span><span class="p">(</span><span class="nv">$colour</span><span class="p">)</span>
    <span class="p">{</span>
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">_car</span><span class="o">-&gt;</span><span class="na">Colour</span> <span class="o">=</span> <span class="nv">$colour</span><span class="p">;</span>
    <span class="p">}</span>

    <span class="k">public</span> <span class="k">function</span> <span class="nf">SetWheels</span><span class="p">(</span><span class="nv">$count</span><span class="p">)</span>
    <span class="p">{</span>
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">_car</span><span class="o">-&gt;</span><span class="na">Wheels</span> <span class="o">=</span> <span class="nv">$count</span><span class="p">;</span>
    <span class="p">}</span>

    <span class="k">public</span> <span class="k">function</span> <span class="nf">GetResult</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="k">return</span> <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">_car</span><span class="p">;</span>
    <span class="p">}</span>
<span class="p">}</span>

<span class="c1">//The director</span>
<span class="k">class</span> <span class="nc">CarBuildDirector</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="nv">$builder</span><span class="p">;</span>
    
    <span class="k">function</span> <span class="nf">__construct</span><span class="p">(</span><span class="nv">$color</span> <span class="o">=</span> <span class="s2">"White"</span><span class="p">,</span> <span class="nv">$wheels</span> <span class="o">=</span> <span class="mi">4</span><span class="p">)</span>
    <span class="p">{</span>
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">builder</span> <span class="o">=</span> <span class="k">new</span> <span class="nx">CarBuilder</span><span class="p">();</span>

        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">builder</span><span class="o">-&gt;</span><span class="na">SetColour</span><span class="p">(</span><span class="nv">$color</span><span class="p">);</span>
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">builder</span><span class="o">-&gt;</span><span class="na">SetWheels</span><span class="p">(</span><span class="nv">$wheels</span><span class="p">);</span>
    <span class="p">}</span>
    
    <span class="k">public</span> <span class="k">function</span> <span class="nf">GetResult</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="k">return</span> <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">builder</span><span class="o">-&gt;</span><span class="na">GetResult</span><span class="p">();</span>
    <span class="p">}</span>
<span class="p">}</span>
</pre></div>
