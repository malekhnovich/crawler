<h1 class="firstHeading" id="firstHeading" lang="en">Data transfer object</h1>
<div class="mw-body-content" id="bodyContent">

<p>A <b>data transfer object</b> (<b>DTO</b><sup class="reference" id="cite_ref-msdn_1-0"><a href="#cite_note-msdn-1">[1]</a></sup><sup class="reference" id="cite_ref-fowler_2-0"><a href="#cite_note-fowler-2">[2]</a></sup>) is an object that carries data between processes. The motivation for its use is that communication between processes is usually done resorting to remote interfaces (e.g. web services), where each call is an expensive operation.<sup class="reference" id="cite_ref-fowler_2-1"><a href="#cite_note-fowler-2">[2]</a></sup> Because the majority of the cost of each call is related to the round-trip time between the client and the server, one way of reducing the number of calls is to use an object (the DTO) that aggregates the data that would have been transferred by the several calls, but that is served by one call only.<sup class="reference" id="cite_ref-fowler_2-2"><a href="#cite_note-fowler-2">[2]</a></sup></p>
<p>The difference between data transfer objects and <a href="/wiki/Business_object" title="Business object">business objects</a> or <a href="/wiki/Data_access_object" title="Data access object">data access objects</a> is that a DTO does not have any behavior except for storage and retrieval of its own data (<a href="/wiki/Mutator_method" title="Mutator method">mutators</a> and <a href="/wiki/Method_(computer_programming)" title="Method (computer programming)">accessors</a>). DTOs are simple objects that should not contain any business logic that would require testing.<sup class="reference" id="cite_ref-msdn_1-1"><a href="#cite_note-msdn-1">[1]</a></sup></p>
<p>This <a href="/wiki/Software_design_pattern" title="Software design pattern">pattern</a> is often incorrectly used outside of remote interfaces. This has triggered a response from its author<sup class="reference" id="cite_ref-fowlerLocalDTO_3-0"><a href="#cite_note-fowlerLocalDTO-3">[3]</a></sup> where he reiterates that the whole purpose of DTOs is to shift data in expensive remote calls.</p>
<h2><span class="mw-headline" id="Terminology">Terminology</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Data_transfer_object&amp;action=edit&amp;section=1" title="Edit section: Terminology">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A <a href="/wiki/Value_object" title="Value object">value object</a> is not a DTO. The two terms have been conflated by Java developers in the past.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup></p>
