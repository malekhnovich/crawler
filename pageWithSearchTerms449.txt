<p><b>C#</b><sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[note 2]</a></sup> (pronounced as <i>see sharp</i>) is a <a class="mw-redirect" href="/wiki/Multi-paradigm_programming_language" title="Multi-paradigm programming language">multi-paradigm programming language</a> encompassing <a class="mw-redirect" href="/wiki/Strong_typing" title="Strong typing">strong typing</a>, <a href="/wiki/Imperative_programming" title="Imperative programming">imperative</a>, <a href="/wiki/Declarative_programming" title="Declarative programming">declarative</a>, <a href="/wiki/Functional_programming" title="Functional programming">functional</a>, <a href="/wiki/Generic_programming" title="Generic programming">generic</a>, <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented</a> (<a class="mw-redirect" href="/wiki/Class_(computer_science)" title="Class (computer science)">class</a>-based), and <a href="/wiki/Component-based_software_engineering" title="Component-based software engineering">component-oriented</a> programming disciplines. It was developed by <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> within its <a href="/wiki/.NET_Framework" title=".NET Framework">.NET</a> initiative and later approved as a standard by <a href="/wiki/Ecma_International" title="Ecma International">Ecma</a> (ECMA-334) and <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">ISO</a> (ISO/IEC 23270:2006). C# is one of the programming languages designed for the <a href="/wiki/Common_Language_Infrastructure" title="Common Language Infrastructure">Common Language Infrastructure</a>.</p>
<p>C# is a general-purpose, object-oriented programming language.<sup class="reference" id="cite_ref-ECMA-334_11-0"><a href="#cite_note-ECMA-334-11">[9]</a></sup> Its development team is led by <a href="/wiki/Anders_Hejlsberg" title="Anders Hejlsberg">Anders Hejlsberg</a>. The most recent version is C# 6.0 which was released in 2015.<sup class="reference" id="cite_ref-:0_12-0"><a href="#cite_note-:0-12">[10]</a></sup></p>

<h2><span class="mw-headline" id="Design_goals">Design goals</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=1" title="Edit section: Design goals">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The ECMA standard lists these design goals for C#:<sup class="reference" id="cite_ref-ECMA-334_11-1"><a href="#cite_note-ECMA-334-11">[9]</a></sup></p>
<ul>
<li>The C# language is intended to be a simple, modern, general-purpose, <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented programming</a> language.</li>
<li>The language, and implementations there of, should provide support for software engineering principles such as <a class="mw-redirect" href="/wiki/Strong_type" title="Strong type">strong type</a> checking, array <a href="/wiki/Bounds_checking" title="Bounds checking">bounds checking</a>, detection of attempts to use <a href="/wiki/Uninitialized_variable" title="Uninitialized variable">uninitialized variables</a>, and automatic <a href="/wiki/Garbage_collection_(computer_science)" title="Garbage collection (computer science)">garbage collection</a>. Software robustness, durability, and programmer productivity are important.</li>
<li>The language is intended for use in developing <a class="mw-redirect" href="/wiki/Software_components" title="Software components">software components</a> suitable for deployment in distributed environments.</li>
<li>Portability is very important for source code and programmers, especially those already familiar with <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> and <a href="/wiki/C%2B%2B" title="C++">C++</a>.</li>
<li>Support for <a href="/wiki/Internationalization_and_localization" title="Internationalization and localization">internationalization</a> is very important.</li>
<li>C# is intended to be suitable for writing applications for both hosted and <a href="/wiki/Embedded_system" title="Embedded system">embedded systems</a>, ranging from the very large that use sophisticated <a href="/wiki/Operating_system" title="Operating system">operating systems</a>, down to the very small having dedicated functions.</li>
<li>Although C# applications are intended to be economical with regard to memory and <a class="mw-redirect" href="/wiki/Processing_power" title="Processing power">processing power</a> requirements, the language was not intended to compete directly on performance and size with C or assembly language.</li>
</ul>
<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>During the development of the .NET Framework, the <a class="mw-redirect" href="/wiki/Base_Class_Library" title="Base Class Library">class libraries</a> were originally written using a <a href="/wiki/Managed_code" title="Managed code">managed code</a> compiler system called <i>Simple Managed C</i> (SMC).<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[11]</a></sup><sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[12]</a></sup> In January 1999, <a href="/wiki/Anders_Hejlsberg" title="Anders Hejlsberg">Anders Hejlsberg</a> formed a team to build a new language at the time called Cool, which stood for "C-like Object Oriented Language".<sup class="reference" id="cite_ref-computerworld2008_15-0"><a href="#cite_note-computerworld2008-15">[13]</a></sup> Microsoft had considered keeping the name "Cool" as the final name of the language, but chose not to do so for trademark reasons. By the time the .NET project was publicly announced at the July 2000 <a href="/wiki/Professional_Developers_Conference" title="Professional Developers Conference">Professional Developers Conference</a>, the language had been renamed C#, and the class libraries and <a href="/wiki/ASP.NET" title="ASP.NET">ASP.NET</a> runtime had been ported to C#.</p>
<p>C#'s principal designer and lead architect at Microsoft is Anders Hejlsberg, who was previously involved with the design of <a href="/wiki/Turbo_Pascal" title="Turbo Pascal">Turbo Pascal</a>, <a class="mw-redirect" href="/wiki/Embarcadero_Delphi" title="Embarcadero Delphi">Embarcadero Delphi</a> (formerly CodeGear Delphi, Inprise Delphi and Borland Delphi), and <a href="/wiki/Visual_J%2B%2B" title="Visual J++">Visual J++</a>. In interviews and technical papers he has stated that flaws<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (February 2010)">citation needed</span></a></i>]</sup> in most major programming languages (e.g. <a href="/wiki/C%2B%2B" title="C++">C++</a>, <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, <a class="mw-redirect" href="/wiki/Embarcadero_Delphi" title="Embarcadero Delphi">Delphi</a>, and <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a>) drove the fundamentals of the <a href="/wiki/Common_Language_Runtime" title="Common Language Runtime">Common Language Runtime</a> (CLR), which, in turn, drove the design of the C# language itself.</p>
<p><a href="/wiki/James_Gosling" title="James Gosling">James Gosling</a>, who created the <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> programming language in 1994, and <a href="/wiki/Bill_Joy" title="Bill Joy">Bill Joy</a>, a co-founder of Sun Microsystems, the originator of Java, called C# an "imitation" of Java; Gosling further said that "[C# is] sort of Java with reliability, productivity and security deleted."<sup class="reference" id="cite_ref-gosling_16-0"><a href="#cite_note-gosling-16">[14]</a></sup><sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[15]</a></sup> Klaus Kreft and Angelika Langer (authors of a C++ streams book) stated in a blog post that "Java and C# are almost identical programming languages. Boring repetition that lacks innovation,"<sup class="reference" id="cite_ref-kreft-langer_18-0"><a href="#cite_note-kreft-langer-18">[16]</a></sup> "Hardly anybody will claim that Java or C# are revolutionary programming languages that changed the way we write programs," and "C# borrowed a lot from Java - and vice versa. Now that C# supports <a class="mw-redirect" href="/wiki/Boxing_(Computer_Science)" title="Boxing (Computer Science)">boxing</a> and unboxing, we'll have a very similar feature in Java."<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[17]</a></sup> In July 2000, <a href="/wiki/Anders_Hejlsberg" title="Anders Hejlsberg">Anders Hejlsberg</a> said that C# is "not a Java clone" and is "much closer to C++" in its design.<sup class="reference" id="cite_ref-JohnOsborn_20-0"><a href="#cite_note-JohnOsborn-20">[18]</a></sup></p>
<p>Since the release of C# 2.0 in November 2005, the C# and Java languages have evolved on increasingly divergent trajectories, becoming somewhat less similar. One of the first major departures came with the addition of <a href="/wiki/Generic_programming" title="Generic programming">generics</a> to both languages, with vastly different implementations. C# makes use of <a href="/wiki/Reification_(computer_science)" title="Reification (computer science)">reification</a> to provide "first-class" generic objects that can be used like any other class, with code generation performed at class-load time.<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[19]</a></sup> Furthermore, C# has added several major features to accommodate functional-style programming, culminating in the <a href="/wiki/Language_Integrated_Query" title="Language Integrated Query">LINQ</a> extensions released with C# 3.0 and its supporting framework of <a class="mw-redirect" href="/wiki/Lambda_expressions" title="Lambda expressions">lambda expressions</a>, <a href="/wiki/Extension_method" title="Extension method">extension methods</a>, and anonymous types.<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[20]</a></sup> These features enable C# programmers to use functional programming techniques, such as <a class="mw-redirect" href="/wiki/Closure_(computer_science)" title="Closure (computer science)">closures</a>, when it is advantageous to their application. The LINQ extensions and the functional imports help developers reduce the amount of "boilerplate" code that is included in common tasks like querying a database, parsing an xml file, or searching through a data structure, shifting the emphasis onto the actual program logic to help improve readability and maintainability.<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[21]</a></sup></p>
<p>C# used to have a <a href="/wiki/Mascot" title="Mascot">mascot</a> called Andy (named after <a href="/wiki/Anders_Hejlsberg" title="Anders Hejlsberg">Anders Hejlsberg</a>). It was retired on January 29, 2004.<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[22]</a></sup></p>
<p>C# was originally submitted to the ISO subcommittee JTC 1/SC 22 for review,<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[23]</a></sup> under ISO/IEC 23270:2003,<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[24]</a></sup> was withdrawn and was then approved under ISO/IEC 23270:2006.<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[25]</a></sup></p>
<h3><span class="mw-headline" id="Name">Name</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=3" title="Edit section: Name">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="thumb tright">
<div class="thumbinner" style="width:102px;"><a class="image" href="/wiki/File:Treblecsharp5.svg"><img alt="" class="thumbimage" data-file-height="228" data-file-width="274" height="83" src="//upload.wikimedia.org/wikipedia/commons/thumb/c/c6/Treblecsharp5.svg/100px-Treblecsharp5.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/c/c6/Treblecsharp5.svg/150px-Treblecsharp5.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/c/c6/Treblecsharp5.svg/200px-Treblecsharp5.svg.png 2x" width="100"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Treblecsharp5.svg" title="Enlarge"></a></div>
<a href="/wiki/C%E2%99%AF_(musical_note)" title="C♯ (musical note)">C-sharp musical note</a></div>
</div>
</div>
<p>The name "C sharp" was inspired by musical notation where a <a href="/wiki/Sharp_(music)" title="Sharp (music)">sharp</a> indicates that the written note should be made a <a href="/wiki/Semitone" title="Semitone">semitone</a> higher in <a href="/wiki/Pitch_(music)" title="Pitch (music)">pitch</a>.<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[26]</a></sup> This is similar to the language name of <a href="/wiki/C%2B%2B" title="C++">C++</a>, where "++" indicates that a variable should be incremented by 1. The sharp symbol also resembles a <a href="/wiki/Typographic_ligature" title="Typographic ligature">ligature</a> of four "+" symbols (in a two-by-two grid), further implying that the language is an increment of C++.<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[27]</a></sup></p>
<p>Due to technical limitations of display (standard fonts, browsers, etc.) and the fact that the sharp symbol (<span class="nowrap"><span style="font-family: monospace, monospace;">U+266F</span></span> <span style="font-size:125%">♯</span> <a href="/wiki/Sharp_(music)" title="Sharp (music)"><span class="smallcaps" style="font-variant: small-caps; font-size: smaller;">MUSIC SHARP SIGN</span></a> (HTML <code>&amp;#9839;</code>)) is not present on most <a href="/wiki/Keyboard_layout" title="Keyboard layout">keyboard layouts</a>, the <a href="/wiki/Number_sign" title="Number sign">number sign</a> (<span class="nowrap"><span style="font-family: monospace, monospace;">U+0023</span></span> <span style="font-size:125%">#</span> <span class="smallcaps" style="font-variant: small-caps; font-size: smaller;">NUMBER SIGN</span> (HTML <code>&amp;#35;</code>)) was chosen to approximate the sharp symbol in the written name of the programming language.<sup class="reference" id="cite_ref-MS_CSharp_FAQ_30-0"><a href="#cite_note-MS_CSharp_FAQ-30">[28]</a></sup> This convention is reflected in the ECMA-334 C# Language Specification.<sup class="reference" id="cite_ref-ECMA-334_11-2"><a href="#cite_note-ECMA-334-11">[9]</a></sup> However, when it is practical to do so (for example, in advertising or in box art<sup class="reference" id="cite_ref-31"><a href="#cite_note-31">[29]</a></sup>), Microsoft uses the intended musical symbol.</p>
<p>The "sharp" suffix has been used by a number of other .NET languages that are variants of existing languages, including <a href="/wiki/J_Sharp" title="J Sharp">J#</a> (a .NET language also designed by Microsoft that is derived from Java 1.1), <a href="/wiki/A_Sharp_(.NET)" title="A Sharp (.NET)">A#</a> (from <a href="/wiki/Ada_(programming_language)" title="Ada (programming language)">Ada</a>), and the <a href="/wiki/Functional_programming" title="Functional programming">functional programming</a> language <a href="/wiki/F_Sharp_(programming_language)" title="F Sharp (programming language)">F#</a>.<sup class="reference" id="cite_ref-MS_FSharp_FAQ_32-0"><a href="#cite_note-MS_FSharp_FAQ-32">[30]</a></sup> The original implementation of <a href="/wiki/EiffelStudio" title="EiffelStudio">Eiffel for .NET</a> was called Eiffel#,<sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[31]</a></sup> a name retired since the full <a href="/wiki/Eiffel_(programming_language)" title="Eiffel (programming language)">Eiffel</a> language is now supported. The suffix has also been used for <a href="/wiki/Library_(computing)" title="Library (computing)">libraries</a>, such as <a class="mw-redirect" href="/wiki/Gtk_Sharp" title="Gtk Sharp">Gtk#</a> (a .NET <a class="mw-redirect" href="/wiki/Wrapper_pattern" title="Wrapper pattern">wrapper</a> for <a href="/wiki/GTK%2B" title="GTK+">GTK+</a> and other <a href="/wiki/GNOME" title="GNOME">GNOME</a> libraries) and <a href="/wiki/Cocoa_Sharp" title="Cocoa Sharp">Cocoa#</a> (a wrapper for <a href="/wiki/Cocoa_(API)" title="Cocoa (API)">Cocoa</a>).</p>

<h2><span class="mw-headline" id="Syntax">Syntax</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=12" title="Edit section: Syntax">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/C_Sharp_syntax" title="C Sharp syntax">C Sharp syntax</a></div>
<div class="hatnote" role="note">See also: <a href="/wiki/Syntax_(programming_languages)" title="Syntax (programming languages)">Syntax (programming languages)</a></div>
<p>The core syntax of C# language is similar to that of other C-style languages such as C, C++ and Java. In particular:</p>
<ul>
<li>Semicolons are used to denote the end of a statement.</li>
<li><a class="mw-redirect" href="/wiki/Curly_brackets" title="Curly brackets">Curly brackets</a> are used to group statements. Statements are commonly grouped into methods (functions), methods into classes, and classes into <a class="mw-redirect" href="/wiki/Namespaces" title="Namespaces">namespaces</a>.</li>
<li>Variables are assigned using an <a href="/wiki/Equals_sign" title="Equals sign">equals sign</a>, but compared using <a class="mw-redirect" href="/wiki/%3D%3D" title="==">two consecutive equals signs</a>.</li>
<li><a class="mw-redirect" href="/wiki/Square_brackets" title="Square brackets">Square brackets</a> are used with <a href="/wiki/Array_data_structure" title="Array data structure">arrays</a>, both to declare them and to get a value at a given index in one of them.</li>
</ul>
<h2><span class="mw-headline" id="Distinguishing_features">Distinguishing features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=13" title="Edit section: Distinguishing features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">See also: <a href="/wiki/Comparison_of_C_Sharp_and_Java" title="Comparison of C Sharp and Java">Comparison of C Sharp and Java</a></div>
<p>Some notable features of C# that distinguish it from C, C++, and Java where noted, are:</p>
<h3><span class="mw-headline" id="Portability">Portability</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=14" title="Edit section: Portability">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>By design, C# is the programming language that most directly reflects the underlying <a href="/wiki/Common_Language_Infrastructure" title="Common Language Infrastructure">Common Language Infrastructure</a> (CLI).<sup class="reference" id="cite_ref-46"><a href="#cite_note-46">[42]</a></sup> Most of its intrinsic types correspond to value-types implemented by the CLI framework. However, the language specification does not state the code generation requirements of the compiler: that is, it does not state that a C# compiler must target a Common Language Runtime, or generate <a href="/wiki/Common_Intermediate_Language" title="Common Intermediate Language">Common Intermediate Language</a> (CIL), or generate any other specific format. Theoretically, a C# compiler could generate machine code like traditional compilers of C++ or <a href="/wiki/Fortran" title="Fortran">Fortran</a>.</p>
<h3><span class="mw-headline" id="Typing">Typing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=15" title="Edit section: Typing">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>C# supports strongly typed implicit variable declarations with the keyword <code>var</code>, and implicitly typed arrays with the keyword <code>new[]</code> followed by a collection initializer.</p>
<p>C# supports a strict <a href="/wiki/Boolean_data_type" title="Boolean data type">Boolean data type</a>, <code>bool</code>. Statements that take conditions, such as <code>while</code> and <code>if</code>, require an expression of a type that implements the <code>true</code> operator, such as the Boolean type. While C++ also has a Boolean type, it can be freely converted to and from integers, and expressions such as <code>if(a)</code> require only that <code>a</code> is convertible to bool, allowing <code>a</code> to be an int, or a pointer. C# disallows this "integer meaning true or false" approach, on the grounds that forcing programmers to use expressions that return exactly <code>bool</code> can prevent certain types of programming mistakes such as <code>if (a = b)</code> (use of assignment <code>=</code> instead of equality <code>==</code>, which while not an error in C or C++, will be caught by the compiler anyway).</p>
<p>C# is more <a href="/wiki/Type_safety" title="Type safety">type safe</a> than C++. The only implicit conversions by default are those that are considered safe, such as widening of integers. This is enforced at compile-time, during <a href="/wiki/Just-in-time_compilation" title="Just-in-time compilation">JIT</a>, and, in some cases, at runtime. No implicit conversions occur between Booleans and integers, nor between enumeration members and integers (except for literal 0, which can be implicitly converted to any enumerated type). Any user-defined conversion must be explicitly marked as explicit or implicit, unlike C++ <a class="mw-redirect" href="/wiki/Copy_constructor" title="Copy constructor">copy constructors</a> and conversion operators, which are both implicit by default.</p>
<p>C# has explicit support for <a href="/wiki/Covariance_and_contravariance_(computer_science)" title="Covariance and contravariance (computer science)">covariance and contravariance</a> in generic types, unlike C++ which has some degree of support for contravariance simply through the semantics of return types on virtual methods.</p>
<p><a href="/wiki/Enumerated_type" title="Enumerated type">Enumeration</a> members are placed in their own <a class="mw-redirect" href="/wiki/Scope_(programming)" title="Scope (programming)">scope</a>.</p>
<p>The C# language does not allow for global variables or functions. All methods and members must be declared within classes. Static members of public classes can substitute for global variables and functions.</p>
<p>Local variables cannot <a href="/wiki/Variable_shadowing" title="Variable shadowing">shadow</a> variables of the enclosing block, unlike C and C++.</p>
<h3><span class="mw-headline" id="Meta_programming">Meta programming</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=16" title="Edit section: Meta programming">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Metaprogramming" title="Metaprogramming">Meta programming</a> via C# attributes is part of the language. Many of these attributes duplicate the functionality of GCC's and VisualC++'s platform-dependent preprocessor directives.</p>
<h3><span class="mw-headline" id="Methods_and_functions">Methods and functions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=17" title="Edit section: Methods and functions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Like C++, and unlike Java, C# programmers must use the keyword <code>virtual</code> to allow methods to be overridden by subclasses.</p>
<p><i>Extension methods</i> in C# allow programmers to use static methods as if they were methods from a class's method table, allowing programmers to add methods to an object that they feel should exist on that object and its derivatives.</p>
<p>The type <code>dynamic</code> allows for run-time method binding, allowing for JavaScript-like method calls and run-time object composition.</p>
<p>C# has support for strongly-typed function pointers via the keyword <code>delegate</code>. Like the Qt framework's pseudo-C++ <i>signal</i> and <i>slot</i>, C# has semantics specifically surrounding publish-subscribe style events, though C# uses delegates to do so.</p>
<p>C# offers Java-like <code>synchronized</code> method calls, via the attribute <code>[MethodImpl(MethodImplOptions.Synchronized)]</code>, and has support for <a href="/wiki/Mutual_exclusion" title="Mutual exclusion">mutually-exclusive locks</a> via the keyword <code>lock</code>.</p>
<h3><span class="mw-headline" id="Property">Property</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=18" title="Edit section: Property">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>C# provides <a href="/wiki/Property_(programming)" title="Property (programming)">properties</a> as <a href="/wiki/Syntactic_sugar" title="Syntactic sugar">syntactic sugar</a> for a common pattern in which a pair of methods, <a href="/wiki/Mutator_method" title="Mutator method">accessor (getter) and mutator (setter)</a> encapsulate operations on a single <a href="/wiki/Attribute_(computing)" title="Attribute (computing)">attribute</a> of a class. No redundant method signatures for the getter/setter implementations need be written, and the property may be accessed using attribute syntax rather than more verbose method calls.</p>
<h3><span class="mw-headline" id="Namespace">Namespace</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=19" title="Edit section: Namespace">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A C# <code>namespace</code> provides the same level of code isolation as a Java <code>package</code> or a C++ <code class="mw-highlight" dir="ltr"><span class="k">namespace</span></code>, with very similar rules and features to a <code>package</code>.</p>
<h3><span class="mw-headline" id="Memory_access">Memory access</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=20" title="Edit section: Memory access">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In C#, memory address pointers can only be used within blocks specifically marked as <i>unsafe</i>, and programs with unsafe code need appropriate permissions to run. Most object access is done through safe object references, which always either point to a "live" object or have the well-defined <a href="/wiki/Nullable_type" title="Nullable type">null</a> value; it is impossible to obtain a reference to a "dead" object (one that has been garbage collected), or to a random block of memory. An unsafe pointer can point to an instance of a value-type, array, string, or a block of memory allocated on a stack. Code that is not marked as unsafe can still store and manipulate pointers through the <code>System.IntPtr</code> type, but it cannot dereference them.</p>
<p>Managed memory cannot be explicitly freed; instead, it is automatically garbage collected. Garbage collection addresses the problem of <a href="/wiki/Memory_leak" title="Memory leak">memory leaks</a> by freeing the programmer of responsibility for releasing memory that is no longer needed.</p>
<h3><span class="mw-headline" id="Exception">Exception</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=21" title="Edit section: Exception">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a class="mw-redirect" href="/wiki/Checked_exceptions" title="Checked exceptions">Checked exceptions</a> are not present in C# (in contrast to Java). This has been a conscious decision based on the issues of scalability and versionability.<sup class="reference" id="cite_ref-47"><a href="#cite_note-47">[43]</a></sup></p>
<h3><span class="mw-headline" id="Polymorphism">Polymorphism</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=22" title="Edit section: Polymorphism">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Unlike <a href="/wiki/C%2B%2B" title="C++">C++</a>, C# does not support <a href="/wiki/Multiple_inheritance" title="Multiple inheritance">multiple inheritance</a>, although a class can implement any number of interfaces. This was a design decision by the language's lead architect to avoid complication and simplify architectural requirements throughout CLI. When implementing multiple interfaces that contain a method with the same signature, C# allows implementing each method depending on which interface that method is being called through, or, like Java, allows implementing the method once, and have that be the one invocation on a call through any of the class's interfaces.</p>
<p>However, unlike <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, C# supports <a href="/wiki/Operator_overloading" title="Operator overloading">operator overloading</a>. Only the most commonly overloaded operators in C++ may be overloaded in C#.</p>
<h3><span class="mw-headline" id="Functional_programming">Functional programming</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=23" title="Edit section: Functional programming">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Though primarily an imperative language, C# 2.0 offered limited support for functional programming through <a class="mw-redirect" href="/wiki/First-class_functions" title="First-class functions">first-class functions</a> and closures in the form of anonymous delegates. C# 3.0 expanded support for functional programming with the introduction of a lightweight syntax for lambda expressions, extension methods (an affordance for modules), and a <a href="/wiki/List_comprehension" title="List comprehension">list comprehension</a> syntax in the form of a "query comprehension" language.</p>
<h2><span class="mw-headline" id="Common_type_system">Common type system</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=24" title="Edit section: Common type system">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>C# has a <i>unified type system</i>. This unified type system is called <a href="/wiki/Common_Type_System" title="Common Type System">Common Type System</a> (CTS).<sup class="reference" id="cite_ref-insidecsharpp2ch4_48-0"><a href="#cite_note-insidecsharpp2ch4-48">[44]</a></sup></p>
<p>A unified type system implies that all types, including primitives such as integers, are subclasses of the <code class="mw-highlight" dir="ltr"><span class="n">System</span><span class="p">.</span><span class="n">Object</span></code> class. For example, every type inherits a <code class="mw-highlight" dir="ltr"><span class="n">ToString</span><span class="p">()</span></code> method.</p>
<h3><span class="mw-headline" id="Categories_of_data_types">Categories of data types</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=25" title="Edit section: Categories of data types">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>CTS separates data types into two categories:<sup class="reference" id="cite_ref-insidecsharpp2ch4_48-1"><a href="#cite_note-insidecsharpp2ch4-48">[44]</a></sup></p>
<ol>
<li>Reference types</li>
<li>Value types</li>
</ol>
<p>Instances of value types do not have referential identity nor referential comparison semantics - equality and inequality comparisons for value types compare the actual data values within the instances, unless the corresponding operators are overloaded. Value types are derived from <code class="mw-highlight" dir="ltr"><span class="n">System</span><span class="p">.</span><span class="n">ValueType</span></code>, always have a default value, and can always be created and copied. Some other limitations on value types are that they cannot derive from each other (but can implement interfaces) and cannot have an explicit default (parameterless) constructor. Examples of value types are all primitive types, such as <code class="mw-highlight" dir="ltr"><span class="kt">int</span></code> (a signed 32-bit integer), <code class="mw-highlight" dir="ltr"><span class="kt">float</span></code> (a 32-bit IEEE floating-point number), <code class="mw-highlight" dir="ltr"><span class="kt">char</span></code> (a 16-bit Unicode code unit), and <code class="mw-highlight" dir="ltr"><span class="n">System</span><span class="p">.</span><span class="n">DateTime</span></code> (identifies a specific point in time with nanosecond precision). Other examples are <code class="mw-highlight" dir="ltr"><span class="k">enum</span></code> (enumerations) and <code class="mw-highlight" dir="ltr"><span class="k">struct</span></code> (user defined structures).</p>
<p>In contrast, reference types have the notion of referential identity - each instance of a reference type is inherently distinct from every other instance, even if the data within both instances is the same. This is reflected in default equality and inequality comparisons for reference types, which test for referential rather than structural equality, unless the corresponding operators are overloaded (such as the case for <code class="mw-highlight" dir="ltr"><span class="n">System</span><span class="p">.</span><span class="n">String</span></code>). In general, it is not always possible to create an instance of a reference type, nor to copy an existing instance, or perform a value comparison on two existing instances, though specific reference types can provide such services by exposing a public constructor or implementing a corresponding interface (such as <code class="mw-highlight" dir="ltr"><span class="n">ICloneable</span></code> or <code class="mw-highlight" dir="ltr"><span class="n">IComparable</span></code>). Examples of reference types are <code class="mw-highlight" dir="ltr"><span class="kt">object</span></code> (the ultimate base class for all other C# classes), <code class="mw-highlight" dir="ltr"><span class="n">System</span><span class="p">.</span><span class="n">String</span></code> (a string of Unicode characters), and <code class="mw-highlight" dir="ltr"><span class="n">System</span><span class="p">.</span><span class="n">Array</span></code> (a base class for all C# arrays).</p>
<p>Both type categories are extensible with user-defined types.</p>
<h3><span class="mw-headline" id="Boxing_and_unboxing">Boxing and unboxing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=26" title="Edit section: Boxing and unboxing">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><i>Boxing</i> is the operation of converting a value-type object into a value of a corresponding reference type.<sup class="reference" id="cite_ref-insidecsharpp2ch4_48-2"><a href="#cite_note-insidecsharpp2ch4-48">[44]</a></sup> Boxing in C# is implicit.</p>
<p><i>Unboxing</i> is the operation of converting a value of a reference type (previously boxed) into a value of a value type.<sup class="reference" id="cite_ref-insidecsharpp2ch4_48-3"><a href="#cite_note-insidecsharpp2ch4-48">[44]</a></sup> Unboxing in C# requires an explicit <a href="/wiki/Type_conversion" title="Type conversion">type cast</a>. A boxed object of type T can only be unboxed to a T (or a nullable T).<sup class="reference" id="cite_ref-49"><a href="#cite_note-49">[45]</a></sup></p>
<p>Example:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kt">int</span> <span class="n">foo</span> <span class="p">=</span> <span class="m">42</span><span class="p">;</span>         <span class="c1">// Value type.</span>
<span class="kt">object</span> <span class="n">bar</span> <span class="p">=</span> <span class="n">foo</span><span class="p">;</span>     <span class="c1">// foo is boxed to bar.</span>
<span class="kt">int</span> <span class="n">foo2</span> <span class="p">=</span> <span class="p">(</span><span class="kt">int</span><span class="p">)</span><span class="n">bar</span><span class="p">;</span>  <span class="c1">// Unboxed back to value type.</span>
</pre></div>
<h2><span class="mw-headline" id="Libraries">Libraries</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=27" title="Edit section: Libraries">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The C# specification details a minimum set of types and class libraries that the compiler expects to have available. In practice, C# is most often used with some implementation of the <a href="/wiki/Common_Language_Infrastructure" title="Common Language Infrastructure">Common Language Infrastructure</a> (CLI), which is standardized as ECMA-335 <i>Common Language Infrastructure (CLI)</i>.</p>
<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=28" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The following is a very simple C# program, a version of the classic "<a class="mw-redirect" href="/wiki/Hello_world" title="Hello world">Hello world</a>" example:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">using</span> <span class="nn">System</span><span class="p">;</span>

<span class="k">class</span> <span class="nc">Program</span>
<span class="p">{</span>
    <span class="k">static</span> <span class="k">void</span> <span class="nf">Main</span><span class="p">(</span><span class="n">String</span><span class="p">[]</span> <span class="n">args</span><span class="p">)</span>
    <span class="p">{</span>
        <span class="n">Console</span><span class="p">.</span><span class="n">WriteLine</span><span class="p">(</span><span class="s">"Hello, world!"</span><span class="p">);</span>
    <span class="p">}</span>
<span class="p">}</span>
</pre></div>
<p>The effect is to write the following text to the output console:</p>
<pre>
Hello, world!
</pre>
<p>Each line has a purpose:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">using</span> <span class="nn">System</span><span class="p">;</span>
</pre></div>
<p>The above line of code tells the compiler to use <code>System</code> as a candidate prefix for types used in the source code. In this case, when the compiler sees use of the <code>Console</code> type later in the source code, it tries to find a type named <code>Console</code>, first in the current assembly, followed by all referenced assemblies. In this case the compiler fails to find such a type, since the name of the type is actually <code>System.Console</code>. The compiler then attempts to find a type named <code>System.Console</code> by using the <code>System</code> prefix from the <code class="mw-highlight" dir="ltr"><span class="k">using</span></code> statement, and this time it succeeds. The <code class="mw-highlight" dir="ltr"><span class="k">using</span></code> statement allows the programmer to state all candidate prefixes to use during compilation instead of always using full type names.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">class</span> <span class="nc">Program</span>
</pre></div>
<p>Above is a <a class="mw-redirect" href="/wiki/Class_(computer_science)" title="Class (computer science)">class</a> definition. Everything between the following pair of braces describes <code class="mw-highlight" dir="ltr"><span class="n">Program</span></code>.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">static</span> <span class="k">void</span> <span class="nf">Main</span><span class="p">(</span><span class="kt">string</span><span class="p">[]</span> <span class="n">args</span><span class="p">)</span>
</pre></div>
<p>This declares the class member method where the program begins execution. The .NET runtime calls the <code class="mw-highlight" dir="ltr"><span class="n">Main</span></code> method. (Note: <code class="mw-highlight" dir="ltr"><span class="n">Main</span></code> may also be called from elsewhere, like any other method, e.g. from another method of <code class="mw-highlight" dir="ltr"><span class="n">Program</span></code>.) The <a href="/wiki/Method_(computer_programming)#Static_methods" title="Method (computer programming)"><code class="mw-highlight" dir="ltr"><span class="k">static</span></code> keyword</a> makes the method accessible without an instance of <code class="mw-highlight" dir="ltr"><span class="n">Program</span></code>. Each console application's <code class="mw-highlight" dir="ltr"><span class="n">Main</span></code> entry point must be declared <code class="mw-highlight" dir="ltr"><span class="k">static</span></code>. Otherwise, the program would require an instance, but any instance would require a program. To avoid that irresolvable <a href="/wiki/Circular_dependency" title="Circular dependency">circular dependency</a>, C# compilers processing <a href="/wiki/Console_application" title="Console application">console applications</a> (like that above) report an error, if there is no <code class="mw-highlight" dir="ltr"><span class="k">static</span> <span class="n">Main</span></code> method. The <code class="mw-highlight" dir="ltr"><span class="k">void</span></code> keyword declares that <code class="mw-highlight" dir="ltr"><span class="n">Main</span></code> has no <a class="mw-redirect" href="/wiki/Return_value" title="Return value">return value</a>.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">Console</span><span class="p">.</span><span class="n">WriteLine</span><span class="p">(</span><span class="s">"Hello, world!"</span><span class="p">);</span>
</pre></div>
<p>This line writes the output. <code class="mw-highlight" dir="ltr"><span class="n">Console</span></code> is a static class in the <code class="mw-highlight" dir="ltr"><span class="n">System</span></code> namespace. It provides an interface to the standard input, output, and error streams for console applications. The program calls the <code class="mw-highlight" dir="ltr"><span class="n">Console</span></code> method <code class="mw-highlight" dir="ltr"><span class="n">WriteLine</span></code>, which displays on the console a line with the argument, the string <code class="mw-highlight" dir="ltr"><span class="s">"Hello, world!"</span></code>.</p>
<p>A <a class="mw-redirect" href="/wiki/GUI" title="GUI">GUI</a> example:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">using</span> <span class="nn">System.Windows.Forms</span><span class="p">;</span>

<span class="k">class</span> <span class="nc">Program</span>
<span class="p">{</span>
    <span class="k">static</span> <span class="k">void</span> <span class="nf">Main</span><span class="p">(</span><span class="kt">string</span><span class="p">[]</span> <span class="n">args</span><span class="p">)</span>
    <span class="p">{</span>
        <span class="n">MessageBox</span><span class="p">.</span><span class="n">Show</span><span class="p">(</span><span class="s">"Hello, world!"</span><span class="p">);</span>
    <span class="p">}</span>
<span class="p">}</span>
</pre></div>
<p>This example is similar to the previous example, except that it generates a <a href="/wiki/Dialog_box" title="Dialog box">dialog box</a> that contains the message "Hello, world!" instead of writing it to the console.</p>
<h2><span class="mw-headline" id="Standardization_and_licensing">Standardization and licensing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=29" title="Edit section: Standardization and licensing">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In August 2001, Microsoft Corporation, Hewlett-Packard and Intel Corporation co-sponsored the submission of specifications for C# as well as the Common Language Infrastructure (CLI) to the standards organization <a href="/wiki/Ecma_International" title="Ecma International">Ecma International</a>. In December 2001, ECMA released ECMA-334 <i>C# Language Specification</i>. C# became an <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">ISO</a> standard in 2003 (ISO/IEC 23270:2003 - <i>Information technology — Programming languages — C#</i>). ECMA had previously adopted equivalent specifications as the 2nd edition of C#, in December 2002.</p>
<p>In June 2005, ECMA approved edition 3 of the C# specification, and updated ECMA-334. Additions included partial classes, anonymous methods, nullable types, and <a href="/wiki/Generic_programming" title="Generic programming">generics</a> (somewhat similar to C++ <a class="mw-redirect" href="/wiki/Template_(programming)" title="Template (programming)">templates</a>).</p>
<p>In July 2005, ECMA submitted to ISO/IEC JTC 1, via the latter's Fast-Track process, the standards and related TRs. This process usually takes 6–9 months.</p>
<p>The C# language definition and the <a href="/wiki/Common_Language_Infrastructure" title="Common Language Infrastructure">CLI</a> are standardized under <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">ISO</a> and <a class="mw-redirect" href="/wiki/Ecma" title="Ecma">Ecma</a> standards that provide <a class="mw-redirect" href="/wiki/Reasonable_and_Non_Discriminatory_Licensing" title="Reasonable and Non Discriminatory Licensing">reasonable and non-discriminatory licensing</a> protection from patent claims.</p>
<p>Microsoft has agreed not to sue open source developers for violating patents in non-profit projects for the part of the framework that is covered by the OSP.<sup class="reference" id="cite_ref-50"><a href="#cite_note-50">[46]</a></sup> Microsoft has also agreed not to enforce patents relating to <a href="/wiki/Novell" title="Novell">Novell</a> products against Novell's paying customers<sup class="reference" id="cite_ref-51"><a href="#cite_note-51">[47]</a></sup> with the exception of a list of products that do not explicitly mention C#, .NET or Novell's implementation of .NET (<a href="/wiki/Mono_(software)" title="Mono (software)">The Mono Project</a>).<sup class="reference" id="cite_ref-52"><a href="#cite_note-52">[48]</a></sup> However, Novell maintains that Mono does not infringe any Microsoft patents.<sup class="reference" id="cite_ref-53"><a href="#cite_note-53">[49]</a></sup> Microsoft has also made a specific agreement not to enforce patent rights related to the <a href="/wiki/Moonlight_(runtime)" title="Moonlight (runtime)">Moonlight browser plugin</a>, which depends on Mono, provided it is obtained through <a href="/wiki/Novell" title="Novell">Novell</a>.<sup class="reference" id="cite_ref-MsCovenant_54-0"><a href="#cite_note-MsCovenant-54">[50]</a></sup></p>
<h2><span class="mw-headline" id="Implementations">Implementations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=C_Sharp_(programming_language)&amp;action=edit&amp;section=30" title="Edit section: Implementations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The reference C# compiler is <a href="/wiki/Microsoft_Visual_C_Sharp" title="Microsoft Visual C Sharp">Microsoft Visual C#</a>, which is open-source.<sup class="reference" id="cite_ref-55"><a href="#cite_note-55">[51]</a></sup></p>
<p>Microsoft is leading the development of a new <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open-source</a> C# compiler and set of tools, previously codenamed "<a class="mw-redirect" href="/wiki/Microsoft_Roslyn" title="Microsoft Roslyn">Roslyn</a>". The compiler, which is entirely written in <a href="/wiki/Managed_code" title="Managed code">managed code</a> (C#), has been opened up and functionality surfaced as APIs. It is thus enabling developers to create refactoring and diagnostics tools.</p>
<p>Other C# compilers exist, often including an implementation of the <a href="/wiki/Common_Language_Infrastructure" title="Common Language Infrastructure">Common Language Infrastructure</a> and the .NET class libraries up to .NET 2.0:</p>
<ul>
<li>The <a href="/wiki/Mono_(software)" title="Mono (software)">Mono</a> project provides an open-source C# compiler, a complete open-source implementation of the Common Language Infrastructure including the required framework libraries as they appear in the ECMA specification, and a nearly complete implementation of the Microsoft proprietary .NET class libraries up to .NET 3.5. As of Mono 2.6, no plans exist to implement <a href="/wiki/Windows_Presentation_Foundation" title="Windows Presentation Foundation">WPF</a>; <a href="/wiki/Windows_Workflow_Foundation" title="Windows Workflow Foundation">WF</a> is planned for a later release; and there are only partial implementations of <a href="/wiki/Language_Integrated_Query#LINQ_to_SQL" title="Language Integrated Query">LINQ to SQL</a> and <a href="/wiki/Windows_Communication_Foundation" title="Windows Communication Foundation">WCF</a>.<sup class="reference" id="cite_ref-56"><a href="#cite_note-56">[52]</a></sup></li>
<li>The <a href="/wiki/DotGNU" title="DotGNU">DotGNU</a> project (now discontinued) also provided an open-source C# compiler, a nearly complete implementation of the Common Language Infrastructure including the required framework libraries as they appear in the ECMA specification, and subset of some of the remaining Microsoft proprietary .NET class libraries up to .NET 2.0 (those not documented or included in the ECMA specification, but included in Microsoft's standard .NET Framework distribution).</li>
<li>Microsoft's Rotor project (currently called <a href="/wiki/Shared_Source_Common_Language_Infrastructure" title="Shared Source Common Language Infrastructure">Shared Source Common Language Infrastructure</a>) (licensed for educational and research use only) provides a <a href="/wiki/Shared_source" title="Shared source">shared source</a> implementation of the CLR runtime and a C# compiler, and a subset of the required <a href="/wiki/Common_Language_Infrastructure" title="Common Language Infrastructure">Common Language Infrastructure</a> framework libraries in the ECMA specification (up to C# 2.0, and supported on Windows XP only).</li>
</ul>
