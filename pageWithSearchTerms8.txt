<p><b>Computer programming</b> (often shortened to <b>programming</b>) is a process that leads from an original formulation of a <a href="/wiki/Computing" title="Computing">computing</a> problem to <a href="/wiki/Executable" title="Executable">executable</a> <a href="/wiki/Computer_program" title="Computer program">computer programs</a>. Programming involves activities such as analysis, developing understanding, generating <a href="/wiki/Algorithm" title="Algorithm">algorithms</a>, verification of requirements of algorithms including their correctness and resources consumption, and implementation (commonly referred to as <b>coding</b><sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup>) of algorithms in a target <a href="/wiki/Programming_language" title="Programming language">programming language</a>. <a href="/wiki/Source_code_editor" title="Source code editor">Source code</a> is written in one or more <a href="/wiki/Programming_language" title="Programming language">programming languages</a>. The purpose of programming is to find a sequence of instructions that will automate performing a specific task or solving a given problem. The process of programming thus often requires expertise in many different subjects, including knowledge of the <a href="/wiki/Application_domain" title="Application domain">application domain</a>, specialized algorithms and <a href="/wiki/Logic" title="Logic">formal logic</a>.</p>
<p>Related tasks include <a href="/wiki/Software_testing" title="Software testing">testing</a>, <a href="/wiki/Debugging" title="Debugging">debugging</a>, and maintaining the <a href="/wiki/Source_code" title="Source code">source code</a>, implementation of the build system, and management of derived artifacts such as <a href="/wiki/Machine_code" title="Machine code">machine code</a> of <a href="/wiki/Computer_program" title="Computer program">computer programs</a>. These might be considered part of the programming process, but often the term <i><a href="/wiki/Software_development" title="Software development">software development</a></i> is used for this larger process with the term <i>programming</i>, <i>implementation</i>, or <i>coding</i> reserved for the actual writing of source code. <a href="/wiki/Software_engineering" title="Software engineering">Software engineering</a> combines <a href="/wiki/Engineering" title="Engineering">engineering</a> techniques with <a href="/wiki/Software_development" title="Software development">software development</a> practices.</p>

<h2><span class="mw-headline" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_programming&amp;action=edit&amp;section=1" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Within <a href="/wiki/Software_engineering" title="Software engineering">software engineering</a>, programming (the <i>implementation</i>) is regarded as one phase in a <a href="/wiki/Software_development_process" title="Software development process">software development process</a>.</p>
<p>There is an ongoing debate on the extent to which the writing of programs is an <a href="/wiki/Art" title="Art">art</a> form, a <a href="/wiki/Craft" title="Craft">craft</a>, or an <a href="/wiki/Engineering" title="Engineering">engineering</a> discipline.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> In general, good programming is considered to be the measured application of all three, with the goal of producing an efficient and evolvable <a href="/wiki/Software" title="Software">software</a> solution (the criteria for "efficient" and "evolvable" vary considerably). The discipline differs from many other technical professions in that <a href="/wiki/Programmer" title="Programmer">programmers</a>, in general, do not need to be licensed or pass any standardized (or governmentally regulated) certification tests in order to call themselves "programmers" or even "software engineers." Because the discipline covers many areas, which may or may not include critical applications, it is debatable whether licensing is required for the profession as a whole. In most cases, the discipline is self-governed by the entities which require the programming, and sometimes very strict environments are defined (e.g. <a href="/wiki/United_States_Air_Force" title="United States Air Force">United States Air Force</a> use of <a class="new" href="/w/index.php?title=AdaCore&amp;action=edit&amp;redlink=1" title="AdaCore (page does not exist)">AdaCore</a> and security clearance). However, representing oneself as a "professional software engineer" without a license from an accredited institution is <a href="/wiki/Regulation_and_licensure_in_engineering" title="Regulation and licensure in engineering">illegal in many parts of the world</a>.</p>
<p>Another ongoing debate is the extent to which the <a href="/wiki/Programming_language" title="Programming language">programming language</a> used in writing computer programs affects the form that the final program takes.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (December 2013)">citation needed</span></a></i>]</sup> This debate is analogous to that surrounding the <a href="/wiki/Linguistic_relativity" title="Linguistic relativity">Sapir–Whorf hypothesis</a><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> in <a href="/wiki/Linguistics" title="Linguistics">linguistics</a> and <a href="/wiki/Cognitive_science" title="Cognitive science">cognitive science</a>, which postulates that a particular spoken language's nature influences the habitual thought of its speakers. Different language patterns yield different patterns of <a href="/wiki/Thought" title="Thought">thought</a>. This idea challenges the possibility of representing the world perfectly with language, because it acknowledges that the mechanisms of any language condition the thoughts of its speaker community.</p>

<p>Ancient cultures seemed to have no conception of computing beyond <a href="/wiki/Arithmetic" title="Arithmetic">arithmetic</a>, <a href="/wiki/Algebra" title="Algebra">algebra</a>, and <a href="/wiki/Geometry" title="Geometry">geometry</a>, occasionally devising computational systems with elements of <a href="/wiki/Calculus" title="Calculus">calculus</a> (e.g. the <a href="/wiki/Method_of_exhaustion" title="Method of exhaustion">method of exhaustion</a>). The only mechanical device that existed for numerical computation at the beginning of human history was the <a href="/wiki/Abacus" title="Abacus">abacus</a>, invented in <a href="/wiki/Sumer" title="Sumer">Sumeria</a> circa 2500 BC. Later, the <a href="/wiki/Antikythera_mechanism" title="Antikythera mechanism">Antikythera mechanism</a>, invented some time around 100 BC in <a href="/wiki/Ancient_Greece" title="Ancient Greece">ancient Greece</a>, is the first known <a href="/wiki/Mechanical_calculator" title="Mechanical calculator">mechanical calculator</a> utilizing gears of various sizes and configuration to perform calculations,<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> which tracked the <a href="/wiki/Metonic_cycle" title="Metonic cycle">metonic cycle</a> still used in lunar-to-solar calendars, and which is consistent for calculating the dates of the <a href="/wiki/Olympiad" title="Olympiad">Olympiads</a>.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></p>
<p>The medieval scientist <a class="mw-redirect" href="/wiki/Al-Jazari" title="Al-Jazari">Al-Jazari</a> built programmable <a class="mw-redirect" href="/wiki/Automata" title="Automata">automata</a> in 1206 AD. One system employed in these devices was the use of pegs and <a href="/wiki/Cam" title="Cam">cams</a> placed into a wooden drum at specific locations, which would sequentially trigger <a href="/wiki/Lever" title="Lever">levers</a> that in turn operated <a href="/wiki/Percussion_instrument" title="Percussion instrument">percussion instruments</a>. The output of this device was a small drummer playing various rhythms and drum patterns.<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> The <a href="/wiki/Jacquard_loom" title="Jacquard loom">Jacquard loom</a>, which Joseph Marie Jacquard developed in 1801, uses a series of <a href="/wiki/Card_stock" title="Card stock">pasteboard</a> cards with holes punched in them. The hole pattern represented the pattern that the loom had to follow in weaving cloth. The loom could produce entirely different weaves using different sets of cards.</p>
<p><a href="/wiki/Charles_Babbage" title="Charles Babbage">Charles Babbage</a> adopted the use of <a class="mw-redirect" href="/wiki/Punched_cards" title="Punched cards">punched cards</a> around 1830 to control his <a href="/wiki/Analytical_Engine" title="Analytical Engine">Analytical Engine</a>. Mathematician <a href="/wiki/Ada_Lovelace" title="Ada Lovelace">Ada Lovelace</a>, a friend of Babbage, between 1842 and 1843 translated an article by Italian military engineer <a class="mw-redirect" href="/wiki/Luigi_Menabrea" title="Luigi Menabrea">Luigi Menabrea</a> on the engine,<sup class="reference" id="cite_ref-fourmilab.ch_8-0"><a href="#cite_note-fourmilab.ch-8">[8]</a></sup> which she supplemented with a set of notes, simply called Notes. These notes include an <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> to calculate a sequence of <a class="mw-redirect" href="/wiki/Bernoulli_numbers" title="Bernoulli numbers">Bernoulli numbers</a>,<sup class="reference" id="cite_ref-IEEE_9-0"><a href="#cite_note-IEEE-9">[9]</a></sup> intended to be carried out by a machine. Despite controversy over scope of her contribution, many consider this algorithm to be the first computer program.<sup class="reference" id="cite_ref-fourmilab.ch_8-1"><a href="#cite_note-fourmilab.ch-8">[8]</a></sup></p>

<p>In the 1880s, <a href="/wiki/Herman_Hollerith" title="Herman Hollerith">Herman Hollerith</a> invented the recording of data on a medium that could then be read by a machine. Prior uses of machine readable media, above, had been for lists of instructions (not data) to drive <a href="/wiki/Program_(machine)" title="Program (machine)">programmed machines</a> such as Jacquard looms and <a href="/wiki/Music_roll" title="Music roll">mechanized musical instruments</a>. "After some initial trials with paper tape, he settled on <a href="/wiki/Punched_card" title="Punched card">punched cards</a>..."<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> To process these punched cards, first known as "Hollerith cards" he invented the <a href="/wiki/Keypunch" title="Keypunch">keypunch</a>, sorter, and <a href="/wiki/Tabulating_machine" title="Tabulating machine">tabulator</a> <a href="/wiki/Unit_record_equipment" title="Unit record equipment">unit record machines</a>.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> These inventions were the foundation of the data processing industry. In 1896 he founded the <i><a class="mw-redirect" href="/wiki/Tabulating_Machine_Company" title="Tabulating Machine Company">Tabulating Machine Company</a></i> (which later became the core of <a href="/wiki/IBM" title="IBM">IBM</a>). The addition of a <a href="/wiki/Plugboard" title="Plugboard">control panel</a> (plugboard) to his 1906 Type I Tabulator allowed it to do different jobs without having to be physically rebuilt. By the late 1940s, there were several unit record calculators, such as the <a href="/wiki/IBM_602" title="IBM 602">IBM 602</a> and <a href="/wiki/IBM_604" title="IBM 604">IBM 604</a>, whose control panels specified a sequence (list) of operations and thus were programmable machines.</p>
<p>The invention of the <a href="/wiki/Von_Neumann_architecture" title="Von Neumann architecture">von Neumann architecture</a> allowed computer programs to be stored in <a href="/wiki/Computer_memory" title="Computer memory">computer memory</a>. Early programs had to be painstakingly crafted using the instructions (elementary operations) of the particular machine, often in <a class="mw-redirect" href="/wiki/Binary_numeral_system" title="Binary numeral system">binary</a> notation. Every model of computer would likely use different instructions (<a class="mw-redirect" href="/wiki/Machine_language" title="Machine language">machine language</a>) to do the same task. Later, <a href="/wiki/Assembly_language" title="Assembly language">assembly languages</a> were developed that let the programmer specify each instruction in a text format, entering abbreviations for each operation code instead of a number and specifying addresses in symbolic form (e.g., ADD X, TOTAL). Entering a program in assembly language is usually more convenient, faster, and less prone to human error than using machine language, but because an assembly language is little more than a different notation for a machine language, any two machines with different instruction sets also have different assembly languages.</p>

<p>The synthesis of numerical calculation, predetermined operation and output, along with a way to organize and input instructions in a manner relatively easy for humans to conceive and produce, led to the modern development of computer programming. In 1954, <a class="mw-redirect" href="/wiki/FORTRAN" title="FORTRAN">FORTRAN</a> was invented; it was the first widely used <a class="mw-redirect" href="/wiki/High-level_language" title="High-level language">high level programming language</a> to have a functional implementation, as opposed to just a design on paper.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup><sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> (A high-level language is, in very general terms, any programming language that allows the programmer to write programs in terms that are more <a class="mw-redirect" href="/wiki/Abstraction_(computer_science)" title="Abstraction (computer science)">abstract</a> than assembly language instructions, i.e. at a level of abstraction "higher" than that of an assembly language.) It allowed programmers to specify calculations by entering a formula directly (e.g. <span class="nowrap">Y = X*2 + 5*X + 9</span>). The program text, or <i>source</i>, is converted into machine instructions using a special program called a <a href="/wiki/Compiler" title="Compiler">compiler</a>, which translates the FORTRAN program into machine language. In fact, the name FORTRAN stands for "Formula Translation". Many other languages were developed, including some for commercial programming, such as <a href="/wiki/COBOL" title="COBOL">COBOL</a>. Programs were mostly still entered using punched cards or <a class="mw-redirect" href="/wiki/Paper_tape" title="Paper tape">paper tape</a>. (See <a class="mw-redirect" href="/wiki/Computer_programming_in_the_punch_card_era" title="Computer programming in the punch card era">computer programming in the punch card era</a>). By the late 1960s, <a href="/wiki/Data_storage_device" title="Data storage device">data storage devices</a> and <a href="/wiki/Computer_terminal" title="Computer terminal">computer terminals</a> became inexpensive enough that programs could be created by typing directly into the computers. <a href="/wiki/Text_editor" title="Text editor">Text editors</a> were developed that allowed changes and corrections to be made much more easily than with punched cards. (Usually, an error in punching a card meant that the card had to be discarded and a new one punched to replace it.)</p>
<p>As time has progressed, computers have made giant leaps in processing power, which have allowed the development of programming languages that are more abstracted from the underlying hardware. Popular programming languages of the modern era include <a href="/wiki/ActionScript" title="ActionScript">ActionScript</a>, <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>, <a href="/wiki/C%2B%2B" title="C++">C++</a>, <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a>, <a href="/wiki/Haskell_(programming_language)" title="Haskell (programming language)">Haskell</a>, <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>, <a href="/wiki/Objective-C" title="Objective-C">Objective-C</a>, <a href="/wiki/Perl" title="Perl">Perl</a>, <a href="/wiki/PHP" title="PHP">PHP</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a>, Smalltalk, <a href="/wiki/SQL" title="SQL">SQL</a>, <a href="/wiki/Visual_Basic" title="Visual Basic">Visual Basic</a>, and dozens more.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> Although these high-level languages usually incur greater <a class="mw-redirect" href="/wiki/Computational_overhead" title="Computational overhead">overhead</a>, the increase in speed of modern computers has made the use of these languages much more practical than in the past. These increasingly abstracted languages are typically easier to learn and allow the programmer to develop applications much more efficiently and with less source code. However, high-level languages are still impractical for a few programs, such as those where low-level hardware control is necessary or where maximum processing speed is vital. Computer programming has become a popular career in the developed world, particularly in the United States, Europe, and Japan. Due to the high labor cost of programmers in these countries, some forms of programming have been increasingly subject to <a href="/wiki/Outsourcing" title="Outsourcing">outsourcing</a> (importing software and services from other countries, usually at a lower wage), making programming career decisions in developed countries more complicated, while increasing economic opportunities for programmers in less developed areas, particularly <a href="/wiki/China" title="China">China</a> and <a href="/wiki/India" title="India">India</a>.</p>

<h3><span class="mw-headline" id="Quality_requirements">Quality requirements</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_programming&amp;action=edit&amp;section=4" title="Edit section: Quality requirements">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Whatever the approach to development may be, the final program must satisfy some fundamental properties. The following properties are among the most important:</p>
<ul>
<li><a href="/wiki/Reliability_engineering#Software_reliability" title="Reliability engineering">Reliability</a>: how often the results of a program are correct. This depends on conceptual correctness of algorithms, and minimization of programming mistakes, such as mistakes in resource management (e.g., <a href="/wiki/Buffer_overflow" title="Buffer overflow">buffer overflows</a> and <a href="/wiki/Race_condition" title="Race condition">race conditions</a>) and logic errors (such as division by zero or <a href="/wiki/Off-by-one_error" title="Off-by-one error">off-by-one errors</a>).</li>
<li><a href="/wiki/Robustness_(computer_science)" title="Robustness (computer science)">Robustness</a>: how well a program anticipates problems due to errors (not bugs). This includes situations such as incorrect, inappropriate or corrupt data, unavailability of needed resources such as memory, operating system services and network connections, user error, and unexpected power outages.</li>
<li><a href="/wiki/Usability" title="Usability">Usability</a>: the <a class="mw-redirect" href="/wiki/Ergonomics" title="Ergonomics">ergonomics</a> of a program: the ease with which a person can use the program for its intended purpose or in some cases even unanticipated purposes. Such issues can make or break its success even regardless of other issues. This involves a wide range of textual, graphical and sometimes hardware elements that improve the clarity, intuitiveness, cohesiveness and completeness of a program's user interface.</li>
<li><a href="/wiki/Software_portability" title="Software portability">Portability</a>: the range of <a href="/wiki/Computer_hardware" title="Computer hardware">computer hardware</a> and <a href="/wiki/Operating_system" title="Operating system">operating system</a> platforms on which the source code of a program can be <a class="mw-redirect" href="/wiki/Compiled" title="Compiled">compiled</a>/<a href="/wiki/Interpreter_(computing)" title="Interpreter (computing)">interpreted</a> and run. This depends on differences in the programming facilities provided by the different platforms, including hardware and operating system resources, expected behavior of the hardware and operating system, and availability of platform specific compilers (and sometimes libraries) for the language of the source code.</li>
<li><a href="/wiki/Maintainability" title="Maintainability">Maintainability</a>: the ease with which a program can be modified by its present or future developers in order to make improvements or customizations, fix <a href="/wiki/Software_bug" title="Software bug">bugs</a> and <a href="/wiki/Vulnerability_(computing)" title="Vulnerability (computing)">security holes</a>, or adapt it to new environments. Good practices<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup> during initial development make the difference in this regard. This quality may not be directly apparent to the end user but it can significantly affect the fate of a program over the long term.</li>
<li><a href="/wiki/Algorithmic_efficiency" title="Algorithmic efficiency">Efficiency</a>/<a href="/wiki/Performance_engineering" title="Performance engineering">performance</a>: Measure of system resources a program consumes (processor time, memory space, slow devices such as disks, network bandwidth and to some extent even user interaction): the less, the better. This also includes careful management of resources, for example cleaning up <a href="/wiki/Temporary_file" title="Temporary file">temporary files</a> and eliminating <a href="/wiki/Memory_leak" title="Memory leak">memory leaks</a>.</li>
</ul>
<h3><span class="mw-headline" id="Readability_of_source_code">Readability of source code</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_programming&amp;action=edit&amp;section=5" title="Edit section: Readability of source code">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In computer programming, <a href="/wiki/Readability" title="Readability">readability</a> refers to the ease with which a human reader can comprehend the purpose, control flow, and operation of source code. It affects the aspects of quality above, including portability, usability and most importantly maintainability.</p>
<p>Readability is important because programmers spend the majority of their time reading, trying to understand and modifying existing source code, rather than writing new source code. Unreadable code often leads to bugs, inefficiencies, and <a class="mw-redirect" href="/wiki/Code_duplication" title="Code duplication">duplicated code</a>. A study<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> found that a few simple readability transformations made code shorter and drastically reduced the time to understand it.</p>
<p>Following a consistent <a href="/wiki/Programming_style" title="Programming style">programming style</a> often helps readability. However, readability is more than just programming style. Many factors, having little or nothing to do with the ability of the computer to efficiently compile and execute the code, contribute to readability.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup> Some of these factors include:</p>
<ul>
<li>Different <a class="mw-redirect" href="/wiki/Indentation_style" title="Indentation style">indentation styles</a> (whitespace)</li>
<li><a href="/wiki/Comment_(computer_programming)" title="Comment (computer programming)">Comments</a></li>
<li><a href="/wiki/Decomposition_(computer_science)" title="Decomposition (computer science)">Decomposition</a></li>
<li><a class="mw-redirect" href="/wiki/Naming_conventions_(programming)" title="Naming conventions (programming)">Naming conventions</a> for objects (such as variables, classes, procedures, etc.)</li>
</ul>
<p>Various <a href="/wiki/Visual_programming_language" title="Visual programming language">visual programming languages</a> have also been developed with the intent to resolve readability concerns by adopting non-traditional approaches to code structure and display. Techniques like <a href="/wiki/Code_refactoring" title="Code refactoring">Code refactoring</a> can enhance readability.</p>
<h3><span class="mw-headline" id="Algorithmic_complexity">Algorithmic complexity</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_programming&amp;action=edit&amp;section=6" title="Edit section: Algorithmic complexity">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The academic field and the engineering practice of computer programming are both largely concerned with discovering and implementing the most efficient algorithms for a given class of problem. For this purpose, algorithms are classified into <i>orders</i> using so-called <a href="/wiki/Big_O_notation" title="Big O notation">Big O notation</a>, which expresses resource use, such as execution time or memory consumption, in terms of the size of an input. Expert programmers are familiar with a variety of well-established algorithms and their respective complexities and use this knowledge to choose algorithms that are best suited to the circumstances.</p>
<h3><span class="mw-headline" id="Methodologies">Methodologies</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_programming&amp;action=edit&amp;section=7" title="Edit section: Methodologies">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The first step in most formal software development processes is requirements analysis, followed by testing to determine value modeling, implementation, and failure elimination (debugging). There exist a lot of differing approaches for each of those tasks. One approach popular for <a href="/wiki/Requirements_analysis" title="Requirements analysis">requirements analysis</a> is <a class="mw-redirect" href="/wiki/Use_Case" title="Use Case">Use Case</a> analysis. Many programmers use forms of <a href="/wiki/Agile_software_development" title="Agile software development">Agile software development</a> where the various stages of formal software development are more integrated together into short cycles that take a few weeks rather than years. There are many approaches to the Software development process.</p>
<p>Popular modeling techniques include Object-Oriented Analysis and Design (<a class="mw-redirect" href="/wiki/OOAD" title="OOAD">OOAD</a>) and Model-Driven Architecture (<a class="mw-redirect" href="/wiki/Model-Driven_Architecture" title="Model-Driven Architecture">MDA</a>). The Unified Modeling Language (<a href="/wiki/Unified_Modeling_Language" title="Unified Modeling Language">UML</a>) is a notation used for both the OOAD and MDA.</p>
<p>A similar technique used for database design is Entity-Relationship Modeling (<a class="mw-redirect" href="/wiki/Entity-Relationship_Model" title="Entity-Relationship Model">ER Modeling</a>).</p>
<p>Implementation techniques include imperative languages (<a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented</a> or <a href="/wiki/Procedural_programming" title="Procedural programming">procedural</a>), <a href="/wiki/Functional_programming" title="Functional programming">functional languages</a>, and <a href="/wiki/Logic_programming" title="Logic programming">logic languages</a>.</p>
<h3><span class="mw-headline" id="Measuring_language_usage">Measuring language usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_programming&amp;action=edit&amp;section=8" title="Edit section: Measuring language usage">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote" role="note">Main article: <a href="/wiki/Measuring_programming_language_popularity" title="Measuring programming language popularity">Measuring programming language popularity</a></div>
<p>It is very difficult to determine what are the most popular of modern programming languages. Methods of measuring programming language popularity include: counting the number of job advertisements that mention the language,<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup> the number of books sold and courses teaching the language (this overestimates the importance of newer languages), and estimates of the number of existing lines of code written in the language (this underestimates the number of users of business languages such as COBOL).</p>
<p>Some languages are very popular for particular kinds of applications, while some languages are regularly used to write many different kinds of applications. For example, <a href="/wiki/COBOL" title="COBOL">COBOL</a> is still strong in corporate data centers<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup> often on large <a href="/wiki/Mainframe_computer" title="Mainframe computer">mainframe computers</a>, <a href="/wiki/Fortran" title="Fortran">Fortran</a> in engineering applications, <a href="/wiki/Scripting_language" title="Scripting language">scripting languages</a> in <a href="/wiki/World_Wide_Web" title="World Wide Web">Web</a> development, and <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> in <a href="/wiki/Embedded_software" title="Embedded software">embedded software</a>. Many applications use a mix of several languages in their construction and use. New languages are generally designed around the syntax of a prior language with new functionality added, (for example <a href="/wiki/C%2B%2B" title="C++">C++</a> adds object-orientation to C, and Java adds memory management and <a href="/wiki/Bytecode" title="Bytecode">bytecode</a> to C++, but as a result, loses efficiency and the ability for low-level manipulation).</p>

<p>Debugging is a very important task in the software development process since having defects in a program can have significant consequences for its users. Some languages are more prone to some kinds of faults because their specification does not require compilers to perform as much checking as other languages. Use of a <a class="mw-redirect" href="/wiki/Static_code_analysis" title="Static code analysis">static code analysis</a> tool can help detect some possible problems.</p>
<p>Debugging is often done with <a href="/wiki/Integrated_development_environment" title="Integrated development environment">IDEs</a> like <a href="/wiki/Eclipse_(software)" title="Eclipse (software)">Eclipse</a>, <a class="mw-redirect" href="/wiki/Visual_Studio" title="Visual Studio">Visual Studio</a>, <a class="mw-redirect" href="/wiki/Kdevelop" title="Kdevelop">Kdevelop</a>, <a href="/wiki/NetBeans" title="NetBeans">NetBeans</a> and <a href="/wiki/Code::Blocks" title="Code::Blocks">Code::Blocks</a>. Standalone debuggers like <a class="mw-redirect" href="/wiki/Gdb" title="Gdb">gdb</a> are also used, and these often provide less of a visual environment, usually using a <a class="mw-redirect" href="/wiki/Command_line" title="Command line">command line</a>.</p>
<h2><span class="mw-headline" id="Programming_languages">Programming languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Computer_programming&amp;action=edit&amp;section=10" title="Edit section: Programming languages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="hatnote" role="note">Main articles: <a href="/wiki/Programming_language" title="Programming language">Programming language</a> and <a href="/wiki/List_of_programming_languages" title="List of programming languages">List of programming languages</a></div>
<p>Different programming languages support different styles of programming (called <i><a href="/wiki/Programming_paradigm" title="Programming paradigm">programming paradigms</a></i>). The choice of language used is subject to many considerations, such as company policy, suitability to task, availability of third-party packages, or individual preference. Ideally, the programming language best suited for the task at hand will be selected. Trade-offs from this ideal involve finding enough programmers who know the language to build a team, the availability of compilers for that language, and the efficiency with which programs written in a given language execute. Languages form an approximate spectrum from "low-level" to "high-level"; "low-level" languages are typically more machine-oriented and faster to execute, whereas "high-level" languages are more abstract and easier to use but execute less quickly. It is usually easier to code in "high-level" languages than in "low-level" ones.</p>
<p><a class="mw-redirect" href="/wiki/Allen_Downey" title="Allen Downey">Allen Downey</a>, in his book <i>How To Think Like A Computer Scientist</i>, writes:</p>
<dl>
<dd>The details look different in different languages, but a few basic instructions appear in just about every language:
<ul>
<li>Input: Gather data from the keyboard, a file, or some other device.</li>
<li>Output: Display data on the screen or send data to a file or other device.</li>
<li>Arithmetic: Perform basic arithmetical operations like addition and multiplication.</li>
<li>Conditional Execution: Check for certain conditions and execute the appropriate sequence of statements.</li>
<li>Repetition: Perform some action repeatedly, usually with some variation.</li>
</ul>
</dd>
</dl>
<p>Many computer languages provide a mechanism to call functions provided by <a class="mw-redirect" href="/wiki/Shared_library" title="Shared library">shared libraries</a>. Provided the functions in a library follow the appropriate run time conventions (e.g., method of passing <a class="mw-redirect" href="/wiki/Argument_(computer_science)" title="Argument (computer science)">arguments</a>), then these functions may be written in any other language.</p>
