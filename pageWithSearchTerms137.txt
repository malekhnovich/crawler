<h1 class="firstHeading" id="firstHeading" lang="en">Active record pattern</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><p>In <a href="/wiki/Software_engineering" title="Software engineering">software engineering</a>, the <b>active record pattern</b> is an <a class="mw-redirect" href="/wiki/Architectural_pattern_(computer_science)" title="Architectural pattern (computer science)">architectural pattern</a> found in software that stores in-memory object data in <a href="/wiki/Relational_database" title="Relational database">relational databases</a>. It was named by <a href="/wiki/Martin_Fowler" title="Martin Fowler">Martin Fowler</a> in his 2003 book <i>Patterns of Enterprise Application Architecture</i>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> The interface of an object conforming to this pattern would include functions such as Insert, Update, and Delete, plus properties that correspond more or less directly to the columns in the underlying database table.</p>
<p>The active record pattern is an approach to accessing data in a <a href="/wiki/Database" title="Database">database</a>. A <a class="mw-redirect" href="/wiki/Database_table" title="Database table">database table</a> or <a class="mw-redirect" href="/wiki/View_(database)" title="View (database)">view</a> is wrapped into a <a class="mw-redirect" href="/wiki/Class_(computer_science)" title="Class (computer science)">class</a>. Thus, an <a href="/wiki/Object_(computer_science)" title="Object (computer science)">object</a> instance is tied to a single row in the table. After creation of an object, a new row is added to the table upon save. Any object loaded gets its information from the database. When an object is updated, the corresponding row in the table is also updated. The wrapper class implements <a class="mw-redirect" href="/wiki/Accessor" title="Accessor">accessor</a> <a href="/wiki/Method_(computer_programming)" title="Method (computer programming)">methods</a> or properties for each column in the table or view.</p>
<p>This pattern is commonly used by object persistence tools and in <a href="/wiki/Object-relational_mapping" title="Object-relational mapping">object-relational mapping</a> (ORM). Typically, <a href="/wiki/Foreign_key" title="Foreign key">foreign key</a> relationships will be exposed as an object instance of the appropriate type via a property.</p>
<p></p>

<h2><span class="mw-headline" id="Implementations">Implementations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Active_record_pattern&amp;action=edit&amp;section=1" title="Edit section: Implementations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Implementations of the concept can be found in various <a href="/wiki/Software_framework" title="Software framework">frameworks</a> for many programming environments. For example, if in a database there is a table <code>parts</code> with columns <code>name</code> (string type) and <code>price</code> (number type), and the Active Record pattern is implemented in the class <code>Part</code>, the pseudo-code</p>
<pre>
part = new Part()
part.name = "Sample part"
part.price = 123.45
part.save()
</pre>
<p>will create a new row in the <code>parts</code> table with the given values, and is roughly equivalent to the <a href="/wiki/SQL" title="SQL">SQL</a> command</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">INSERT</span> <span class="k">INTO</span> <span class="n">parts</span> <span class="p">(</span><span class="n">name</span><span class="p">,</span> <span class="n">price</span><span class="p">)</span> <span class="k">VALUES</span> <span class="p">(</span><span class="s1">'Sample part'</span><span class="p">,</span> <span class="mi">123</span><span class="p">.</span><span class="mi">45</span><span class="p">);</span>
</pre></div>
<p>Conversely, the class can be used to query the database:</p>
<pre>
b = Part.find_first("name", "gearbox")
</pre>
<p>This will find a new <code>Part</code> object based on the first matching row from the <code>parts</code> table whose <code>name</code> column has the value "gearbox". The SQL command used might be similar to the following, depending on the SQL implementation details of the database:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="k">SELECT</span> <span class="o">*</span> <span class="k">FROM</span> <span class="n">parts</span> <span class="k">WHERE</span> <span class="n">name</span> <span class="o">=</span> <span class="s1">'gearbox'</span> <span class="k">LIMIT</span> <span class="mi">1</span><span class="p">;</span> <span class="c1">-- MySQL or PostgreSQL</span>
</pre></div>
<h3><span class="mw-headline" id="ColdFusion">ColdFusion</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Active_record_pattern&amp;action=edit&amp;section=2" title="Edit section: ColdFusion">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a class="mw-redirect" href="/wiki/ColdFusion" title="ColdFusion">ColdFusion</a> has an open source implementation of the active record pattern.</p>
<p>The <a href="/wiki/ColdFusion_on_Wheels" title="ColdFusion on Wheels">ColdFusion on Wheels</a> framework has an implementation of the active record pattern. It is open source and has the added advantage of requiring no complex configuration.</p>
<h3><span class="mw-headline" id="PHP">PHP</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Active_record_pattern&amp;action=edit&amp;section=3" title="Edit section: PHP">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>PHP ActiveRecord is one <a class="mw-redirect" href="/wiki/Open-source" title="Open-source">open-source</a> library designed to fulfill the active record pattern.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p>Several open-source <a href="/wiki/PHP" title="PHP">PHP</a> frameworks also bundle their own <a href="/wiki/Object-relational_mapping" title="Object-relational mapping">ORM</a> implementing the active record pattern. Most implementations support relationships, behaviors, <a class="mw-redirect" href="/wiki/Input_validation" title="Input validation">validation</a>, <a href="/wiki/Serialization" title="Serialization">serialization</a> and support for multiple data sources.</p>
<ul>
<li><a class="new" href="/w/index.php?title=Boiler_MVC&amp;action=edit&amp;redlink=1" title="Boiler MVC (page does not exist)">Boiler</a>, an <a class="mw-redirect" href="/wiki/MVC_model" title="MVC model">MVC</a> framework for PHP, contains a set of tools for auto-generation of active record models.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> The project, designed for data-centered projects, aims to automate as much of the development process as possible,<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> using <a href="/wiki/Apache_Ant" title="Apache Ant">Apache Ant</a>. Although a new addition to Open Source market,<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> the project is already in use in many live applications, both commercially and open. The framework currently only supports <a href="/wiki/MySQL" title="MySQL">MySQL</a> though the developers have reported some commercial work in Postgres.</li>
<li><a class="new" href="/w/index.php?title=Cygnite_PHP_Framework&amp;action=edit&amp;redlink=1" title="Cygnite PHP Framework (page does not exist)">Cygnite PHP Framework</a>'s default database layer implements Active Record pattern which closely resemble with <a href="/wiki/Ruby_on_Rails" title="Ruby on Rails">Ruby on Rails</a>.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup></li>
<li><a href="/wiki/Laravel" title="Laravel">Laravel</a> contains an ORM called 'Eloquent' which implements the active record pattern, closely resembling that of <a href="/wiki/Ruby_on_Rails" title="Ruby on Rails">Ruby on Rails</a> <sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></li>
<li><a href="/wiki/CakePHP" title="CakePHP">CakePHP</a>'s ORM implements the active record pattern,<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> but as of version 2.x queries return arrays of data, with levels of <b>related data</b> as required. Version 3.0 uses objects.</li>
<li><a href="/wiki/Lithium_(software)" title="Lithium (software)">Lithium</a>'s ORM implements active record.</li>
<li><a href="/wiki/Symfony" title="Symfony">Symfony</a>'s default database layer and ORM "<a href="/wiki/Doctrine_(PHP)" title="Doctrine (PHP)">Doctrine</a>" does not implement active record but rather a <a href="/wiki/Data_mapper_pattern" title="Data mapper pattern">data mapper</a> approach.</li>
<li><span id="codeigniter"><a href="/wiki/EllisLab#CodeIgniter" title="EllisLab">CodeIgniter</a></span> has a query builder it calls "ActiveRecord", but which does not implement the Active Record pattern. Instead, it implements what the user guide refers to as a modified version of the pattern. The Active Record functionality in CodeIgniter can be achieved by using either CodeIgniter DataMapper library or CodeIgniter Gas ORM library.</li>
<li><a href="/wiki/Yii" title="Yii">Yii</a>'s ORM also implements the active record pattern.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup></li>
<li><a href="/wiki/Propel_(PHP)" title="Propel (PHP)">Propel</a> also implements the active record pattern.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></li>
<li>Paris is A lightweight Active Record implementation for PHP5, built on top of Idiorm.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup></li>
</ul>
<h3><span class="mw-headline" id="Ruby">Ruby</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Active_record_pattern&amp;action=edit&amp;section=4" title="Edit section: Ruby">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a> library ActiveRecord implements ORM. It creates a <a href="/wiki/Persistence_(computer_science)" title="Persistence (computer science)">persistable</a> domain model from business objects and database tables, where logic and data are presented as a unified package. ActiveRecord adds <a class="mw-redirect" href="/wiki/Inheritance_(computer_science)" title="Inheritance (computer science)">inheritance</a> and <a href="/wiki/Association_(object-oriented_programming)" title="Association (object-oriented programming)">associations</a> to the pattern above, solving two substantial limitations of that pattern. A set of macros acts as a domain language for the latter, and the <a href="/wiki/Single_Table_Inheritance" title="Single Table Inheritance">Single Table Inheritance</a> pattern is integrated for the former; thus, ActiveRecord increases the functionality of the active record pattern approach to database interaction. ActiveRecord is the default ‘model’ component of the <a class="mw-redirect" href="/wiki/Model-view-controller" title="Model-view-controller">model-view-controller</a> web-application framework <a href="/wiki/Ruby_on_Rails" title="Ruby on Rails">Ruby on Rails</a>, and is also a stand-alone ORM package for other Ruby applications. In both forms, it was conceived of by <a href="/wiki/David_Heinemeier_Hansson" title="David Heinemeier Hansson">David Heinemeier Hansson</a>, and has been improved upon by a number of contributors.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup></p>
<p>Other, less popular ORMs have been released since ActiveRecord first took the stage. For example, <a class="mw-redirect" href="/wiki/DataMapper_(Ruby)" title="DataMapper (Ruby)">DataMapper</a> and <a class="new" href="/w/index.php?title=Sequel_(Ruby)&amp;action=edit&amp;redlink=1" title="Sequel (Ruby) (page does not exist)">Sequel</a> show major improvements over the original ActiveRecord framework.<sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Neutral_point_of_view" title="Wikipedia:Neutral point of view"><span title="This statement is possibly biased. (May 2012)">neutrality</span></a> is <a href="/wiki/Talk:Active_record_pattern" title="Talk:Active record pattern">disputed</a></i>]</sup> As a response to their release and adoption by the Rails community, Ruby on Rails v3.0 is independent of an ORM system, so Rails users can easily plug in DataMapper or Sequel to use as their ORM of choice.</p>
<h3><span class="mw-headline" id="Java">Java</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Active_record_pattern&amp;action=edit&amp;section=5" title="Edit section: Java">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java language</a> implements the Active Record pattern via the <a href="/wiki/ActiveJDBC" title="ActiveJDBC">ActiveJDBC</a> library. ActiveJDBC is an implementation of Active Record design pattern inspired by <a href="/wiki/Ruby_on_Rails" title="Ruby on Rails">Ruby on Rails</a> ActiveRecord. ActiveJDBC is lightweight, fast, small and does not require any configuration.</p>
<p><a href="/wiki/ActiveJPA" title="ActiveJPA">ActiveJPA</a> and <a href="/wiki/Java_Object_Oriented_Querying" title="Java Object Oriented Querying">jOOQ (for Java Object Oriented Querying)</a> implements the Active record pattern, combining active records with <a href="/wiki/Automatic_programming" title="Automatic programming">source code generation</a> and a querying DSL similar to <a href="/wiki/SQL" title="SQL">SQL</a> allowing for retrieving active records using complex SQL statements.</p>
<p>The <a class="mw-redirect" href="/wiki/Play_framework" title="Play framework">Play framework</a> is a Java web framework which implements the Active Record pattern, using ideas from <a href="/wiki/Ruby_on_Rails" title="Ruby on Rails">Ruby on Rails</a>.</p>
<p><a class="external text" href="https://github.com/doe300/jactiverecord" rel="nofollow">JActiveRecord</a> is yet another library providing easy <a href="/wiki/Object-relational_mapping" title="Object-relational mapping">ORM</a> mapping for Java, inspired by <a href="/wiki/Ruby_on_Rails" title="Ruby on Rails">Ruby on Rails</a> ActiveRecord but more focused on Java's type-safety.</p>
<h3><span class="mw-headline" id="Dart">Dart</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Active_record_pattern&amp;action=edit&amp;section=6" title="Edit section: Dart">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The <a href="/wiki/Dart_(programming_language)" title="Dart (programming language)">Dart</a> language implements the Active Record pattern via the <a class="external text" href="https://github.com/HannesRammer/Dartabase/tree/master/dartabase_migration" rel="nofollow">Dartabase Migration</a> and <a class="external text" href="https://github.com/HannesRammer/Dartabase/tree/master/dartabase_model" rel="nofollow">Dartabase Model</a> packages</p>
<p><a class="external text" href="https://github.com/HannesRammer/Dartabase/tree/master/dartabase_migration" rel="nofollow">Dartabase Migration</a> Serverside Database Object Models for simple data manipulation currently supporting MySQL/PGSQL inspired by Ruby on Rails Migration</p>
<p><a class="external text" href="https://github.com/HannesRammer/Dartabase/tree/master/dartabase_model" rel="nofollow">Dartabase Model</a> Serverside Database migration for simple version controlled database structure manipulation currently supporting MySQL/PGSQL inspired by Ruby on Rails Model now console and GUI (build in Polymer 1.0)</p>
<h3><span class="mw-headline" id="Other_languages">Other languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Active_record_pattern&amp;action=edit&amp;section=7" title="Edit section: Other languages">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>There are several open source implementations of the Active Record pattern in other languages, including <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> (e.g., <a class="new" href="/w/index.php?title=ActiveJS&amp;action=edit&amp;redlink=1" title="ActiveJS (page does not exist)">ActiveJS</a>'s Active Record<sup class="reference" id="cite_ref-ActiveJS_13-0"><a href="#cite_note-ActiveJS-13">[13]</a></sup>), <a href="/wiki/Perl" title="Perl">Perl</a> (<a href="/wiki/DBIx::Class" title="DBIx::Class">DBIx::Class</a>), <a href="/wiki/ActionScript" title="ActionScript">ActionScript</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/Haxe" title="Haxe">Haxe</a> (SPOD<sup class="reference" id="cite_ref-HaxeSPOD_14-0"><a href="#cite_note-HaxeSPOD-14">[14]</a></sup>), <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a>,<sup class="reference" id="cite_ref-CastleActiveRecord_15-0"><a href="#cite_note-CastleActiveRecord-15">[15]</a></sup> <a href="/wiki/Objective-C" title="Objective-C">Objective-C</a><sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup> and <a href="/wiki/Scala_(programming_language)" title="Scala (programming language)">Scala</a>.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup></p>
<h2><span class="mw-headline" id="Criticism">Criticism</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Active_record_pattern&amp;action=edit&amp;section=8" title="Edit section: Criticism">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Testability">Testability</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Active_record_pattern&amp;action=edit&amp;section=9" title="Edit section: Testability">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Due to the coupling of database interaction and application logic when using the active record pattern, unit testing an active record object without a database becomes difficult<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2015)">citation needed</span></a></i>]</sup>. The negative effects on testability in the active record pattern can be minimized by using <a href="/wiki/Mock_object" title="Mock object">mocking</a> or <a href="/wiki/Dependency_injection" title="Dependency injection">dependency injection</a> frameworks to substitute the real data tier with a simulated one<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2015)">citation needed</span></a></i>]</sup>.</p>
<h3><span class="mw-headline" id="Single_Responsibility_Principle_and_Separation_of_Concerns">Single Responsibility Principle and Separation of Concerns</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Active_record_pattern&amp;action=edit&amp;section=10" title="Edit section: Single Responsibility Principle and Separation of Concerns">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Another critique of the active record pattern is that, also due to the strong coupling of database interaction and application logic, an active record object does not follow the <a href="/wiki/Single_responsibility_principle" title="Single responsibility principle">single responsibility principle</a> and <a href="/wiki/Separation_of_concerns" title="Separation of concerns">separation of concerns</a> as opposed to <a href="/wiki/Multitier_architecture" title="Multitier architecture">multitier architecture</a> which properly addresses these practices. Because of this, the active record pattern is best and most often employed in simple applications that are all forms-over-data with <a href="/wiki/Create,_read,_update_and_delete" title="Create, read, update and delete">CRUD</a> functionality, or only as one part of an architecture. Typically that part is data access and why several ORMs implement the active record pattern.</p>
