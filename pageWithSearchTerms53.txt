<h1 class="firstHeading" id="firstHeading" lang="en">Front controller</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">

<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><p>The <b>front controller</b> <a href="/wiki/Software_design_pattern" title="Software design pattern">software design pattern</a> is listed in several pattern catalogs and related to the design of web applications. It is "a controller that handles all requests for a Web site",<sup class="reference" id="cite_ref-:0_1-0"><a href="#cite_note-:0-1">[1]</a></sup> which is a useful structure for web application developers to achieve the flexibility and reuse without code redundancy.</p>

<h2><span class="mw-headline" id="Instruction">Instruction</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Front_controller&amp;action=edit&amp;section=1" title="Edit section: Instruction">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Front_Controller.svg"><img alt="" class="thumbimage" data-file-height="262" data-file-width="368" height="157" src="//upload.wikimedia.org/wikipedia/commons/thumb/8/8e/Front_Controller.svg/220px-Front_Controller.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/8/8e/Front_Controller.svg/330px-Front_Controller.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/8/8e/Front_Controller.svg/440px-Front_Controller.svg.png 2x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Front_Controller.svg" title="Enlarge"></a></div>
A typical front controller structure.</div>
</div>
</div>
<p>Front controllers are often used in web applications to implement workflows. While not strictly required, it is much easier to control navigation across a set of related pages (for instance, multiple pages used in an online purchase) from a front controller than it is to make the individual pages responsible for navigation.</p>
<p>The front controller may be implemented as a <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> object, or as a script in a <a class="mw-redirect" href="/wiki/Script_language" title="Script language">script language</a> like <a href="/wiki/PHP" title="PHP">PHP</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a> or <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a> that is called on every request of a web <a href="/wiki/Session_(computer_science)" title="Session (computer science)">session</a>. This script, for example an <i>index.php</i>, would handle all tasks that are common to the application or the framework, such as session handling, <a class="mw-redirect" href="/wiki/Caching_(computing)" title="Caching (computing)">caching</a>, and input filtering. Based on the specific request, it would then instantiate further objects and call methods to handle the particular task(s) required.</p>
<p>The alternative to a front controller would be individual scripts like <i>login.php</i> and <i>order.php</i> that would each then satisfy the type of request. Each script would have to duplicate code or objects that are common to all tasks. However, each script might also have more flexibility to implement the particular task required.</p>
<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Front_controller&amp;action=edit&amp;section=2" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Several web-tier application frameworks implement the front controller pattern, among them:</p>
<ul>
<li><a href="/wiki/Model%E2%80%93view%E2%80%93controller" title="Model–view–controller">MVC</a> frameworks written in <a href="/wiki/PHP" title="PHP">PHP</a>. For example, <a href="/wiki/Yii" title="Yii">Yii</a>, <a class="mw-redirect" href="/wiki/Cake_PHP" title="Cake PHP">CakePHP</a>, <a href="/wiki/Laravel" title="Laravel">Laravel</a>, <a href="/wiki/Symfony" title="Symfony">Symfony</a>, <a href="/wiki/CodeIgniter" title="CodeIgniter">CodeIgniter</a> and <a href="/wiki/Zend_Framework" title="Zend Framework">Zend Framework</a></li>
<li><a href="/wiki/Drupal" title="Drupal">Drupal</a></li>
<li><a href="/wiki/ASP.NET_MVC" title="ASP.NET MVC">ASP.NET MVC</a>.</li>
<li><a href="/wiki/Spring_Framework" title="Spring Framework">Spring Framework</a><sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></li>
<li><a class="mw-redirect" href="/wiki/Cairngorm_(Flex_framework)" title="Cairngorm (Flex framework)">Cairngorm</a> framework in <a class="mw-redirect" href="/wiki/Adobe_Flex" title="Adobe Flex">Adobe Flex</a>.</li>
<li><a href="/wiki/Yesod_(web_framework)" title="Yesod (web framework)">Yesod web application framework</a> written in <a href="/wiki/Haskell_(programming_language)" title="Haskell (programming language)">Haskell</a>.</li>
<li><a class="mw-redirect" href="/wiki/Apache_Struts" title="Apache Struts">Apache Struts</a>.</li>
</ul>
<h2><span class="mw-headline" id="Implementation">Implementation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Front_controller&amp;action=edit&amp;section=3" title="Edit section: Implementation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>To better understand front controller pattern, there is an example to implement front controller in Java.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> It can be defined in 3 components:</p>
<ol>
<li><a href="/wiki/XML" title="XML">XML</a> Mapping: files which map requests to the class that will handle the request processing.</li>
<li>Request Processor: used for dealing with the request processing (and modifying or retrieving the appropriate model).</li>
<li>Flow Manager: first get the request and the output of the processing, then determine what will show on the next page.</li>
</ol>
<h3><span class="mw-headline" id="Participants_and_responsibilities">Participants and responsibilities</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Front_controller&amp;action=edit&amp;section=4" title="Edit section: Participants and responsibilities">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<table class="wikitable">
<tr>
<th>Controller</th>
<th>Dispatcher</th>
<th>Helper</th>
<th>View</th>
</tr>
<tr>
<td>The controller is an entrance for users to handle requests in the system. It realizes authentication by playing the role of delegating helper or initiate contact retrieval.</td>
<td>Dispatchers can be used for navigation and managing the view output. Users will receive next view which is determined by the dispatcher. Dispatchers are also flexible: they can be encapsulated within the controller directly or separated to another component. The dispatcher provides a static view along with the dynamic mechanism.
<p>It also uses the RequestDispatcher object (supported in the servlet specification) and encapsulates some additional processing.</p>
</td>
<td>A helper helps view or controller to process. Thus helper can achieve various goals.
<p>At view side, the helper collects data and sometimes stores data as an intermediate station. Before view's process, helpers serve to adapt the data model for it. Helpers do certain pre-processes such as formatting the data to Web content or providing direct access to the raw data. Multiple helpers can collaborate with one view for most conditions. They are implemented as <a href="/wiki/JavaBeans_Activation_Framework" title="JavaBeans Activation Framework">JavaBeans</a> components in <a href="/wiki/JavaServer_Pages" title="JavaServer Pages">JSP</a> 1.0+ and custom tags in JSP 1.1+. Additionally, a helper also works as a transformer which is used to adapt and convert the model into the suitable format.</p>
</td>
<td>With the collaboration of helpers, view displays information to the client. It processes data from a model. The view will display if the processing succeeds and vice versa.</td>
</tr>
</table>

<h2><span class="mw-headline" id="Benefits_and_liabilities">Benefits and liabilities</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Front_controller&amp;action=edit&amp;section=6" title="Edit section: Benefits and liabilities">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>There are three benefits for using front controller pattern.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<ul>
<li><b>Centralized control</b>. Front controller handles all the requests to the web application. This implementation of centralized control which avoids using multiple controllers is desirable for enforcing application-wide policies such as users tracking and security.</li>
<li><b>Thread-safety</b>. A new command object arises when receiving a new request and the command objects are not meant to be thread safe. Thus, it will be safe in the command classes. Though safety is not guaranteed when threading issues are gathered, codes that act with command is still thread safe.</li>
<li><b>Configurability</b>. Since only one front controller is needed in web application, the configuration of web applications implementation is largely simplified. The handler accomplishes the rest of dispatching so that it is not required to change anything before adding new commands with dynamic ones.</li>
</ul>
<p>In terms of liability, front controllers that determine the following activities by searching the database or XML documents, performance might be decreased. And implementation of front controller to existed systems always involving replacing the current ones, which makes it harder for beginners to start with.</p>
<h2><span class="mw-headline" id="Relationship_with_MVC_pattern">Relationship with MVC pattern</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Front_controller&amp;action=edit&amp;section=7" title="Edit section: Relationship with MVC pattern">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ol>
<li>In order to improve system reliability and maintainability, duplicated codes should be avoided and centralized when they are of the same common logic through the whole system.</li>
<li>The data for the application is better to be handled in one location, thus there will be no need to duplicate database retrieval code.</li>
<li>Different roles in the MVC pattern should be separated to increase testability, which is also true for controller part in the MVC pattern.</li>
</ol>
<h2><span class="mw-headline" id="Comparison">Comparison</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Front_controller&amp;action=edit&amp;section=8" title="Edit section: Comparison">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Page controller is an alternative to front controller in MVC model.</p>
<table class="wikitable">
<tr>
<th></th>
<th>Page Controller</th>
<th>Front Controller</th>
</tr>
<tr>
<td>Base class</td>
<td>Base class is needed and will grow simultaneously with the development of the application.</td>
<td>The centralization of solving all requests is easier to modify than base class method.</td>
</tr>
<tr>
<td>Security</td>
<td>Low security because various objects react differently without consistency.</td>
<td>High. The controller is implemented in coordinated fashion, making the application safer.</td>
</tr>
<tr>
<td>Logical Page</td>
<td>Single object on each logical page.</td>
<td>Only one controller handles all requests.</td>
</tr>
