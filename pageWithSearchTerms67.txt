<p>In <a href="/wiki/Software_engineering" title="Software engineering">software engineering</a>, the <b>servant pattern</b> defines an object used to offer some functionality to a group of <a class="mw-redirect" href="/wiki/Class_(computer_science)" title="Class (computer science)">classes</a> without defining that functionality in each of them. A Servant is a class whose <a href="/wiki/Object_(computer_science)" title="Object (computer science)">instance</a> (or even just class) provides <a class="mw-redirect" href="/wiki/Method_(computer_science)" title="Method (computer science)">methods</a> that take care of a desired service, while objects for which (or with whom) the servant does something, are taken as <a class="mw-redirect" href="/wiki/Parameter_(computer_science)" title="Parameter (computer science)">parameters</a>.</p>
<p></p>

<h2><span class="mw-headline" id="Description_and_simple_example">Description and simple example</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Servant_(design_pattern)&amp;action=edit&amp;section=1" title="Edit section: Description and simple example">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Servant is used for providing some behavior to a group of classes. Instead of defining that behavior in each class - or when we cannot factor out this behavior in the common parent class - it is defined once in the Servant.</p>
<p>For example: we have a few classes representing geometric objects (rectangle, ellipse, and triangle). We can draw these objects on some canvas. When we need to provide a “move” method for these objects we could implement this method in each class, or we can define an interface they implement and then offer the “move” functionality in a servant. An interface is defined to ensure that serviced classes have methods, that servant needs to provide desired behavior. If we continue in our example, we define an Interface “Movable” specifying that every class implementing this interface needs to implement method “getPosition” and “setPosition”. The first method gets the position of an object on a canvas and second one sets the position of an object and draws it on a canvas. Then we define a servant class “MoveServant”, which has two methods “moveTo(Movable movedObject, Position where)” and moveBy(Movable movedObject, int dx, int dy). The Servant class can now be used to move every object which implements the Movable. Thus the “moving” code appears in only one class which respects the “Separation of Concerns” rule.</p>
<h2><span class="mw-headline" id="Two_ways_of_implementation">Two ways of implementation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Servant_(design_pattern)&amp;action=edit&amp;section=2" title="Edit section: Two ways of implementation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>There are two ways to implement this design pattern.</p>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:DesignPatternServantFigure1.png"><img alt="" class="thumbimage" data-file-height="244" data-file-width="364" height="147" src="//upload.wikimedia.org/wikipedia/commons/thumb/3/38/DesignPatternServantFigure1.png/220px-DesignPatternServantFigure1.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/3/38/DesignPatternServantFigure1.png/330px-DesignPatternServantFigure1.png 1.5x, //upload.wikimedia.org/wikipedia/commons/3/38/DesignPatternServantFigure1.png 2x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:DesignPatternServantFigure1.png" title="Enlarge"></a></div>
Figure 1: User uses servant to achieve some functionality and passes the serviced objects as parameters.</div>
</div>
</div>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:DesignPatternServantFigure2.png"><img alt="" class="thumbimage" data-file-height="186" data-file-width="444" height="92" src="//upload.wikimedia.org/wikipedia/commons/thumb/4/41/DesignPatternServantFigure2.png/220px-DesignPatternServantFigure2.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/4/41/DesignPatternServantFigure2.png/330px-DesignPatternServantFigure2.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/4/41/DesignPatternServantFigure2.png/440px-DesignPatternServantFigure2.png 2x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:DesignPatternServantFigure2.png" title="Enlarge"></a></div>
Figure 2: User requests operations from serviced instances, which then asks servant to do it for them.</div>
</div>
</div>
<ol>
<li>User knows the servant (in which case he doesn’t need to know the serviced classes) and sends messages with his requests to the servant instances, passing the serviced objects as parameters.</li>
<li>Serviced instances know the servant and the user sends them messages with his requests (in which case she doesn’t have to know the servant). The serviced instances then send messages to the instances of servant, asking for service.</li>
<li>The serviced classes (geometric objects from our example) don’t know about servant, but they implement the “IServiced” interface. The user class just calls the method of servant and passes serviced objects as parameters. This situation is shown on figure 1.</li>
<li>On figure 2 is shown opposite situation, where user don’t know about servant class and calls directly serviced classes. Serviced classes then asks servant themselves to achieve desired functionality.</li>
</ol>
<h2><span class="mw-headline" id="How_to_implement_Servant">How to implement Servant</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Servant_(design_pattern)&amp;action=edit&amp;section=3" title="Edit section: How to implement Servant">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<ol>
<li>Analyze what behavior servant should take care of. State what methods servant will define and what these methods will need from serviced parameter. By other words, what serviced instance must provide, so that servants methods can achieve their goals.</li>
<li>Analyze what abilities serviced classes must have, so they can be properly serviced.</li>
<li>We define an interface, which will enforce implementation of declared methods.</li>
<li>Define an interface specifying requested behavior of serviced objects. If some instance wants to be served by servant, it must implement this interface.</li>
<li>Define (or acquire somehow) specified servant (his class).</li>
<li>Implement defined interface with serviced classes.</li>
</ol>
<h2><span class="mw-headline" id="Example">Example</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Servant_(design_pattern)&amp;action=edit&amp;section=4" title="Edit section: Example">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>This simple example shows the situation described above. This example is only illustrative and will not offer any actual drawing of geometric objects, nor specification of what they look like.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">// Servant class, offering its functionality to classes implementing</span>
<span class="c1">// Movable Interface</span>
<span class="kd">public</span> <span class="kd">class</span> <span class="nc">MoveServant</span> <span class="o">{</span>
	<span class="c1">// Method, which will move Movable implementing class to position where</span>
	<span class="kd">public</span> <span class="kt">void</span> <span class="nf">moveTo</span><span class="o">(</span><span class="n">Movable</span> <span class="n">serviced</span><span class="o">,</span> <span class="n">Position</span> <span class="n">where</span><span class="o">)</span> <span class="o">{</span>
		<span class="c1">// Do some other stuff to ensure it moves smoothly and nicely, this is</span>
		<span class="c1">// the place to offer the functionality</span>
		<span class="n">serviced</span><span class="o">.</span><span class="na">setPosition</span><span class="o">(</span><span class="n">where</span><span class="o">);</span>
	<span class="o">}</span>

	<span class="c1">// Method, which will move Movable implementing class by dx and dy</span>
	<span class="kd">public</span> <span class="kt">void</span> <span class="nf">moveBy</span><span class="o">(</span><span class="n">Movable</span> <span class="n">serviced</span><span class="o">,</span> <span class="kt">int</span> <span class="n">dx</span><span class="o">,</span> <span class="kt">int</span> <span class="n">dy</span><span class="o">)</span> <span class="o">{</span>
		<span class="c1">// this is the place to offer the functionality</span>
		<span class="n">dx</span> <span class="o">+=</span> <span class="n">serviced</span><span class="o">.</span><span class="na">getPosition</span><span class="o">().</span><span class="na">xPosition</span><span class="o">;</span>
		<span class="n">dy</span> <span class="o">+=</span> <span class="n">serviced</span><span class="o">.</span><span class="na">getPosition</span><span class="o">().</span><span class="na">yPosition</span><span class="o">;</span>
		<span class="n">serviced</span><span class="o">.</span><span class="na">setPosition</span><span class="o">(</span><span class="k">new</span> <span class="n">Position</span><span class="o">(</span><span class="n">dx</span><span class="o">,</span> <span class="n">dy</span><span class="o">));</span>
	<span class="o">}</span>
<span class="o">}</span>

<span class="c1">// Interface specifying what serviced classes needs to implement, to be</span>
<span class="c1">// serviced by servant.</span>
<span class="kd">public</span> <span class="kd">interface</span> <span class="nc">Movable</span> <span class="o">{</span>
	<span class="kd">public</span> <span class="kt">void</span> <span class="nf">setPosition</span><span class="o">(</span><span class="n">Position</span> <span class="n">p</span><span class="o">);</span>

	<span class="kd">public</span> <span class="n">Position</span> <span class="nf">getPosition</span><span class="o">();</span>
<span class="o">}</span>

<span class="c1">// One of geometric classes</span>
<span class="kd">public</span> <span class="kd">class</span> <span class="nc">Triangle</span> <span class="kd">implements</span> <span class="n">Movable</span> <span class="o">{</span>
	<span class="c1">// Position of the geometric object on some canvas</span>
	<span class="kd">private</span> <span class="n">Position</span> <span class="n">p</span><span class="o">;</span>

        <span class="c1">// Method, which sets position of geometric object</span>
	<span class="kd">public</span> <span class="kt">void</span> <span class="nf">setPosition</span><span class="o">(</span><span class="n">Position</span> <span class="n">p</span><span class="o">)</span> <span class="o">{</span>
		<span class="k">this</span><span class="o">.</span><span class="na">p</span> <span class="o">=</span> <span class="n">p</span><span class="o">;</span>
	<span class="o">}</span>

<h2><span class="mw-headline" id="Similar_design_pattern:_Command">Similar design pattern: Command</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Servant_(design_pattern)&amp;action=edit&amp;section=5" title="Edit section: Similar design pattern: Command">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Design patterns <a href="/wiki/Command_pattern" title="Command pattern">Command</a> and Servant are very similar and implementations of them are often virtually the same. The difference between them is the approach to the problem.</p>
<ul>
<li>For the Servant pattern we have some objects to which we want to offer some functionality. We create a class whose instances offer that functionality and which defines an interface that serviced objects must implement. Serviced instances are then passed as parameters to the servant.</li>
<li>For the Command pattern we have some objects that we want to modify with some functionality. So, we define an interface which commands which desired functionality must be implemented. Instances of those commands are then passed to original objects as parameters of their methods.</li>
</ul>
<p>Even though design patterns Command and Servant are similar it doesn’t mean it’s always like that. There are a number of situations where use of design pattern Command doesn’t relate to the design pattern Servant. In these situations we usually need to pass to called methods just a reference to another method, which it will need in accomplishing its goal. Since we can’t pass references to methods in many languages, we have to pass an object implementing an interface which declares the signature of passed method.</p>
