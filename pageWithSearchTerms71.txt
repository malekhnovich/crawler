<h1 class="firstHeading" id="firstHeading" lang="en">Strategy pattern</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><p>In <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a>, the <b>strategy pattern</b> (also known as the <b>policy pattern</b>) is a <a class="mw-redirect" href="/wiki/Design_pattern_(computer_science)" title="Design pattern (computer science)">software design pattern</a> that enables an <a href="/wiki/Algorithm" title="Algorithm">algorithm</a>'s behavior to be selected at runtime. The strategy pattern</p>
<ul>
<li>defines a family of algorithms,</li>
<li>encapsulates each algorithm, and</li>
<li>makes the algorithms interchangeable within that family.</li>
</ul>
<p>Strategy lets the algorithm vary independently from clients that use it.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> Strategy is one of the patterns included in the influential book <i><a href="/wiki/Design_Patterns" title="Design Patterns">Design Patterns</a></i> by Gamma et al. that popularized the concept of using patterns to describe software design.</p>
<p>For instance, a class that performs validation on incoming data may use a strategy pattern to select a validation algorithm based on the type of data, the source of the data, user choice, or other discriminating factors. These factors are not known for each case until run-time, and may require radically different validation to be performed. The validation strategies, encapsulated separately from the validating object, may be used by other validating objects in different areas of the system (or even different systems) without code duplication.</p>
<p>The essential requirement in the <a href="/wiki/Programming_language" title="Programming language">programming language</a> is the ability to store a reference to some code in a data structure and retrieve it. This can be achieved by mechanisms such as the native <a href="/wiki/Function_pointer" title="Function pointer">function pointer</a>, the <a href="/wiki/First-class_function" title="First-class function">first-class function</a>, classes or class instances in <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented programming</a> languages, or accessing the language implementation's internal storage of code via <a class="mw-redirect" href="/wiki/Reflection_(computer_science)" title="Reflection (computer science)">reflection</a>.</p>
<p></p>

        <span class="k">this</span><span class="o">.</span><span class="na">drinks</span> <span class="o">=</span> <span class="k">new</span> <span class="n">ArrayList</span><span class="o">&lt;</span><span class="n">Double</span><span class="o">&gt;();</span>
        <span class="k">this</span><span class="o">.</span><span class="na">strategy</span> <span class="o">=</span> <span class="n">strategy</span><span class="o">;</span>
    <span class="o">}</span>

    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">add</span><span class="o">(</span><span class="kt">double</span> <span class="n">price</span><span class="o">,</span> <span class="kt">int</span> <span class="n">quantity</span><span class="o">)</span> <span class="o">{</span>
        <span class="n">drinks</span><span class="o">.</span><span class="na">add</span><span class="o">(</span><span class="n">strategy</span><span class="o">.</span><span class="na">getActPrice</span><span class="o">(</span><span class="n">price</span> <span class="o">*</span> <span class="n">quantity</span><span class="o">));</span>
    <span class="o">}</span>

    <span class="c1">// Payment of bill</span>
    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">printBill</span><span class="o">()</span> <span class="o">{</span>
        <span class="kt">double</span> <span class="n">sum</span> <span class="o">=</span> <span class="mi">0</span><span class="o">;</span>
        <span class="k">for</span> <span class="o">(</span><span class="n">Double</span> <span class="n">i</span> <span class="o">:</span> <span class="n">drinks</span><span class="o">)</span> <span class="o">{</span>
            <span class="n">sum</span> <span class="o">+=</span> <span class="n">i</span><span class="o">;</span>
        <span class="o">}</span>
        <span class="n">System</span><span class="o">.</span><span class="na">out</span><span class="o">.</span><span class="na">println</span><span class="o">(</span><span class="s">"Total due: "</span> <span class="o">+</span> <span class="n">sum</span><span class="o">);</span>
        <span class="n">drinks</span><span class="o">.</span><span class="na">clear</span><span class="o">();</span>
    <span class="o">}</span>

    <span class="c1">// Set Strategy</span>
    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">setStrategy</span><span class="o">(</span><span class="n">BillingStrategy</span> <span class="n">strategy</span><span class="o">)</span> <span class="o">{</span>
        <span class="k">this</span><span class="o">.</span><span class="na">strategy</span> <span class="o">=</span> <span class="n">strategy</span><span class="o">;</span>
    <span class="o">}</span>

<span class="o">}</span>

<span class="kd">interface</span> <span class="nc">BillingStrategy</span> <span class="o">{</span>
    <span class="kd">public</span> <span class="kt">double</span> <span class="nf">getActPrice</span><span class="o">(</span><span class="kt">double</span> <span class="n">rawPrice</span><span class="o">);</span>
<span class="o">}</span>

<span class="c1">// Normal billing strategy (unchanged price)</span>
<span class="kd">class</span> <span class="nc">NormalStrategy</span> <span class="kd">implements</span> <span class="n">BillingStrategy</span> <span class="o">{</span>

    <span class="nd">@Override</span>
    <span class="kd">public</span> <span class="kt">double</span> <span class="nf">getActPrice</span><span class="o">(</span><span class="kt">double</span> <span class="n">rawPrice</span><span class="o">)</span> <span class="o">{</span>
        <span class="k">return</span> <span class="n">rawPrice</span><span class="o">;</span>
    <span class="o">}</span>

<span class="o">}</span>

<span class="c1">// Strategy for Happy hour (50% discount)</span>
<span class="kd">class</span> <span class="nc">HappyHourStrategy</span> <span class="kd">implements</span> <span class="n">BillingStrategy</span> <span class="o">{</span>

    <span class="nd">@Override</span>
    <span class="kd">public</span> <span class="kt">double</span> <span class="nf">getActPrice</span><span class="o">(</span><span class="kt">double</span> <span class="n">rawPrice</span><span class="o">)</span> <span class="o">{</span>
        <span class="k">return</span> <span class="n">rawPrice</span> <span class="o">*</span> <span class="mf">0.5</span><span class="o">;</span>
    <span class="o">}</span>

<span class="o">}</span>
</pre></div>

<h2><span class="mw-headline" id="Strategy_and_open.2Fclosed_principle">Strategy and open/closed principle</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Strategy_pattern&amp;action=edit&amp;section=5" title="Edit section: Strategy and open/closed principle">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:402px;"><a class="image" href="/wiki/File:StrategyPattern_IBrakeBehavior.svg"><img alt="" class="thumbimage" data-file-height="270" data-file-width="530" height="204" src="//upload.wikimedia.org/wikipedia/commons/thumb/4/4b/StrategyPattern_IBrakeBehavior.svg/400px-StrategyPattern_IBrakeBehavior.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/4/4b/StrategyPattern_IBrakeBehavior.svg/600px-StrategyPattern_IBrakeBehavior.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/4/4b/StrategyPattern_IBrakeBehavior.svg/800px-StrategyPattern_IBrakeBehavior.svg.png 2x" width="400"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:StrategyPattern_IBrakeBehavior.svg" title="Enlarge"></a></div>
Accelerate and <a href="/wiki/Brake" title="Brake">brake</a> behaviors must be declared in each new <a href="/wiki/Car_model" title="Car model">car model</a>.</div>
</div>
</div>
<p>According to the strategy pattern, the behaviors of a class should not be inherited. Instead they should be encapsulated using interfaces. As an example, consider a car class. Two possible functionalities for car are <i>brake</i> and <i>accelerate</i>.</p>
<p>Since accelerate and brake behaviors change frequently between models, a common approach is to implement these behaviors in subclasses. This approach has significant drawbacks: accelerate and brake behaviors must be declared in each new Car model. The work of managing these behaviors increases greatly as the number of models increases, and requires code to be duplicated across models. Additionally, it is not easy to determine the exact nature of the behavior for each model without investigating the code in each.</p>
<p>The strategy pattern uses composition instead of inheritance. In the strategy pattern, behaviors are defined as separate interfaces and specific classes that implement these interfaces. This allows better decoupling between the behavior and the class that uses the behavior. The behavior can be changed without breaking the classes that use it, and the classes can switch between behaviors by changing the specific implementation used without requiring any significant code changes. Behaviors can also be changed at run-time as well as at design-time. For instance, a car object’s brake behavior can be changed from <code>BrakeWithABS()</code> to <code>Brake()</code> by changing the <code>brakeBehavior</code> member to:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="n">brakeBehavior</span> <span class="o">=</span> <span class="k">new</span> <span class="n">Brake</span><span class="p">();</span>
</pre></div>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="cm">/* Encapsulated family of Algorithms </span>
<span class="cm"> * Interface and its implementations</span>
<span class="cm"> */</span>
<span class="kd">public</span> <span class="kd">interface</span> <span class="nc">IBrakeBehavior</span> <span class="o">{</span>
    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">brake</span><span class="o">();</span> 
<span class="o">}</span>

<span class="kd">public</span> <span class="kd">class</span> <span class="nc">BrakeWithABS</span> <span class="kd">implements</span> <span class="n">IBrakeBehavior</span> <span class="o">{</span>
    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">brake</span><span class="o">()</span> <span class="o">{</span>
        <span class="n">System</span><span class="o">.</span><span class="na">out</span><span class="o">.</span><span class="na">println</span><span class="o">(</span><span class="s">"Brake with ABS applied"</span><span class="o">);</span>
    <span class="o">}</span>
<span class="o">}</span>

<span class="kd">public</span> <span class="kd">class</span> <span class="nc">Brake</span> <span class="kd">implements</span> <span class="n">IBrakeBehavior</span> <span class="o">{</span>
    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">brake</span><span class="o">()</span> <span class="o">{</span>
        <span class="n">System</span><span class="o">.</span><span class="na">out</span><span class="o">.</span><span class="na">println</span><span class="o">(</span><span class="s">"Simple Brake applied"</span><span class="o">);</span>
    <span class="o">}</span>
<span class="o">}</span>


<span class="cm">/* Client which can use the algorithms above interchangeably */</span>
<span class="kd">public</span> <span class="kd">abstract</span> <span class="kd">class</span> <span class="nc">Car</span> <span class="o">{</span>
    <span class="kd">protected</span> <span class="n">IBrakeBehavior</span> <span class="n">brakeBehavior</span><span class="o">;</span>
    
    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">applyBrake</span><span class="o">()</span> <span class="o">{</span>
        <span class="n">brakeBehavior</span><span class="o">.</span><span class="na">brake</span><span class="o">();</span>
    <span class="o">}</span>

    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">setBrakeBehavior</span><span class="o">(</span><span class="n">IBrakeBehavior</span> <span class="n">brakeType</span><span class="o">)</span> <span class="o">{</span>
        <span class="k">this</span><span class="o">.</span><span class="na">brakeBehavior</span> <span class="o">=</span> <span class="n">brakeType</span><span class="o">;</span>
    <span class="o">}</span>
<span class="o">}</span>

<span class="cm">/* Client 1 uses one algorithm (Brake) in the constructor */</span>
<span class="kd">public</span> <span class="kd">class</span> <span class="nc">Sedan</span> <span class="kd">extends</span> <span class="n">Car</span> <span class="o">{</span>
    <span class="kd">public</span> <span class="nf">Sedan</span><span class="o">()</span> <span class="o">{</span>
        <span class="k">this</span><span class="o">.</span><span class="na">brakeBehavior</span> <span class="o">=</span> <span class="k">new</span> <span class="n">Brake</span><span class="o">();</span>
    <span class="o">}</span>
<span class="o">}</span>

<span class="cm">/* Client 2 uses another algorithm (BrakeWithABS) in the constructor */</span>
<span class="kd">public</span> <span class="kd">class</span> <span class="nc">SUV</span> <span class="kd">extends</span> <span class="n">Car</span> <span class="o">{</span>
    <span class="kd">public</span> <span class="nf">SUV</span><span class="o">()</span> <span class="o">{</span>
        <span class="k">this</span><span class="o">.</span><span class="na">brakeBehavior</span> <span class="o">=</span> <span class="k">new</span> <span class="n">BrakeWithABS</span><span class="o">();</span>
    <span class="o">}</span>
<span class="o">}</span>


<span class="cm">/* Using the Car Example */</span>
<span class="kd">public</span> <span class="kd">class</span> <span class="nc">CarExample</span> <span class="o">{</span>
    <span class="kd">public</span> <span class="kd">static</span> <span class="kt">void</span> <span class="nf">main</span><span class="o">(</span><span class="n">String</span><span class="o">[]</span> <span class="n">args</span><span class="o">)</span> <span class="o">{</span>
        <span class="n">Car</span> <span class="n">sedanCar</span> <span class="o">=</span> <span class="k">new</span> <span class="n">Sedan</span><span class="o">();</span>
        <span class="n">sedanCar</span><span class="o">.</span><span class="na">applyBrake</span><span class="o">();</span>  <span class="c1">// This will invoke class "Brake"</span>

        <span class="n">Car</span> <span class="n">suvCar</span> <span class="o">=</span> <span class="k">new</span> <span class="n">SUV</span><span class="o">();</span> 
        <span class="n">suvCar</span><span class="o">.</span><span class="na">applyBrake</span><span class="o">();</span>    <span class="c1">// This will invoke class "BrakeWithABS"</span>

        <span class="c1">// set brake behavior dynamically</span>
        <span class="n">suvCar</span><span class="o">.</span><span class="na">setBrakeBehavior</span><span class="o">(</span> <span class="k">new</span> <span class="n">Brake</span><span class="o">()</span> <span class="o">);</span> 
        <span class="n">suvCar</span><span class="o">.</span><span class="na">applyBrake</span><span class="o">();</span>    <span class="c1">// This will invoke class "Brake" </span>
    <span class="o">}</span>
<span class="o">}</span>
</pre></div>
<p>This gives greater flexibility in design and is in harmony with the <a href="/wiki/Open/closed_principle" title="Open/closed principle">Open/closed principle</a> (OCP) that states that classes should be open for extension but closed for modification.</p>
