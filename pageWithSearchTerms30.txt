<h1 class="firstHeading" id="firstHeading" lang="en">Behavioral pattern</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><p>In <a href="/wiki/Software_engineering" title="Software engineering">software engineering</a>, <b>behavioral design patterns</b> are <a class="mw-redirect" href="/wiki/Design_pattern_(computer_science)" title="Design pattern (computer science)">design patterns</a> that identify common communication patterns between objects and realize these patterns. By doing so, these patterns increase flexibility in carrying out this communication.</p>
<p>Examples of this type of design pattern include:</p>
<ul>
<li><a class="mw-redirect" href="/wiki/Chain_of_responsibility_pattern" title="Chain of responsibility pattern">Chain of responsibility pattern</a>: Command objects are handled or passed on to other objects by logic-containing processing objects</li>
<li><a href="/wiki/Command_pattern" title="Command pattern">Command pattern</a>: Command objects encapsulate an action and its parameters</li>
<li>"Externalize the Stack": Turn a recursive function into an iterative one that uses a stack<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></li>
<li><a href="/wiki/Interpreter_pattern" title="Interpreter pattern">Interpreter pattern</a>: Implement a specialized computer language to rapidly solve a specific set of problems</li>
<li><a href="/wiki/Iterator_pattern" title="Iterator pattern">Iterator pattern</a>: Iterators are used to access the elements of an aggregate object sequentially without exposing its underlying representation</li>
<li><a href="/wiki/Mediator_pattern" title="Mediator pattern">Mediator pattern</a>: Provides a unified interface to a set of interfaces in a subsystem</li>
<li><a href="/wiki/Memento_pattern" title="Memento pattern">Memento pattern</a>: Provides the ability to restore an object to its previous state (rollback)</li>
<li><a href="/wiki/Null_Object_pattern" title="Null Object pattern">Null Object pattern</a>: Designed to act as a default value of an object</li>
<li><a href="/wiki/Observer_pattern" title="Observer pattern">Observer pattern</a>: aka Publish/Subscribe or Event Listener. Objects register to observe an event that may be raised by another object
<ul>
<li>Weak reference pattern: De-couple an observer from an observable<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></li>
</ul>
</li>
<li><a href="/wiki/Protocol_stack" title="Protocol stack">Protocol stack</a>: Communications are handled by multiple layers, which form an encapsulation hierarchy<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></li>
<li><a href="/wiki/Scheduled-task_pattern" title="Scheduled-task pattern">Scheduled-task pattern</a>: A task is scheduled to be performed at a particular interval or clock time (used in <a href="/wiki/Real-time_computing" title="Real-time computing">real-time computing</a>)</li>
<li><a href="/wiki/Single-serving_visitor_pattern" title="Single-serving visitor pattern">Single-serving visitor pattern</a>: Optimise the implementation of a visitor that is allocated, used only once, and then deleted</li>
<li><a href="/wiki/Specification_pattern" title="Specification pattern">Specification pattern</a>: Recombinable business logic in a <a href="/wiki/Boolean_algebra" title="Boolean algebra">boolean</a> fashion</li>
<li><a href="/wiki/State_pattern" title="State pattern">State pattern</a>: A clean way for an object to partially change its type at runtime</li>
<li><a href="/wiki/Strategy_pattern" title="Strategy pattern">Strategy pattern</a>: Algorithms can be selected on the fly</li>
<li><a href="/wiki/Template_method_pattern" title="Template method pattern">Template method pattern</a>: Describes the <a class="mw-redirect" href="/wiki/Program_skeleton" title="Program skeleton">program skeleton</a> of a program</li>
<li><a href="/wiki/Visitor_pattern" title="Visitor pattern">Visitor pattern</a>: A way to separate an algorithm from an object</li>
</ul>
