<h1 class="firstHeading" id="firstHeading" lang="en">Proactor pattern</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><p><b>Proactor</b> is a <a class="mw-redirect" href="/wiki/Design_pattern_(computer_science)" title="Design pattern (computer science)">software design pattern</a> for <a class="mw-redirect" href="/wiki/Event_handling" title="Event handling">event handling</a> in which long running activities are running in an asynchronous part. A <i>completion handler</i> is called after the asynchronous part has terminated. The proactor pattern can be considered to be an <a class="extiw" href="https://en.wiktionary.org/wiki/asynchronous" title="wikt:asynchronous">asynchronous</a> variant of the <a href="/wiki/Synchronization_(computer_science)" title="Synchronization (computer science)">synchronous</a> <a href="/wiki/Reactor_pattern" title="Reactor pattern">reactor pattern</a>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<p></p>

<p>Operation specific actors:</p>
<ul>
<li>The <i>Proactive Initiator</i> starts the asynchronous operation via the Asynchronous Operation Processor and defines the Completion Handler</li>
<li><i>Completion Handler</i> is a call at the end of the operation from the Asynchronous Operation Processor</li>
<li><i>Asynchronous Operation</i></li>
</ul>
<p>Standardized actors</p>
<ul>
<li>The <i>Asynchronous Operation Processor</i> controls the whole asynchronous operation</li>
<li>The <i>Completion Dispatcher</i> handles the call, depending on the execution environment.</li>
</ul>
