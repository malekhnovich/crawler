<h1 class="firstHeading" id="firstHeading" lang="en">Metamodeling</h1>

<p>A metamodel or surrogate model is a model of a model, and metamodeling is the process of generating such metamodels. <b>Metamodeling</b> or <b>meta-modeling</b> is the analysis, construction and development of the frames, rules, constraints, models and theories applicable and useful for <a href="/wiki/Scientific_modelling" title="Scientific modelling">modeling</a> a predefined class of problems. As its name implies, this concept applies the notions of <a class="mw-redirect" href="/wiki/Meta-" title="Meta-">meta-</a> and modeling in <a href="/wiki/Software_engineering" title="Software engineering">software engineering</a> and <a href="/wiki/Systems_engineering" title="Systems engineering">systems engineering</a>. Metamodels are of many types and have diverse applications. A thorough discussion is presented in the following text.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>

<h2><span class="mw-headline" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Metamodeling&amp;action=edit&amp;section=1" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A metamodel or surrogate model is a model of the model, i.e. a simplified model of an actual model of a circuit, system, or software like entity.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup><sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> Metamodel can be a mathematical relation or algorithm representing input and output relations. A <a class="mw-redirect" href="/wiki/Model_(abstract)" title="Model (abstract)">model</a> is an abstraction of phenomena in the <a class="mw-redirect" href="/wiki/Real_life_(reality)" title="Real life (reality)">real world</a>; a metamodel is yet another abstraction, highlighting properties of the model itself. A model conforms to its metamodel in the way that a computer program conforms to the grammar of the programming language in which it is written. Various types of metamodels include polynomial equations, neural network, <a href="/wiki/Kriging" title="Kriging">Kriging</a>, etc. "Metamodeling" is the construction of a collection of "concepts" (things, terms, etc.) within a certain domain. Metamodeling typically involves studying the output and input relationships and then fitting right metamodels to represent that behavior.</p>
<p>Common uses for metamodels are:</p>
<ul>
<li>As a <a class="mw-redirect" href="/wiki/Logical_schema" title="Logical schema">schema</a> for <a class="mw-redirect" href="/wiki/Semantic" title="Semantic">semantic</a> data that needs to be exchanged or stored</li>
<li>As a language that supports a particular method or process</li>
<li>As a language to express additional semantics of existing information</li>
<li>As a mechanism to create tools that work with a broad class of models at run time</li>
<li>As a schema for modeling and automatically exploring sentences of a language with applications to automated test synthesis</li>
<li>As an approximation of a higher-fidelity model for use when reducing computational cost is necessary</li>
</ul>
<p>Because of the "meta" character of metamodeling, both the <a href="/wiki/Praxis_(process)" title="Praxis (process)">praxis</a> and theory of metamodels are of relevance to <a class="extiw" href="https://en.wiktionary.org/wiki/metascience" title="wikt:metascience">metascience</a>, <a href="/wiki/Metaphilosophy" title="Metaphilosophy">metaphilosophy</a>, <a href="/wiki/Metatheory" title="Metatheory">metatheories</a> and <a href="/wiki/Systemics" title="Systemics">systemics</a>, and meta-consciousness. The concept can be useful in <a href="/wiki/Mathematics" title="Mathematics">mathematics</a>, and has practical applications in <a href="/wiki/Computer_science" title="Computer science">computer science</a> and <a href="/wiki/Computer_engineering" title="Computer engineering">computer engineering</a>/<a href="/wiki/Software_engineering" title="Software engineering">software engineering</a>. The latter are the main focus of this article.</p>

<h3><span class="mw-headline" id="Definition">Definition</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Metamodeling&amp;action=edit&amp;section=3" title="Edit section: Definition">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>In <a href="/wiki/Software_engineering" title="Software engineering">software engineering</a>, the use of <a class="mw-redirect" href="/wiki/Model_(abstract)" title="Model (abstract)">models</a> is more and more recommended. This should be contrasted with the classical code-based development techniques. A model always conforms to a unique metamodel. One of the currently most active branch of <a class="mw-redirect" href="/wiki/Model_Driven_Engineering" title="Model Driven Engineering">Model Driven Engineering</a> is the approach named <a href="/wiki/Model-driven_architecture" title="Model-driven architecture">model-driven architecture</a> proposed by <a href="/wiki/Object_Management_Group" title="Object Management Group">OMG</a>. This approach is based on the utilization of a language to write metamodels called the <a href="/wiki/Meta-Object_Facility" title="Meta-Object Facility">Meta Object Facility or MOF</a>. Typical metamodels proposed by <a href="/wiki/Object_Management_Group" title="Object Management Group">OMG</a> are <a href="/wiki/Unified_Modeling_Language" title="Unified Modeling Language">UML</a>, <a class="mw-redirect" href="/wiki/SysML" title="SysML">SysML</a>, SPEM or CWM. <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">ISO</a> has also published the standard metamodel <a href="/wiki/ISO/IEC_24744" title="ISO/IEC 24744">ISO/IEC 24744</a>.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> All the languages presented below could be defined as MOF metamodels.</p>
<h3><span class="mw-headline" id="Metadata_modeling">Metadata modeling</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Metamodeling&amp;action=edit&amp;section=4" title="Edit section: Metadata modeling">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Metadata_modeling" title="Metadata modeling">Metadata modeling</a> is a type of metamodeling used in <a href="/wiki/Software_engineering" title="Software engineering">software engineering</a> and <a href="/wiki/Systems_engineering" title="Systems engineering">systems engineering</a> for the analysis and construction of models applicable and useful to some predefined class of problems. (see also: <a href="/wiki/Data_modeling" title="Data modeling">data modeling</a>).</p>
<h3><span class="mw-headline" id="Model_transformations">Model transformations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Metamodeling&amp;action=edit&amp;section=5" title="Edit section: Model transformations">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>One important move in <a class="mw-redirect" href="/wiki/Model_Driven_Engineering" title="Model Driven Engineering">Model Driven Engineering</a> is the systematic use of <a class="mw-redirect" href="/wiki/Model_Transformation_Language" title="Model Transformation Language">Model Transformation Languages</a>. The OMG has proposed a standard for this called <a href="/wiki/QVT" title="QVT">QVT</a> for Queries/Views/Transformations. <a href="/wiki/QVT" title="QVT">QVT</a> is based on the <a href="/wiki/Meta-Object_Facility" title="Meta-Object Facility">Meta-Object Facility</a> or MOF. Among many other <a class="mw-redirect" href="/wiki/Model_Transformation_Language" title="Model Transformation Language">Model Transformation Languages</a> (MTLs), some examples of implementations of this standard are AndroMDA, <a href="/wiki/VIATRA" title="VIATRA">VIATRA</a>, <a href="/wiki/Tefkat" title="Tefkat">Tefkat</a>, <a class="new" href="/w/index.php?title=MT_(Model_Transformation_Language)&amp;action=edit&amp;redlink=1" title="MT (Model Transformation Language) (page does not exist)">MT</a>, <a class="new" href="/w/index.php?title=ManyDesigns_Portofino&amp;action=edit&amp;redlink=1" title="ManyDesigns Portofino (page does not exist)">ManyDesigns Portofino</a>.</p>
<h3><span class="mw-headline" id="Relationship_to_ontologies">Relationship to ontologies</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Metamodeling&amp;action=edit&amp;section=6" title="Edit section: Relationship to ontologies">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Meta-models are closely related to <a class="mw-redirect" href="/wiki/Ontology_(computer_science)" title="Ontology (computer science)">ontologies</a>. Both are often used to describe and analyze the relations between concepts<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup></p>
<ul>
<li>Ontologies: express something meaningful within a specified universe or <a href="/wiki/Domain_of_discourse" title="Domain of discourse">domain of discourse</a> by utilizing a grammar for using vocabulary. The grammar specifies what it means to be a well-formed statement, assertion, query, etc. (formal constraints) on how terms in the ontology’s controlled vocabulary can be used together.<sup class="reference" id="cite_ref-PidcockArticle_8-0"><a href="#cite_note-PidcockArticle-8">[8]</a></sup></li>
<li>Meta-modeling: can be considered as an <i>explicit</i> description (constructs and rules) of how a domain-specific model is built. In particular, this comprises a formalized specification of the domain-specific notations. Typically, metamodels are – and always should follow - a strict rule set.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> "A valid metamodel is an ontology, but not all ontologies are modeled <i>explicitly</i> as metamodels".<sup class="reference" id="cite_ref-PidcockArticle_8-1"><a href="#cite_note-PidcockArticle-8">[8]</a></sup></li>
</ul>
<h3><span class="mw-headline" id="Types_of_metamodels">Types of metamodels</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Metamodeling&amp;action=edit&amp;section=7" title="Edit section: Types of metamodels">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>For software engineering, several <i>types</i> of models (and their corresponding modeling activities) can be distinguished:</p>
<ul>
<li><a href="/wiki/Metadata_modeling" title="Metadata modeling">Metadata modeling</a> (MetaData Model)</li>
<li><a class="mw-redirect" href="/wiki/Meta-Process_Modeling" title="Meta-Process Modeling">Meta-Process Modeling</a> (MetaProcess Model)</li>
<li>Executable Meta-Modeling (combining both of the above and much more, as in the general purpose tool <a href="/wiki/Kermeta" title="Kermeta">Kermeta</a>)</li>
<li><a class="mw-redirect" href="/wiki/Model_Transformation_Language" title="Model Transformation Language">Model Transformation Language</a> (see below)</li>
<li><a class="new" href="/w/index.php?title=Polynomial_Metamodels&amp;action=edit&amp;redlink=1" title="Polynomial Metamodels (page does not exist)">Polynomial Metamodels</a><sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup></li>

<h3><span class="mw-headline" id="Zoos_of_metamodels">Zoos of metamodels</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Metamodeling&amp;action=edit&amp;section=8" title="Edit section: Zoos of metamodels">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>A library of similar metamodels has been called a Zoo of metamodels.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> There are several types of meta-model zoos.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> Some are expressed in ECore. Others are written in <a href="/wiki/Meta-Object_Facility" title="Meta-Object Facility">MOF</a> 1.4 - <a href="/wiki/XML_Metadata_Interchange" title="XML Metadata Interchange">XMI</a> 1.2. The metamodels expressed in <a href="/wiki/Unified_Modeling_Language" title="Unified Modeling Language">UML</a>-<a href="/wiki/XML_Metadata_Interchange" title="XML Metadata Interchange">XMI</a>1.2 may be uploaded in <a href="/wiki/Poseidon_for_UML" title="Poseidon for UML">Poseidon for UML</a>, a <a href="/wiki/Unified_Modeling_Language" title="Unified Modeling Language">UML</a> <a href="/wiki/Computer-aided_software_engineering" title="Computer-aided software engineering">CASE</a> tool.</p>
