<p>In <a href="/wiki/Software_engineering" title="Software engineering">software engineering</a>, the <b>template method pattern</b> is a <a href="/wiki/Behavioral_pattern" title="Behavioral pattern">behavioral</a> <a href="/wiki/Software_design_pattern" title="Software design pattern">design pattern</a> that defines the <a class="mw-redirect" href="/wiki/Program_skeleton" title="Program skeleton">program skeleton</a> of an <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> in an operation, deferring some steps to <a class="mw-redirect" href="/wiki/Subclass_(computer_science)" title="Subclass (computer science)">subclasses</a>.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> It lets one redefine certain steps of an algorithm without changing the algorithm's structure.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>

<h2><span class="mw-headline" id="Introduction">Introduction</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Template_method_pattern&amp;action=edit&amp;section=1" title="Edit section: Introduction">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In the <i>template method</i> of this design pattern, one or more algorithm steps can be overridden by subclasses to allow differing behaviors while ensuring that the overarching algorithm is still followed.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2016)">citation needed</span></a></i>]</sup></p>
<p>In object-oriented programming, first a class is created that provides the basic steps of an <a href="/wiki/Algorithm_design" title="Algorithm design">algorithm design</a>. These steps are implemented using <a class="mw-redirect" href="/wiki/Abstract_method" title="Abstract method">abstract methods</a>. Later on, subclasses change the abstract methods to implement real actions. Thus the general algorithm is saved in one place but the concrete steps may be changed by the subclasses.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2016)">citation needed</span></a></i>]</sup></p>
<p>The template method pattern thus manages the larger picture of task <a href="/wiki/Semantics" title="Semantics">semantics</a>, and more refined implementation details of selection and sequence of methods. This larger picture calls abstract and non-abstract methods for the task at hand. The non-abstract methods are completely controlled by the template method, but the abstract methods, implemented in subclasses, provide the pattern's expressive power and degree of freedom. Template method's abstract class may also define hook methods that may be overridden by subclasses.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup></p>
<p>Some or all of the abstract methods can be specialized in a subclass, allowing the writer of the subclass to provide particular behavior with minimal modifications to the larger semantics. The template method (that is non-abstract) remains unchanged in this pattern, ensuring that the subordinate non-abstract methods and abstract methods are called in the originally intended sequence.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2016)">citation needed</span></a></i>]</sup></p>
<p>The template method pattern occurs frequently, at least in its simplest case, where a method calls only one abstract method, with object oriented languages. If a software writer uses a <a class="mw-redirect" href="/wiki/Polymorphism_in_object-oriented_programming" title="Polymorphism in object-oriented programming">polymorphic</a> method at all, this design pattern may be a rather natural consequence. This is because a method calling an abstract or polymorphic function is simply the reason for being of the abstract or polymorphic method. The template method pattern may be used to add immediate present value to the software or with a vision to enhancements in the future. It is strongly related to the <a href="/wiki/Non-virtual_interface_pattern" title="Non-virtual interface pattern">non-virtual interface</a> (NVI) pattern.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2016)">citation needed</span></a></i>]</sup></p>
<p>The template method pattern implements the protected variations <a href="/wiki/GRASP_(object-oriented_design)" title="GRASP (object-oriented design)">GRASP</a> principle, like the <a href="/wiki/Adapter_pattern" title="Adapter pattern">adapter pattern</a> does. The difference is that adapter gives the same interface for several operations, while template method does so only for one.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2016)">citation needed</span></a></i>]</sup></p>

<h2><span class="mw-headline" id="Usage">Usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Template_method_pattern&amp;action=edit&amp;section=3" title="Edit section: Usage">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The template method is used in frameworks, where each implements the invariant parts of a domain's architecture, leaving "placeholders" for customisation options. This is an example of <a href="/wiki/Inversion_of_control" title="Inversion of control">inversion of control</a>. The template method is used for the following reasons:<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup></p>
<ul>
<li>Let subclasses implement (through <a class="mw-redirect" href="/wiki/Method_overriding_(programming)" title="Method overriding (programming)">method overriding</a>) behavior that can vary.</li>
<li>Avoid duplication in the code: the general workflow structure is implemented once in the abstract class's <a href="/wiki/Algorithm" title="Algorithm">algorithm</a>, and necessary variations are implemented in each of the subclasses.</li>
<li>Control at what point(s) <a class="mw-redirect" href="/wiki/Inheritance_(computer_science)" title="Inheritance (computer science)">subclassing</a> is allowed. As opposed to a simple polymorphic override, where the base method would be entirely rewritten allowing radical change to the workflow, only the specific details of the workflow are allowed to change.</li>
</ul>

<span class="cm">/**</span>
<span class="cm"> * An abstract class that is common to several games in</span>
<span class="cm"> * which players play against the others, but only one is</span>
<span class="cm"> * playing at a given time.</span>
<span class="cm"> */</span>

<span class="kd">abstract</span> <span class="kd">class</span> <span class="nc">Game</span> <span class="o">{</span>
    <span class="cm">/* Hook methods. Concrete implementation may differ in each subclass*/</span>
    <span class="kd">protected</span> <span class="kt">int</span> <span class="n">playersCount</span><span class="o">;</span>
    <span class="kd">abstract</span> <span class="kt">void</span> <span class="nf">initializeGame</span><span class="o">();</span>
    <span class="kd">abstract</span> <span class="kt">void</span> <span class="nf">makePlay</span><span class="o">(</span><span class="kt">int</span> <span class="n">player</span><span class="o">);</span>
    <span class="kd">abstract</span> <span class="kt">boolean</span> <span class="nf">endOfGame</span><span class="o">();</span>
    <span class="kd">abstract</span> <span class="kt">void</span> <span class="nf">printWinner</span><span class="o">();</span>

    <span class="cm">/* A template method : */</span>
    <span class="kd">public</span> <span class="kd">final</span> <span class="kt">void</span> <span class="nf">playOneGame</span><span class="o">(</span><span class="kt">int</span> <span class="n">playersCount</span><span class="o">)</span> <span class="o">{</span>
        <span class="k">this</span><span class="o">.</span><span class="na">playersCount</span> <span class="o">=</span> <span class="n">playersCount</span><span class="o">;</span>
        <span class="n">initializeGame</span><span class="o">();</span>
        <span class="kt">int</span> <span class="n">j</span> <span class="o">=</span> <span class="mi">0</span><span class="o">;</span>
        <span class="k">while</span> <span class="o">(!</span><span class="n">endOfGame</span><span class="o">())</span> <span class="o">{</span>
            <span class="n">makePlay</span><span class="o">(</span><span class="n">j</span><span class="o">);</span>
            <span class="n">j</span> <span class="o">=</span> <span class="o">(</span><span class="n">j</span> <span class="o">+</span> <span class="mi">1</span><span class="o">)</span> <span class="o">%</span> <span class="n">playersCount</span><span class="o">;</span>
        <span class="o">}</span>
        <span class="n">printWinner</span><span class="o">();</span>
    <span class="o">}</span>
<span class="o">}</span>

<span class="c1">//Now we can extend this class in order </span>
<span class="c1">//to implement actual games:</span>

<span class="kd">class</span> <span class="nc">Monopoly</span> <span class="kd">extends</span> <span class="n">Game</span> <span class="o">{</span>

    <span class="cm">/* Implementation of necessary concrete methods */</span>
    <span class="kt">void</span> <span class="nf">initializeGame</span><span class="o">()</span> <span class="o">{</span>
        <span class="c1">// Initialize players</span>
        <span class="c1">// Initialize money</span>
    <span class="o">}</span>
    <span class="kt">void</span> <span class="nf">makePlay</span><span class="o">(</span><span class="kt">int</span> <span class="n">player</span><span class="o">)</span> <span class="o">{</span>
        <span class="c1">// Process one turn of player</span>
    <span class="o">}</span>
    <span class="kt">boolean</span> <span class="nf">endOfGame</span><span class="o">()</span> <span class="o">{</span>
        <span class="c1">// Return true if game is over </span>
        <span class="c1">// according to Monopoly rules</span>
    <span class="o">}</span>
    <span class="kt">void</span> <span class="nf">printWinner</span><span class="o">()</span> <span class="o">{</span>
        <span class="c1">// Display who won</span>
    <span class="o">}</span>
    <span class="cm">/* Specific declarations for the Monopoly game. */</span>

    <span class="c1">// ...</span>
<span class="o">}</span>

<span class="kd">class</span> <span class="nc">Chess</span> <span class="kd">extends</span> <span class="n">Game</span> <span class="o">{</span>

    <span class="cm">/* Implementation of necessary concrete methods */</span>
    <span class="kt">void</span> <span class="nf">initializeGame</span><span class="o">()</span> <span class="o">{</span>

        <span class="c1">// Put the pieces on the board</span>
    <span class="o">}</span>
    <span class="kt">void</span> <span class="nf">makePlay</span><span class="o">(</span><span class="kt">int</span> <span class="n">player</span><span class="o">)</span> <span class="o">{</span>
