<h1 class="firstHeading" id="firstHeading" lang="en">State pattern</h1>
<div class="mw-body-content" id="bodyContent">
<div id="siteSub">From Wikipedia, the free encyclopedia</div>
<div id="contentSub"></div>
<div class="mw-jump" id="jump-to-nav">
					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>
</div>
<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><p>The <b>state pattern</b> is a <a href="/wiki/Behavioral_pattern" title="Behavioral pattern">behavioral</a> <a href="/wiki/Software_design_pattern" title="Software design pattern">software design pattern</a> that implements a <a class="mw-redirect" href="/wiki/State_machine" title="State machine">state machine</a> in an <a class="mw-redirect" href="/wiki/Object-oriented" title="Object-oriented">object-oriented</a> way. With the state pattern, a state machine is implemented by implementing each individual state as a derived class of the state pattern interface, and implementing state transitions by invoking <a href="/wiki/Method_(computer_programming)" title="Method (computer programming)">methods</a> defined by the pattern's superclass.</p>
<p>The state pattern can be interpreted as a <a href="/wiki/Strategy_pattern" title="Strategy pattern">strategy pattern</a> which is able to switch the current strategy through invocations of methods defined in the pattern's interface.</p>
<p>This pattern is used in <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a> to encapsulate varying behavior for the same <a href="/wiki/Object_(computer_science)" title="Object (computer science)">object</a> based on its internal state. This can be a cleaner way for an object to change its behavior at runtime without resorting to large monolithic conditional statements<sup class="reference" id="cite_ref-GOF_1-0"><a href="#cite_note-GOF-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:395</sup> and thus improve maintainability.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<p></p>

<h2><span class="mw-headline" id="Example">Example</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=State_pattern&amp;action=edit&amp;section=2" title="Edit section: Example">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Java">Java</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=State_pattern&amp;action=edit&amp;section=3" title="Edit section: Java">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The state interface and two implementations. The state’s method has a reference to the context object and is able to change its state.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kd">interface</span> <span class="nc">Statelike</span> <span class="o">{</span>

    <span class="kt">void</span> <span class="nf">writeName</span><span class="o">(</span><span class="n">StateContext</span> <span class="n">context</span><span class="o">,</span> <span class="n">String</span> <span class="n">name</span><span class="o">);</span>
    
<span class="o">}</span>

<span class="kd">class</span> <span class="nc">StateLowerCase</span> <span class="kd">implements</span> <span class="n">Statelike</span> <span class="o">{</span>

    <span class="nd">@Override</span>
    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">writeName</span><span class="o">(</span><span class="kd">final</span> <span class="n">StateContext</span> <span class="n">context</span><span class="o">,</span> <span class="kd">final</span> <span class="n">String</span> <span class="n">name</span><span class="o">)</span> <span class="o">{</span>
        <span class="n">System</span><span class="o">.</span><span class="na">out</span><span class="o">.</span><span class="na">println</span><span class="o">(</span><span class="n">name</span><span class="o">.</span><span class="na">toLowerCase</span><span class="o">());</span>
        <span class="n">context</span><span class="o">.</span><span class="na">setState</span><span class="o">(</span><span class="k">new</span> <span class="n">StateMultipleUpperCase</span><span class="o">());</span>
    <span class="o">}</span>

<span class="o">}</span>

<span class="kd">class</span> <span class="nc">StateMultipleUpperCase</span> <span class="kd">implements</span> <span class="n">Statelike</span> <span class="o">{</span>
    <span class="cm">/** Counter local to this state */</span>
    <span class="kd">private</span> <span class="kt">int</span> <span class="n">count</span> <span class="o">=</span> <span class="mi">0</span><span class="o">;</span>

    <span class="nd">@Override</span>
    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">writeName</span><span class="o">(</span><span class="kd">final</span> <span class="n">StateContext</span> <span class="n">context</span><span class="o">,</span> <span class="kd">final</span> <span class="n">String</span> <span class="n">name</span><span class="o">)</span> <span class="o">{</span>
        <span class="n">System</span><span class="o">.</span><span class="na">out</span><span class="o">.</span><span class="na">println</span><span class="o">(</span><span class="n">name</span><span class="o">.</span><span class="na">toUpperCase</span><span class="o">());</span>
        <span class="cm">/* Change state after StateMultipleUpperCase's writeName() gets invoked twice */</span>
        <span class="k">if</span><span class="o">(++</span><span class="n">count</span> <span class="o">&gt;</span> <span class="mi">1</span><span class="o">)</span> <span class="o">{</span>
            <span class="n">context</span><span class="o">.</span><span class="na">setState</span><span class="o">(</span><span class="k">new</span> <span class="n">StateLowerCase</span><span class="o">());</span>
        <span class="o">}</span>
    <span class="o">}</span>
    
<span class="o">}</span>
</pre></div>
<p>The context class has a state variable that it instantiates in an initial state, in this case <code lang="Java" xml:lang="Java">StateLowerCase</code>. In its method, it uses the corresponding methods of the state object.</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kd">class</span> <span class="nc">StateContext</span> <span class="o">{</span>
    <span class="kd">private</span> <span class="n">Statelike</span> <span class="n">myState</span><span class="o">;</span>
    <span class="n">StateContext</span><span class="o">()</span> <span class="o">{</span>
        <span class="n">setState</span><span class="o">(</span><span class="k">new</span> <span class="n">StateLowerCase</span><span class="o">());</span>
    <span class="o">}</span>

    <span class="cm">/**</span>
<span class="cm">     * Setter method for the state.</span>
<span class="cm">     * Normally only called by classes implementing the State interface.</span>
<span class="cm">     * @param newState the new state of this context</span>
<span class="cm">     */</span>
    <span class="kt">void</span> <span class="nf">setState</span><span class="o">(</span><span class="kd">final</span> <span class="n">Statelike</span> <span class="n">newState</span><span class="o">)</span> <span class="o">{</span>
        <span class="n">myState</span> <span class="o">=</span> <span class="n">newState</span><span class="o">;</span>
    <span class="o">}</span>

    <span class="kd">public</span> <span class="kt">void</span> <span class="nf">writeName</span><span class="o">(</span><span class="kd">final</span> <span class="n">String</span> <span class="n">name</span><span class="o">)</span> <span class="o">{</span>
        <span class="n">myState</span><span class="o">.</span><span class="na">writeName</span><span class="o">(</span><span class="k">this</span><span class="o">,</span> <span class="n">name</span><span class="o">);</span>
    <span class="o">}</span>
<span class="o">}</span>
</pre></div>
<p>The demonstration below shows the usage:</p>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="kd">public</span> <span class="kd">class</span> <span class="nc">DemoOfClientState</span> <span class="o">{</span>
    <span class="kd">public</span> <span class="kd">static</span> <span class="kt">void</span> <span class="nf">main</span><span class="o">(</span><span class="n">String</span><span class="o">[]</span> <span class="n">args</span><span class="o">)</span> <span class="o">{</span>
        <span class="kd">final</span> <span class="n">StateContext</span> <span class="n">sc</span> <span class="o">=</span> <span class="k">new</span> <span class="n">StateContext</span><span class="o">();</span>

        <span class="n">sc</span><span class="o">.</span><span class="na">writeName</span><span class="o">(</span><span class="s">"Monday"</span><span class="o">);</span>
        <span class="n">sc</span><span class="o">.</span><span class="na">writeName</span><span class="o">(</span><span class="s">"Tuesday"</span><span class="o">);</span>
        <span class="n">sc</span><span class="o">.</span><span class="na">writeName</span><span class="o">(</span><span class="s">"Wednesday"</span><span class="o">);</span>
        <span class="n">sc</span><span class="o">.</span><span class="na">writeName</span><span class="o">(</span><span class="s">"Thursday"</span><span class="o">);</span>
        <span class="n">sc</span><span class="o">.</span><span class="na">writeName</span><span class="o">(</span><span class="s">"Friday"</span><span class="o">);</span>
        <span class="n">sc</span><span class="o">.</span><span class="na">writeName</span><span class="o">(</span><span class="s">"Saturday"</span><span class="o">);</span>
        <span class="n">sc</span><span class="o">.</span><span class="na">writeName</span><span class="o">(</span><span class="s">"Sunday"</span><span class="o">);</span>
    <span class="o">}</span>
<span class="o">}</span>
</pre></div>
<p>With the above code, the output of <code lang="Java" xml:lang="Java">main()</code> from <code lang="Java" xml:lang="Java">DemoOfClientState</code> should be:</p>
<pre>
monday
TUESDAY
WEDNESDAY
thursday
FRIDAY
SATURDAY
sunday
</pre>
