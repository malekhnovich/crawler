<div class="hatnote" role="note">For other uses, see <a href="/wiki/Software_prototyping" title="Software prototyping">Software prototyping</a>.</div>
<div class="hatnote" role="note">Not to be confused with <a href="/wiki/Prototype-based_programming" title="Prototype-based programming">Prototype-based programming</a> or <a href="/wiki/Function_prototype" title="Function prototype">Function prototype</a>.</div>
<p>The <b>prototype pattern</b> is a creational <a class="mw-redirect" href="/wiki/Design_pattern_(computer_science)" title="Design pattern (computer science)">design pattern</a> in <a href="/wiki/Software_development" title="Software development">software development</a>. It is used when the type of <a href="/wiki/Object_(computer_science)" title="Object (computer science)">objects</a> to create is determined by a <a href="/wiki/Prototype" title="Prototype">prototypical</a> <a href="/wiki/Instance_(computer_science)" title="Instance (computer science)">instance</a>, which is cloned to produce new objects. This pattern is used to:</p>
<ul>
<li>avoid <a class="mw-redirect" href="/wiki/Subclass_(computer_science)" title="Subclass (computer science)">subclasses</a> of an object creator in the client application, like the <a href="/wiki/Abstract_factory_pattern" title="Abstract factory pattern">abstract factory pattern</a> does.</li>
<li>avoid the inherent cost of creating a new object in the standard way (e.g., using the '<a class="mw-redirect" href="/wiki/New_(C%2B%2B)" title="New (C++)">new</a>' keyword) when it is prohibitively expensive for a given application.</li>
</ul>
<p>To implement the pattern, declare an abstract <a class="mw-redirect" href="/wiki/Base_class" title="Base class">base class</a> that specifies a <a class="mw-redirect" href="/wiki/Virtual_method#Abstract_classes_and_pure_virtual_functions" title="Virtual method">pure virtual</a> <i>clone()</i> method. Any class that needs a "<a href="/wiki/Polymorphism_(computer_science)" title="Polymorphism (computer science)">polymorphic</a> <a class="mw-redirect" href="/wiki/Constructor_(computer_science)" title="Constructor (computer science)">constructor</a>" capability derives itself from the abstract base class, and implements the <i>clone()</i> operation.</p>
<p>The client, instead of writing code that invokes the "new" operator on a hard-coded class name, calls the <i>clone()</i> method on the prototype, calls a <a class="mw-redirect" href="/wiki/Factory_method" title="Factory method">factory method</a> with a <a href="/wiki/Parameter" title="Parameter">parameter</a> designating the particular concrete <a class="mw-redirect" href="/wiki/Derived_class" title="Derived class">derived class</a> desired, or invokes the <i>clone()</i> method through some mechanism provided by another design pattern.</p>
<p>The <a href="/wiki/Mitosis" title="Mitosis">mitotic division</a> of a cell — resulting in two identical cells — is an example of a prototype that plays an active role in copying itself and thus, demonstrates the Prototype pattern. When a cell splits, two cells of identical genotype result. In other words, the cell clones itself.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup></p>
<p></p>

<h2><span class="mw-headline" id="Rules_of_thumb">Rules of thumb</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Prototype_pattern&amp;action=edit&amp;section=2" title="Edit section: Rules of thumb">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Sometimes <a href="/wiki/Creational_pattern" title="Creational pattern">creational patterns</a> overlap — there are cases when either prototype or <a href="/wiki/Abstract_factory_pattern" title="Abstract factory pattern">abstract factory</a> would be appropriate. At other times they complement each other: abstract factory might store a set of prototypes from which to clone and return product objects (<a class="mw-redirect" href="/wiki/Gang_of_Four_(software)" title="Gang of Four (software)">GoF</a>, p126). Abstract factory, <a href="/wiki/Builder_pattern" title="Builder pattern">builder</a>, and prototype can use <a href="/wiki/Singleton_pattern" title="Singleton pattern">singleton</a> in their implementations. (GoF, p81, 134). Abstract factory classes are often implemented with factory methods (creation through <a class="mw-redirect" href="/wiki/Inheritance_(computer_science)" title="Inheritance (computer science)">inheritance</a>), but they can be implemented using prototype (creation through <a class="mw-redirect" href="/wiki/Delegation_(programming)" title="Delegation (programming)">delegation</a>). (GoF, p95)</p>
<p>Often, designs start out using Factory Method (less complicated, more customizable, subclasses proliferate) and evolve toward abstract factory, prototype, or builder (more flexible, more complex) as the designer discovers where more flexibility is needed. (GoF, p136)</p>
<p>Prototype does not require subclassing, but it does require an "initialize" operation. Factory method requires subclassing, but does not require initialization. (GoF, p116)</p>
<p>Designs that make heavy use of the <a href="/wiki/Composite_pattern" title="Composite pattern">composite</a> and <a href="/wiki/Decorator_pattern" title="Decorator pattern">decorator</a> patterns often can benefit from Prototype as well. (GoF, p126)</p>
<p>The rule of thumb could be that you would need to clone() an <i>Object</i> when you want to create another Object <i>at runtime</i> that is a <i>true copy</i> of the Object you are cloning. <i>True copy</i> means all the attributes of the newly created Object should be the same as the Object you are cloning. If you could have <i>instantiated</i> the class by using <i>new</i> instead, you would get an Object with all attributes as their initial values. For example, if you are designing a system for performing bank account transactions, then you would want to make a copy of the Object that holds your account information, perform transactions on it, and then replace the original Object with the modified one. In such cases, you would want to use clone() instead of new.</p>
<h2><span class="mw-headline" id="Pseudocode">Pseudocode</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Prototype_pattern&amp;action=edit&amp;section=3" title="Edit section: Pseudocode">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Let's write an occurrence browser class for a text. This class lists the occurrences of a word in a text. Such an object is expensive to create as the locations of the occurrences need an expensive process to find. So, to duplicate such an object, we use the prototype pattern:</p>
<pre>
<b>class</b> WordOccurrences <b>is</b>
  <b>field</b> occurrences <b>is</b>
    The list of the index of each occurrence of the word in the text.

  <b>constructor</b> WordOccurrences(text, word) <b>is</b>
      <b>input:</b> the <i>text</i> in which the occurrences have to be found
      <b>input:</b> the <i>word</i> that should appear in the text
    Empty the <i>occurrences</i> list
    <b>for each</b> textIndex <b>in</b> text
      isMatching := true
      <b>for each</b> wordIndex <b>in</b> word
        <b>if</b> the current word character does not match the current text character <b>then</b>
          isMatching := false
      <b>if</b> isMatching is true <b>then</b>
        Add the current textIndex into the <i>occurrences</i> list

  <b>method</b> getOneOccurrenceIndex(n) <b>is</b>
      <b>input:</b> a number to point on the <i>n</i>th occurrence.
      <b>output:</b> the index of the <i>n</i>th occurrence.
    Return the <i>n</i>th item of the <i>occurrences</i> field if any.

  <b>method</b> clone() <b>is</b>
      <b>output:</b> a WordOccurrences object containing the same data.
    Call clone() on the super class.
    On the returned object, set the <i>occurrences</i> field with the value of the local <i>occurrences</i> field.
    Return the cloned object.

text := "The prototype pattern is a creational design pattern in software development first described in design patterns, the book."
word := "pattern"d
searchEngine := new WordOccurrences(text, word)
anotherSearchEngine := searchEngine.clone()
</pre>
<p><i>(the search algorithm is not optimized; it is a basic algorithm to illustrate the pattern implementation)</i></p>
<h2><span class="mw-headline" id="C.23_Example">C# Example</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Prototype_pattern&amp;action=edit&amp;section=4" title="Edit section: C# Example">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>This pattern creates the kind of object using its prototype. In other words, while creating the object of Prototype object, the class actually creates a clone of it and returns it as prototype.You can see here, we have used MemberwiseClone method to clone the prototype when required.</p>

<pre>

<span class="k">public</span> <span class="k">class</span> <span class="nc">ConcretePrototype1</span> <span class="p">:</span> <span class="n">Prototype</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="k">override</span> <span class="n">Prototype</span> <span class="nf">Clone</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="k">return</span> <span class="p">(</span><span class="n">Prototype</span><span class="p">)</span><span class="k">this</span><span class="p">.</span><span class="n">MemberwiseClone</span><span class="p">();</span> <span class="c1">// Clones the concrete class.</span>
    <span class="p">}</span>
<span class="p">}</span>

<span class="k">public</span> <span class="k">class</span> <span class="nc">ConcretePrototype2</span> <span class="p">:</span> <span class="n">Prototype</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="k">override</span> <span class="n">Prototype</span> <span class="nf">Clone</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="k">return</span> <span class="p">(</span><span class="n">Prototype</span><span class="p">)</span><span class="k">this</span><span class="p">.</span><span class="n">MemberwiseClone</span><span class="p">();</span> <span class="c1">// Clones the concrete class.</span>

<h2><span class="mw-headline" id="PHP_Example">PHP Example</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Prototype_pattern&amp;action=edit&amp;section=6" title="Edit section: PHP Example">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="mw-highlight mw-content-ltr" dir="ltr">
<pre>
<span class="c1">// The Prototype pattern in PHP is done with the use of built-in PHP function __clone()</span>

<span class="k">abstract</span> <span class="k">class</span> <span class="nc">Prototype</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="nv">$a</span><span class="p">;</span>
    <span class="k">public</span> <span class="nv">$b</span><span class="p">;</span>
    
    <span class="k">public</span> <span class="k">function</span> <span class="nf">displayCONS</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="k">echo</span> <span class="s2">"CONS: </span><span class="si">{</span><span class="nv">$this</span><span class="o">-&gt;</span><span class="na">a</span><span class="si">}</span><span class="se">\n</span><span class="s2">"</span><span class="p">;</span>
        <span class="k">echo</span> <span class="s2">"CONS: </span><span class="si">{</span><span class="nv">$this</span><span class="o">-&gt;</span><span class="na">b</span><span class="si">}</span><span class="se">\n</span><span class="s2">"</span><span class="p">;</span>
    <span class="p">}</span>
    
    <span class="k">public</span> <span class="k">function</span> <span class="nf">displayCLON</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="k">echo</span> <span class="s2">"CLON: </span><span class="si">{</span><span class="nv">$this</span><span class="o">-&gt;</span><span class="na">a</span><span class="si">}</span><span class="se">\n</span><span class="s2">"</span><span class="p">;</span>
        <span class="k">echo</span> <span class="s2">"CLON: </span><span class="si">{</span><span class="nv">$this</span><span class="o">-&gt;</span><span class="na">b</span><span class="si">}</span><span class="se">\n</span><span class="s2">"</span><span class="p">;</span>
    <span class="p">}</span>

    <span class="k">abstract</span> <span class="k">function</span> <span class="nf">__clone</span><span class="p">();</span>
<span class="p">}</span>

<span class="k">class</span> <span class="nc">ConcretePrototype1</span> <span class="k">extends</span> <span class="nx">Prototype</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="k">function</span> <span class="nf">__construct</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">a</span> <span class="o">=</span> <span class="s2">"A1"</span><span class="p">;</span>
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">b</span> <span class="o">=</span> <span class="s2">"B1"</span><span class="p">;</span>
        
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">displayCONS</span><span class="p">();</span>
    <span class="p">}</span>

    <span class="k">function</span> <span class="nf">__clone</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">displayCLON</span><span class="p">();</span>
    <span class="p">}</span>
<span class="p">}</span>

<span class="k">class</span> <span class="nc">ConcretePrototype2</span> <span class="k">extends</span> <span class="nx">Prototype</span>
<span class="p">{</span>
    <span class="k">public</span> <span class="k">function</span> <span class="nf">__construct</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">a</span> <span class="o">=</span> <span class="s2">"A2"</span><span class="p">;</span>
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">b</span> <span class="o">=</span> <span class="s2">"B2"</span><span class="p">;</span>
        
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">displayCONS</span><span class="p">();</span>
    <span class="p">}</span>

    <span class="k">function</span> <span class="nf">__clone</span><span class="p">()</span>
    <span class="p">{</span>
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">a</span> <span class="o">=</span> <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">a</span> <span class="o">.</span><span class="s2">"-C"</span><span class="p">;</span>
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">b</span> <span class="o">=</span> <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">b</span> <span class="o">.</span><span class="s2">"-C"</span><span class="p">;</span>
        
        <span class="nv">$this</span><span class="o">-&gt;</span><span class="na">displayCLON</span><span class="p">();</span>
    <span class="p">}</span>
<span class="p">}</span>
