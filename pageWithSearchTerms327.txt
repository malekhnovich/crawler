<h1 class="firstHeading" id="firstHeading" lang="en">Integrated development environment</h1>
<div class="mw-body-content" id="bodyContent">

<p>An <b>integrated development environment</b> (<b>IDE</b>) is a <a href="/wiki/Application_software" title="Application software">software application</a> that provides comprehensive facilities to <a class="mw-redirect" href="/wiki/Computer_programmer" title="Computer programmer">computer programmers</a> for <a href="/wiki/Software_development" title="Software development">software development</a>. An IDE normally consists of a <a href="/wiki/Source_code_editor" title="Source code editor">source code editor</a>, <a href="/wiki/Build_automation" title="Build automation">build automation</a> tools and a <a href="/wiki/Debugger" title="Debugger">debugger</a>. Most modern IDEs have <a href="/wiki/Intelligent_code_completion" title="Intelligent code completion">intelligent code completion</a>. Some IDEs, such as <a href="/wiki/NetBeans" title="NetBeans">NetBeans</a> and <a href="/wiki/Eclipse_(software)" title="Eclipse (software)">Eclipse</a>, contain a <a href="/wiki/Compiler" title="Compiler">compiler</a>, <a href="/wiki/Interpreter_(computing)" title="Interpreter (computing)">interpreter</a>, or both; others, such as <a href="/wiki/SharpDevelop" title="SharpDevelop">SharpDevelop</a> and <a href="/wiki/Lazarus_(IDE)" title="Lazarus (IDE)">Lazarus</a>, do not. The boundary between an integrated development environment and other parts of the broader <i>software development environment</i> is not well-defined. Sometimes a <a class="mw-redirect" href="/wiki/Version_control_system" title="Version control system">version control system</a>, or various tools to simplify the construction of a Graphical User Interface (<a class="mw-redirect" href="/wiki/GUI" title="GUI">GUI</a>), are integrated. Many modern IDEs also have a <a href="/wiki/Class_browser" title="Class browser">class browser</a>, an <a href="/wiki/Object_browser" title="Object browser">object browser</a>, and a <a href="/wiki/Class_hierarchy" title="Class hierarchy">class hierarchy</a> <a href="/wiki/Diagram" title="Diagram">diagram</a>, for use in <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented software development</a>.</p>

<h2><span class="mw-headline" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Integrated_development_environment&amp;action=edit&amp;section=1" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Integrated development environments are designed to maximize programmer productivity by providing tight-knit components with similar <a href="/wiki/User_interface" title="User interface">user interfaces</a>. IDEs present a single program in which all development is done. This program typically provides many features for authoring, modifying, compiling, deploying and debugging software. This contrasts with software development using unrelated tools, such as <a href="/wiki/Vi" title="Vi">vi</a>, <a href="/wiki/GNU_Compiler_Collection" title="GNU Compiler Collection">GCC</a> or <a href="/wiki/Make_(software)" title="Make (software)">make</a>.</p>
<p>One aim of the IDE is to reduce the configuration necessary to piece together multiple development utilities, instead providing the same set of capabilities as a cohesive unit. Reducing that setup time can increase developer productivity, in cases where learning to use the IDE is faster than manually integrating all of the individual tools. Tighter integration of all development tasks has the potential to improve overall productivity beyond just helping with setup tasks. For example, code can be continuously parsed while it is being edited, providing instant feedback when syntax errors are introduced. That can speed learning a new programming language and its associated libraries.</p>
<p>Some IDEs are dedicated to a specific <a href="/wiki/Programming_language" title="Programming language">programming language</a>, allowing a feature set that most closely matches the <a href="/wiki/Programming_paradigm" title="Programming paradigm">programming paradigms</a> of the language. However, there are many multiple-language IDEs, such as <a href="/wiki/Eclipse_(software)" title="Eclipse (software)">Eclipse</a>, <a class="mw-redirect" href="/wiki/ActiveState_Komodo" title="ActiveState Komodo">ActiveState Komodo</a>, <a href="/wiki/IntelliJ_IDEA" title="IntelliJ IDEA">IntelliJ IDEA</a>, <a href="/wiki/MyEclipse" title="MyEclipse">MyEclipse</a>, <a class="mw-redirect" href="/wiki/Oracle_JDeveloper" title="Oracle JDeveloper">Oracle JDeveloper</a>, <a href="/wiki/NetBeans" title="NetBeans">NetBeans</a>, <a class="new" href="/w/index.php?title=Codenvy&amp;action=edit&amp;redlink=1" title="Codenvy (page does not exist)">Codenvy</a> and <a href="/wiki/Microsoft_Visual_Studio" title="Microsoft Visual Studio">Microsoft Visual Studio</a>. <a href="/wiki/Xcode" title="Xcode">Xcode</a>, <a href="/wiki/Xojo" title="Xojo">Xojo</a> and <a class="mw-redirect" href="/wiki/Embarcadero_Delphi" title="Embarcadero Delphi">Delphi</a> are dedicated to a closed language or set of programming languages.</p>
<p>While most modern IDEs are graphical, text-based IDEs such as <a href="/wiki/Turbo_Pascal" title="Turbo Pascal">Turbo Pascal</a> were in popular use before the widespread availability of windowing systems like <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> and the <a href="/wiki/X_Window_System" title="X Window System">X Window System</a> (X11). They commonly use function keys or <a href="/wiki/Keyboard_shortcut" title="Keyboard shortcut">hotkeys</a> to execute frequently used commands or macros.</p>
<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Integrated_development_environment&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright">
<div class="thumbinner" style="width:222px;"><a class="image" href="/wiki/File:Emacs-screenshot.png"><img alt="" class="thumbimage" data-file-height="614" data-file-width="672" height="201" src="//upload.wikimedia.org/wikipedia/commons/thumb/5/5a/Emacs-screenshot.png/220px-Emacs-screenshot.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/5/5a/Emacs-screenshot.png/330px-Emacs-screenshot.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/5/5a/Emacs-screenshot.png/440px-Emacs-screenshot.png 2x" width="220"/></a>
<div class="thumbcaption">
<div class="magnify"><a class="internal" href="/wiki/File:Emacs-screenshot.png" title="Enlarge"></a></div>
<a href="/wiki/GNU_Emacs" title="GNU Emacs">GNU Emacs</a>, an extensible editor that is commonly used as an IDE on <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> systems</div>
</div>
</div>
<p>IDEs initially became possible when developing via a <a href="/wiki/System_console" title="System console">console</a> or <a href="/wiki/Computer_terminal" title="Computer terminal">terminal</a>. Early systems could not support one, since programs were prepared using flowcharts, entering programs with <a href="/wiki/Punched_card" title="Punched card">punched cards</a> (or paper tape, etc.) before submitting them to a <a href="/wiki/Compiler" title="Compiler">compiler</a>. <a href="/wiki/Dartmouth_BASIC" title="Dartmouth BASIC">Dartmouth BASIC</a> was the first language to be created with an IDE (and was also the first to be designed for use while sitting in front of a console or terminal).<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (April 2016)">citation needed</span></a></i>]</sup> Its IDE (part of the <a href="/wiki/Dartmouth_Time_Sharing_System" title="Dartmouth Time Sharing System">Dartmouth Time Sharing System</a>) was command-based, and therefore did not look much like the menu-driven, graphical IDEs popular as of 2015. However it integrated editing, file management, compilation, debugging and execution in a manner consistent with a modern IDE.</p>
<p><a href="/wiki/Maestro_I" title="Maestro I">Maestro I</a> is a product from Softlab Munich and was the world's first integrated development environment<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> for software. <a href="/wiki/Maestro_I" title="Maestro I">Maestro I</a> was installed for 22,000 programmers worldwide. Until 1989, 6,000 installations existed in the <a href="/wiki/West_Germany" title="West Germany">Federal Republic of Germany</a>. Maestro was arguably the world leader in this field during the 1970s and 1980s. Today one of the last Maestro I can be found in the Museum of Information Technology at Arlington.</p>
<p>One of the first IDEs with a plug-in concept was <a href="/wiki/Softbench" title="Softbench">Softbench</a>. In 1995 <i>Computerwoche</i> commented that the use of an IDE was not well received by developers since it would fence in their creativity.</p>
<p>As of March 2015, the most popular IDEs are <a href="/wiki/Eclipse_(software)" title="Eclipse (software)">Eclipse</a> and <a class="mw-redirect" href="/wiki/Visual_Studio" title="Visual Studio">Visual Studio</a>.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup></p>
<h2><span class="mw-headline" id="Topics">Topics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Integrated_development_environment&amp;action=edit&amp;section=3" title="Edit section: Topics">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Visual_programming">Visual programming</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Integrated_development_environment&amp;action=edit&amp;section=4" title="Edit section: Visual programming">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a class="mw-redirect" href="/wiki/Visual_programming" title="Visual programming">Visual programming</a> is a usage scenario in which an IDE is generally required. Visual IDEs allow users to create new applications by moving programming, building blocks, or code nodes to create flowcharts or structure diagrams that are then compiled or interpreted. These flowcharts often are based on the <a href="/wiki/Unified_Modeling_Language" title="Unified Modeling Language">Unified Modeling Language</a>.</p>
<p>This interface has been popularized with the <a href="/wiki/Lego_Mindstorms" title="Lego Mindstorms">Lego Mindstorms</a> system, and is being actively pursued by a number of companies wishing to capitalize on the power of custom browsers like those found at <a href="/wiki/Mozilla" title="Mozilla">Mozilla</a>. <a class="mw-redirect" href="/wiki/KTechlab" title="KTechlab">KTechlab</a> supports flowcode and is a popular opensource IDE and Simulator for developing software for microcontrollers. Visual programming is also responsible for the power of <a class="mw-redirect" href="/wiki/Distributed_programming" title="Distributed programming">distributed programming</a> (cf. <a href="/wiki/LabVIEW" title="LabVIEW">LabVIEW</a> and <a href="/wiki/EICASLAB" title="EICASLAB">EICASLAB</a> software). An early visual programming system, <a href="/wiki/Max_(software)" title="Max (software)">Max</a>, was modelled after analog <a href="/wiki/Synthesizer" title="Synthesizer">synthesizer</a> design and has been used to develop real-time music performance software since the 1980s. Another early example was <a href="/wiki/Prograph" title="Prograph">Prograph</a>, a <a href="/wiki/Dataflow_programming" title="Dataflow programming">dataflow</a>-based system originally developed for the <a href="/wiki/Macintosh" title="Macintosh">Macintosh</a>. The graphical programming environment "Grape" is used to program <a class="mw-redirect" href="/wiki/Qfix_robot_kits" title="Qfix robot kits">qfix robot kits</a>.</p>
<p>This approach is also used in specialist software such as <a href="/wiki/Openlab" title="Openlab">Openlab</a>, where the end users want the flexibility of a full programming language, without the traditional learning curve associated with one.</p>
<h3><span class="mw-headline" id="Language_support">Language support</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Integrated_development_environment&amp;action=edit&amp;section=5" title="Edit section: Language support">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Some IDEs support multiple languages, such as <a href="/wiki/GNU_Emacs" title="GNU Emacs">GNU Emacs</a> based on <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> and <a href="/wiki/Emacs_Lisp" title="Emacs Lisp">Emacs Lisp</a>, and <a href="/wiki/IntelliJ_IDEA" title="IntelliJ IDEA">IntelliJ IDEA</a>, <a href="/wiki/Eclipse_(software)" title="Eclipse (software)">Eclipse</a>, <a href="/wiki/MyEclipse" title="MyEclipse">MyEclipse</a> or <a href="/wiki/NetBeans" title="NetBeans">NetBeans</a>, all based on <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, or <a href="/wiki/MonoDevelop" title="MonoDevelop">MonoDevelop</a>, based on <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a>.</p>
<p>Support for alternative languages is often provided by <a href="/wiki/Plug-in_(computing)" title="Plug-in (computing)">plugins</a>, allowing them to be installed on the same IDE at the same time. For example, <a class="new" href="/w/index.php?title=Flycheck&amp;action=edit&amp;redlink=1" title="Flycheck (page does not exist)">Flycheck</a> is a modern on-the-fly syntax checking extension for <a href="/wiki/GNU_Emacs" title="GNU Emacs">GNU Emacs</a> 24 with support for 39 languages.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup> <a href="/wiki/Eclipse_(software)" title="Eclipse (software)">Eclipse</a>, and <a class="mw-redirect" href="/wiki/Netbeans" title="Netbeans">Netbeans</a> have plugins for <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>/<a href="/wiki/C%2B%2B" title="C++">C++</a>, <a href="/wiki/Ada_(programming_language)" title="Ada (programming language)">Ada</a>, <a href="/wiki/GNAT" title="GNAT">GNAT</a> (for example <a href="/wiki/AdaGIDE" title="AdaGIDE">AdaGIDE</a>), <a href="/wiki/Perl" title="Perl">Perl</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a>, and <a href="/wiki/PHP" title="PHP">PHP</a>, which are selected between automatically based on file extension, environment or project settings.</p>
<h3><span class="mw-headline" id="Attitudes_across_different_computing_platforms">Attitudes across different computing platforms</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Integrated_development_environment&amp;action=edit&amp;section=6" title="Edit section: Attitudes across different computing platforms">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Unix" title="Unix">Unix</a> programmers can combine <a class="mw-redirect" href="/wiki/Command-line" title="Command-line">command-line</a> <a href="/wiki/POSIX" title="POSIX">POSIX</a> tools into a complete development environment, capable of developing large programs such as the <a href="/wiki/Linux_kernel" title="Linux kernel">Linux kernel</a> and its environment.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> In this sense, the entire Unix system functions as an IDE.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup> The free software <a href="/wiki/GNU" title="GNU">GNU</a> tools (<a href="/wiki/GNU_Compiler_Collection" title="GNU Compiler Collection">GNU Compiler Collection</a> (GCC), <a href="/wiki/GNU_Debugger" title="GNU Debugger">GNU Debugger</a> (gdb), and <a class="mw-redirect" href="/wiki/GNU_make" title="GNU make">GNU make</a>) are available on many platforms, including Windows.<sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup> The pervasive Unix philosphy of "everything is a text stream" enables developers who favor <a class="mw-redirect" href="/wiki/Command-line" title="Command-line">command-line</a> oriented tools can use editors with support for many of the standard Unix and GNU build tools, building an IDE with programs like <a href="/wiki/Emacs" title="Emacs">Emacs</a><sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup><sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup><sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> or <a href="/wiki/Vim_(text_editor)" title="Vim (text editor)">Vim</a>. <a href="/wiki/Data_Display_Debugger" title="Data Display Debugger">Data Display Debugger</a> is intended to be an advanced graphical front-end for many text-based <a href="/wiki/Debugger" title="Debugger">debugger</a> standard tools. Some programmers prefer managing <a href="/wiki/Makefile" title="Makefile">makefiles</a> and their derivatives to the similar code building tools included in a full IDE. For example, most contributors to the <a href="/wiki/PostgreSQL" title="PostgreSQL">PostgreSQL</a> database use <a href="/wiki/Makefile" title="Makefile">make</a> and <a href="/wiki/GNU_Debugger" title="GNU Debugger">gdb</a> directly to develop new features.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup> Even when building PostgreSQL for <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> using <a href="/wiki/Visual_C%2B%2B" title="Visual C++">Visual C++</a>, <a href="/wiki/Perl" title="Perl">Perl</a> scripts are used as a replacement for <a href="/wiki/Makefile" title="Makefile">make</a> rather than relying on any IDE features.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> Some Linux IDEs such as <a href="/wiki/Geany" title="Geany">Geany</a> attempt to provide a graphical front end to traditional build operations.</p>
<p>On the various <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> platforms, command-line tools for development are seldom used. Accordingly, there are many commercial and non-commercial products. However, each has a different design commonly creating incompatibilities. Most major compiler vendors for Windows still provide free copies of their command-line tools, including <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> (<a href="/wiki/Visual_C%2B%2B" title="Visual C++">Visual C++</a>, <a class="mw-redirect" href="/wiki/Platform_SDK" title="Platform SDK">Platform SDK</a>, <a href="/wiki/.NET_Framework" title=".NET Framework">.NET Framework</a> SDK, <a class="mw-redirect" href="/wiki/Nmake" title="Nmake">nmake</a> utility), <a href="/wiki/Embarcadero_Technologies" title="Embarcadero Technologies">Embarcadero Technologies</a> (<a class="new" href="/w/index.php?title=Bcc32&amp;action=edit&amp;redlink=1" title="Bcc32 (page does not exist)">bcc32</a> compiler, <a href="/wiki/Make_(software)" title="Make (software)">make</a> utility).</p>
<p>IDEs have always been popular on the Apple Macintosh's <a href="/wiki/Classic_Mac_OS" title="Classic Mac OS">classic Mac OS</a> and <a href="/wiki/MacOS" title="MacOS">macOS</a>, dating back to <a href="/wiki/Macintosh_Programmer%27s_Workshop" title="Macintosh Programmer's Workshop">Macintosh Programmer's Workshop</a>, <a href="/wiki/Turbo_Pascal" title="Turbo Pascal">Turbo Pascal</a>, THINK Pascal and <a href="/wiki/THINK_C" title="THINK C">THINK C</a> environments of the mid-1980s. Currently macOS programmers can choose between native IDEs like <a href="/wiki/Xcode" title="Xcode">Xcode</a> and open-source tools such as <a href="/wiki/Eclipse_(software)" title="Eclipse (software)">Eclipse</a> and <a class="mw-redirect" href="/wiki/Netbeans" title="Netbeans">Netbeans</a>. <a class="mw-redirect" href="/wiki/ActiveState_Komodo" title="ActiveState Komodo">ActiveState Komodo</a> is a proprietary multilanguage IDE supported on macOS.</p>
<p>With the advent of <a href="/wiki/Cloud_computing" title="Cloud computing">cloud computing</a>, some <a href="/wiki/Web_integrated_development_environment" title="Web integrated development environment">IDEs are available online</a> and run within web browsers.</p>
