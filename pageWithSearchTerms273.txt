<p>In <a href="/wiki/Computing" title="Computing">computing</a>, <b>object model</b> has two related but distinct meanings:</p>
<ol>
<li>The properties of <a href="/wiki/Object_(computer_science)" title="Object (computer science)">objects</a> in general in a specific computer <a href="/wiki/Programming_language" title="Programming language">programming language</a>, technology, notation or <a href="/wiki/Methodology" title="Methodology">methodology</a> that uses them. Examples are the object models of <i><a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a></i>, the <i><a href="/wiki/Component_Object_Model" title="Component Object Model">Component Object Model</a> (COM)</i>, or <i><a href="/wiki/Object-modeling_technique" title="Object-modeling technique">Object-Modeling Technique</a> (OMT)</i>. Such object models are usually defined using concepts such as <a class="mw-redirect" href="/wiki/Class_(computer_science)" title="Class (computer science)">class</a>, <a href="/wiki/Generic_function" title="Generic function">generic function</a>, <a class="mw-redirect" href="/wiki/Message_(computer_science)" title="Message (computer science)">message</a>, <a class="mw-redirect" href="/wiki/Inheritance_(computer_science)" title="Inheritance (computer science)">inheritance</a>, <a class="mw-redirect" href="/wiki/Polymorphism_in_object-oriented_programming" title="Polymorphism in object-oriented programming">polymorphism</a>, and <a href="/wiki/Information_hiding" title="Information hiding">encapsulation</a>. There is an extensive literature on formalized object models as a subset of the <a class="mw-redirect" href="/wiki/Formal_semantics_of_programming_languages" title="Formal semantics of programming languages">formal semantics of programming languages</a>.</li>
<li>A collection of objects or classes through which a <a href="/wiki/Computer_program" title="Computer program">program</a> can examine and manipulate some specific parts of its world. In other words, the object-oriented <a class="mw-redirect" href="/wiki/Interface_(computer_science)" title="Interface (computer science)">interface</a> to some service or system. Such an interface is said to be the <i>object model of</i> the represented service or system. For example, the <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a> (DOM) <a class="external autonumber" href="http://www.w3.org/DOM/" rel="nofollow">[1]</a> is a collection of objects that represent a <a href="/wiki/Web_page" title="Web page">page</a> in a <a href="/wiki/Web_browser" title="Web browser">web browser</a>, used by <a href="/wiki/Scripting_language" title="Scripting language">script</a> programs to examine and dynamically change the page. There is a <a href="/wiki/Microsoft_Excel" title="Microsoft Excel">Microsoft Excel</a> object model <a class="external autonumber" href="http://msdn2.microsoft.com/en-us/library/wss56bz7.aspx" rel="nofollow">[2]</a> for controlling Microsoft Excel from another program, and the <a href="/wiki/ASCOM_(standard)" title="ASCOM (standard)">ASCOM</a> Telescope Driver <a class="external autonumber" href="http://ascom-standards.org/standards.html" rel="nofollow">[3]</a> is an object model for controlling an astronomical telescope.</li>
</ol>
<p>An object model consists of the following important features:</p>
<dl>
<dt>Object Reference</dt>
<dd>Objects can be accessed via object references. To invoke a method in an object, the object reference and method name are given, together with any arguments.</dd>
</dl>
<dl>
<dt>Interfaces</dt>
<dd>An interface provides a definition of the signature of a set of methods without specifying their implementation. An object will provide a particular interface if its class contains code that implement the method of that interface. An interface also defines types that can be used to declare the type of variables or parameters and <a class="mw-redirect" href="/wiki/Return_value" title="Return value">return values</a> of methods.</dd>
</dl>
<dl>
<dt>Actions</dt>
<dd>An action in <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented programming</a> (OOP) is initiated by an object invoking a method in another object. An invocation can include additional information needed to carry out the method. The receiver executes the appropriate method and then returns control to the invoking object, sometimes supplying a result.</dd>
</dl>
<dl>
<dt>Exceptions</dt>
<dd>Programs can encounter various errors and unexpected conditions of varying seriousness. During the execution of the method many different problems may be discovered. Exceptions provide a clean way to deal with error conditions without complicating the code. A block of code may be defined to throw an exception whenever particular unexpected conditions or errors arise. This means that control passes to another block of code that catches the exception.</dd>
</dl>
